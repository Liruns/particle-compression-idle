## 0. 우리 현 화면 진단 (적용 대상)

직접 캡처한 우리 현 화면(`overhaul-desktop-wide.png`)에서 관찰된 "배치가 별로"의 구체적 원인:

| # | 현상 | 원인 | 대응 섹션 |
|---|---|---|---|
| D1 | 좌측 자원 카드 4개가 세로로 크고 헐겁다. 카드당 여백 과다, 정보 밀도 낮음 | 카드 패딩·높이가 정보량 대비 큼. "대시보드"가 아니라 "위젯 4개 띄움"으로 보임 | §2-A, §3-P0-1 |
| D2 | 8단 체인 테이블이 중앙 하단에 **뜬 블록**으로 분리. r 게이지와 시각적 연결 약함 | 체인이 메인 루프인데 화면 하단 1/3에 밀려 별개 패널처럼 보임 | §2-C, §3-P0-2 |
| D3 | 우측 컬럼 하단(저장 버튼 아래)이 **통째로 비어있음** | 우측 패널이 콘텐츠보다 길다. 의도된 여백이 아니라 빈 공간 | §2-A, §3-P0-4 |
| D4 | 체인 행이 평면적 — 구매 가능/잠금 상태가 텍스트 흐림으로만 구분. "구매" 버튼이 작은 알약 | 행 상태(배경 틴트·버튼 색)가 없음. 스캔 난이도 높음 | §2-C, §3-P0-2 |
| D5 | r 게이지(메인 포컬)가 주변 크롬 대비 작다. 화면을 압도하지 못함 | 게이지 직경이 중앙 컬럼 폭을 다 안 씀. 포컬 위계 부족 | §2-B, §3-P0-3 |
| D6 | 비용 컬럼이 "다음 비용" 헤더로 분리돼 버튼과 떨어짐 | 결정(얼마/비용)과 행동(구매)이 시각적으로 분리 | §2-C, §3-P0-2 |

핵심 진단: **우리 화면은 "3개의 느슨한 컬럼"이고, 성공작은 "하나의 응집된 계기판"이다.** 여백이 의도가 아니라 사고처럼 보인다. 아래 레퍼런스가 이를 어떻게 푸는지 본다.

## 1. 게임별 티어다운

각 게임을 7축으로 분해: ① 정보 아키텍처/배치 ② 포컬 포인트 ③ 생성기 리스트 ④ 진행/프레스티지 내비 ⑤ 비주얼 스타일 ⑥ 진행감/스케일 표현 ⑦ 데스크톱 vs 모바일.

### 1-A. Antimatter Dimensions — 장르의 레이아웃 교과서 ★최우선 구조 레퍼런스

URL: https://ivark.github.io/AntimatterDimensions/ (라이브 캡처: `ad-start.png`, `ad-full.png`)

8단 차원 체인 + 다층 프레스티지의 원형. 우리 8단 압축 체인의 직계 구조적 조상. 직접 8차원을 모두 구매해 풀 행 레이아웃을 캡처했다.

```
┌──────┬──────────────────────────────────────────────────────────────┐
│ 10.0 │  [중앙 상단]  You have 798.98 Oc antimatter.   ← 헤드라인 수치  │
│ Anti │               You are getting 2.18 Qa/sec      ← 생산율 1줄    │
│ ─────│  ── (얇은 구분선) ──                                          │
│ Dim. │  [Buy 10] [Until 10]          [Max All (M)]   ← 벌크구매 바    │
│ Achv │  ┌────────────────────────────────────────────────────────┐  │
│ Stat │  │ 1st Antimatter Dimension  79.62 B  +150%/s │ Buy / Cost │  │ ← 행: 좌名+배율 / 중수량+생산 / 우버튼
│ Opt  │  │ 2nd Antimatter Dimension  349 M    +130%/s │ Buy / Cost │  │   (구매가능=녹색, 불가=암적, 잠금=회색)
│ Shop │  │ 3rd … 4th …  (붉은 배경 틴트 = 보유/활성)              │  │
│      │  │ 5th…8th  (배경 거의 투명 = 잠금)         │ Locked     │  │
│      │  └────────────────────────────────────────────────────────┘  │
│      │     Dimension Boost (0)        Antimatter Galaxies (0)        │ ← 프레스티지 버튼(설명 내장)
└──────┴──────────────────────────────────────────────────────────────┘
```

1. **IA/배치**: **좌측 세로 탭 레일**(Dimensions/Achievements/Statistics/Options/Shop). 상단 탭이 아니다. 헤드라인 수치(antimatter)가 탭 레일 위 좌상단 코너에 상주. 콘텐츠는 우측 대형 영역 하나에 몰빵. → 우리는 상단 탭이지만, "헤드라인 수치 상주 + 단일 대형 콘텐츠 영역" 원칙은 동일하게 강력.
2. **포컬**: 화면 상단 정중앙의 "You have X antimatter" — 크고, 유채색(붉은 핑크), 모노 느낌 숫자. 그 아래 생산율 1줄. **포컬은 게이지가 아니라 거대한 수치 그 자체.**
3. **생성기 리스트(★우리 체인 직결)**: 8개 풀폭 행. 행 구조 = `[이름+배율(좌)] … [보유량+생산/s(중)] … [구매버튼+비용(우)]`. **비용이 버튼 안에 들어있다**("Buy 0 / Cost: 10 No AM"). 버튼 색이 상태: 녹색=구매가능, 암적=E부족, 회색="Locked". **행 배경 틴트로 활성/잠금 구분**(보유 차원=옅은 적색 밴드, 잠금=거의 투명). 행간 간격 최소 → 고밀도.
4. **프레스티지 내비**: Dimension Boost / Antimatter Galaxies = 콘텐츠 영역 내 **설명이 박힌 큰 버튼**("Reset your Dimensions to unlock the 5th Dimension and give ×2.0…"). 상위 프레스티지(Infinity/Eternity/Reality)는 탭 레일 항목으로 승격. Time Study는 별도 트리 화면(노드+연결선 분기 트리).
5. **비주얼**: 다크 배경 + 모노스페이스 + 채도 낮은 유채(핑크/녹/적). 장식 없음. 거의 텍스트. "스프레드시트가 살아있는" 룩.
6. **진행/스케일**: 하단 얇은 진행 바(다음 마일스톤까지 %, "0.32%"). 큰 수는 과학표기/접두사. 진행감은 행마다 +%/s 라이브 증가로 전달.
7. **데스크톱 vs 모바일**: 모바일은 행 요소를 세로로 쌓고, 웹은 팝업으로 분리. 텍스트 크기 조절 옵션 제공.

**우리에게**: 8단 체인 행 레이아웃을 이대로 차용. 특히 (a) 비용을 버튼 안으로, (b) 행 배경 틴트로 상태 구분, (c) 행간 압축으로 8행이 한 덩어리로 보이게.

### 1-B. Universal Paperclips — 미니멀 텍스트·단계적 공개의 정점

URL: https://www.decisionproblem.com/paperclips/ (라이브 캡처: `paperclips.png`)

```
┌───────────────────────────────────────────────────────────────┐
│ > Welcome to Universal Paperclips                  [iPhone/And] │ ← 상단 콘솔 1줄 로그
│                                                                │
│ Paperclips: 0          ← 헤드라인(굵게, 큼)                    │
│ [Make Paperclip]       ← 단일 액션 버튼                        │
│                                                                │
│ Business  ──────────   ← 섹션 헤더 + 가로 룰                   │
│ Available Funds: $0.00                                         │
│ [lower][raise] Price per Clip: $.25    ← 인라인 소형 버튼+값  │
│ [Marketing] Level: 1 / Cost: $100                             │
│                                                                │
│ Manufacturing  ───────                                        │
│ Clips per Second: 0                                           │
│ [Wire] 1,000 inches / Cost: $26                              │
│                          (화면 우측 2/3는 통째로 비어있음 — 의도)│
└───────────────────────────────────────────────────────────────┘
```

1. **IA/배치**: 게임 전체가 **좁은 좌측 1컬럼(~280px)**. 우측 2/3는 의도적 흑백 여백(후반 시스템이 채움). 섹션은 굵은 헤더+가로 룰로 구획(Business/Manufacturing/…).
2. **포컬**: "Paperclips: N" — 굵고 큰 텍스트. 바로 아래 단일 [Make Paperclip] 버튼. 포컬=수치+1차 액션.
3. **생성기 리스트**: 리스트가 아니라 **인라인 행**: `[소형 회색 버튼] 레이블: 값`. 버튼이 값과 한 줄에 붙음. 극도로 압축.
4. **진행/프레스티지 내비**: 탭 없음. 모든 게 한 컬럼에 세로로 쌓이고, 새 섹션이 **아래로 자라남**. 프로젝트는 동적으로 등장하는 버튼 그리드.
5. **비주얼**: 흰 배경·검정 텍스트·시스템 폰트. 거의 디자인을 안 한 것이 디자인. **공백에 대한 자신감.**
6. **진행/스케일**: 텍스트 수치만. "Public Demand: 32%" 같은 인라인 %.
7. **데스크톱 vs 모바일**: 좁은 단일 컬럼이라 모바일 그대로 이식.

**우리에게**: (1) **단계적 공개를 "아래로 자라는 단일 컬럼"으로** — 우리 FTUE도 이미 progressive disclosure지만, Paperclips는 빈 공간을 부끄러워하지 않는다. 우리 D3(빈 우측 하단)은 "아직 안 열림"이면 그 자체로 OK일 수 있으나, 그러려면 **나머지가 꽉 차야** 빈 곳이 "다음이 온다"로 읽힌다. (2) 섹션 헤더+가로 룰 구획법 차용.

### 1-C. Exponential Idle — 클린 다크·수식 노트북 ★우리 미감 최근접

URL: https://conicgames.github.io/exponentialidle/ (라이브 캡처: `expidle.png` — 마케팅 페이지 내 실제 게임 스크린샷)

우리 "계측기/과학 노트북 미학"과 톤이 가장 가까운 레퍼런스. 다크+모노+수식 타이포가 포컬이 될 수 있음을 증명.

```
┌─────────────────────────────────────────────┐
│ dt = 1.00      db = 3.05e6   ← 좌우 2열 key:val 리드아웃(모노/수식)
│  t = 854.00     b = 0.001                    │
│ f(t)= $ee12.9   x = 4.98e11                   │
│ ┌──────────────────────────┐  x₁ = 4.26e13  │ ← 우측 변수 리스트
│ │   f(t+dt)=f(t)×e^(bx₂dt)  │  x₂ = 3.00e15  │   (x₃…x₈ = "locked" 회색)
│ │   [스파크라인 그래프 배경] │  x₃ = locked   │
│ │  (»») ← 스텝 버튼          │  x₄ = locked   │
│ └──────────────────────────┘  …             │
│  1.000e27 ϖ  (+0.23)         ← 통화 라인     │
│ ── Variables ──────── (max) Cost ──          │ ← 표 헤더
│  x = y².⁶+2×1.8           Free / Level:18    │ ← 행: 수식(좌) / 비용+레벨(우)
│  y = z+2⁴×95          (×2) $ee12.9/Level:95  │
│  z = 16s+2⁴×77 (회색=구매불가) (×1) $ee13.0  │
└─────────────────────────────────────────────┘
```

1. **IA/배치**: 단일 세로 스크롤 컬럼(모바일 우선). 상단=수식 포컬 패널, 그 안에 리드아웃, 아래=Variables 표.
2. **포컬**: **실제 수식이 포컬**(`f(t+dt)=f(t)×e^(bx₂dt)`). 뒤에 성장 스파크라인 그래프. 좌하단 원형 스텝 버튼(»»). → 우리 r게이지의 사촌: "수치/공식이 곧 비주얼".
3. **생성기 리스트**: Variables 표. 컬럼 `Variables | (max) Cost`. 행=`수식(좌) | 비용+레벨(우)`. **비용 우측 정렬, 잠금=회색**. AD와 동일 패턴.
4. **진행/프레스티지 내비**: 상단 햄버거/탭으로 다른 화면. 기본은 단일 화면 응집.
5. **비주얼**: **다크 그레이·거의 모노크롬·수식 세리프 타이포.** 절제된 여백. 과학 계산기/노트 룩. **우리가 목표하는 미감과 거의 동일.**
6. **진행/스케일**: 스파크라인 그래프 + 과학표기 수치. `(+0.23)` 증분 인라인.
7. **데스크톱 vs 모바일**: 태생이 모바일 단일 컬럼. 데스크톱은 가운데 정렬 카드.

**우리에게**: **이것이 우리 모바일의 정답 톤이다.** (1) r게이지 패널 안에 좌우 2열 key:val 리드아웃을 박아 "계기 패널"화. (2) 다크 모노크롬 + 악센트 최소가 정답임을 확인(art-direction과 일치). (3) 우리 체인 표도 이 "수식 좌 / 비용 우" 행을 따르되 우리는 티어 배지로.

### 1-D. Cookie Clicker — 3컬럼 클리커 IA의 표준

URL: https://orteil.dashnet.org/cookieclicker/ (구조는 위키·리뷰로 확인)

```
┌────────────────┬──────────────────────┬─────────────────────┐
│  LEFT          │  CENTER              │  RIGHT              │
│  [거대 쿠키]   │  [정보/업그레이드     │  [건물 상점 리스트] │
│  (클릭 대상)   │   탭 패널]           │  Cursor   15        │
│  은행: N쿠키   │  Store/Upgrades      │  Grandma  100       │
│  초당 N        │  통계/옵션 탭        │  Farm     1.1K      │
│  (황금쿠키 등) │                      │  … (세로 스크롤)    │
└────────────────┴──────────────────────┴─────────────────────┘
```

1. **IA/배치**: 고전 3컬럼. **좌=클릭 포컬(거대 쿠키)+헤드라인 수치, 중=정보/업그레이드 탭, 우=건물 상점(구매 리스트)**.
2. **포컬**: 화면 좌측을 채우는 거대 쿠키. 그 위에 누적/초당 수치. 클릭 대상이 곧 포컬.
3. **생성기 리스트**: **우측 세로 리스트**. 각 행=`[아이콘] 이름 / 보유수(우측 큰 숫자)`, 가격은 행에. 구매 가능하면 밝게, 불가하면 어둡게. 마우스오버 툴팁으로 상세.
4. **프레스티지 내비**: "Legacy"(천상의 빵부스러기) — 별도 오버레이 트리(천체 그리드).
5. **비주얼**: 따뜻한 우드/베이킹 텍스처. 리치 아트. (우리와 반대 방향이지만 IA는 참고.)
6. **진행/스케일**: 큰 수 접두사. 건물 수가 시각적 진행.
7. **데스크톱 vs 모바일**: 모바일은 3컬럼을 탭/아코디언으로 접음.

## 2. 교차 패턴 합성

직접 캡처·분석한 11종에서 반복 출현한 레이아웃·비주얼 원칙. (✓=성공 패턴, ✗=안티패턴)

### 2-A. 정보 아키텍처 — "헤드라인 수치 상주 + 단일 콘텐츠 영역"

- ✓ **헤드라인 수치는 항상 화면 상단(중앙 또는 좌상단)에 크게 상주.** AD(상단중앙 antimatter), Paperclips(좌상단 Paperclips:N), Cookie(쿠키 위 누적), Exponential(상단 f(t) 리드아웃). 탭을 바꿔도 "지금 얼마"는 안 사라진다.
- ✓ **콘텐츠는 하나의 큰 영역에 몰빵, 내비는 가장자리 얇게.** AD/Melvor=좌측 얇은 레일 + 거대 콘텐츠. Paperclips/Exponential=단일 컬럼. 내비가 콘텐츠 공간을 잡아먹지 않는다.
- ✗ **사이드 패널이 포컬을 좁히는 것**(Cell to Singularity가 6주 재설계로 고친 병, 우리 현 상태 D1/D3).
- ✗ **탭이 수십 개로 늘어 위계가 사라짐**(NGU Idle).
- **밀도 관리 기법**: 필터 서브탭(Kittens: 전체/사용가능/활성/토글), 위젯 축소 보존(우리 ux.md에 이미 있음), 마우스오버 툴팁으로 상세 이연(Cookie/AD).

### 2-B. 포컬 포인트 — "단일 압도적 대상, 수치가 곧 비주얼"

- ✓ **화면에 단 하나의 명백한 주인공.** 거대 쿠키(Cookie), 거대 antimatter 수치(AD), 수식 패널(Exponential), 디오라마(Cell). 시선이 갈 곳에 의심이 없다.
- ✓ **수치/공식 자체를 비주얼로 키운다.** AD는 그래픽 없이 큰 색수치로, Exponential은 실제 수식+스파크라인으로 포컬을 만든다. → **우리 r게이지는 "수치를 비주얼화"한 좋은 사례지만, 충분히 크지 않다(D5).**
- ✓ **포컬은 화면 폭/높이의 상당 부분을 점유**(쿠키는 좌측 1/3, 수식 패널은 컬럼 폭 전체).

### 2-C. 생성기/구매 리스트 — "풀폭 행 + 비용 버튼 내장 + 상태 색"

성공작이 거의 만장일치로 공유하는 행 패턴(우리 8단 체인 직결):

```
[티어/이름 + 배율]      [보유량 + 생산율/s]      [비용이 박힌 구매 버튼]
   좌측, 식별              중앙, 라이브 피드백        우측, 색=상태
```

- ✓ **비용을 별도 컬럼이 아니라 구매 버튼 안에**(AD: "Buy / Cost: 10"). 결정과 행동을 한 클릭 타깃에. (우리 D6 해결)
- ✓ **버튼 색이 상태**: 녹=구매가능, 암적=자원부족, 회색=잠금(AD). 텍스트 흐림보다 훨씬 빠르게 스캔됨. (우리 D4 해결)
- ✓ **행 배경 틴트로 활성/보유/잠금 구분**(AD: 보유차원 옅은 색 밴드). 평면 행보다 위계 명확.
- ✓ **보유량은 "총량 / (구매량)" 이중 표기**(AD/Exponential의 Level 표기) — 자동생성 폭증해도 하위 티어 의미 유지.
- ✓ **벌크구매 컨트롤을 리스트 상단에 클러스터**(AD: Buy10/Until10/Max All — 우리는 ×1/×10/×100/Max를 이미 가짐, 위치만 점검).
- ✗ 행이 너무 떨어져 리스트가 "여러 카드"로 보이는 것 → 행간 압축해 한 덩어리로.

### 2-D. 진행·프레스티지 내비 — "탭→트리/맵으로 격상"

- ✓ **초기 프레스티지는 설명 박힌 큰 버튼**(AD Dimension Boost), **상위 메타는 탭/트리로 격상**(AD Infinity/Eternity/Reality, Time Study 트리).
- ✓ **트리 메타포: 노드+연결선, 분기점에서 갈림.** 구매=채워짐/유채색, 잠금=흐림, 선행=연결선. (AD Time Study, Cell to Singularity Tech Tree of Life)
- ✓ **"다음 해금"을 공간적으로 당겨 보여주기**(Cell: 빈 원으로 가는 흰 선 + 거리감). 목표를 시각적 미끼로.
- → 우리 연구 트리(ux.md §3)·도감·상전이가 이 메타포 후보. 특히 도감/연구를 "노드 트리"로 시각화하면 진행감↑.

### 2-E. 비주얼 스타일 — "다크 모노크롬 + 악센트 최소가 과학 미감의 표준"

- ✓ **우리 미감(계측기/다크/모노)은 Exponential Idle이 이미 검증.** 다크 그레이 + 수식 타이포 + 절제 여백이 "과학적"으로 읽힌다.
- ✓ **유채색은 악센트로만**(AD의 채도 낮은 핑크/녹). 배경은 침묵, 빛이 정보(우리 art-direction §1-A와 일치).
- ✓ **모노스페이스 수치 = 자리 안 흔들림 + "데이터" 느낌**(AD/Exponential).
- 스펙트럼 상 우리 위치: **Exponential Idle과 A Dark Room 사이** — Exponential의 다크 노트북 룩 + A Dark Room의 프로즈 절제. 리치 아트(Cookie/Cell) 쪽이 아니다(art-direction 거부 룩과 일치).

### 2-F. 진행감·스케일 표현 — "바 + 그래프 + 줌 + 거리감"

- ✓ **얇은 진행 바로 "다음 마일스톤까지 %"**(AD 하단 0.32%, 우리 decade 바와 동형).
- ✓ **스파크라인/미니 그래프로 성장 추세**(Exponential 수식 뒤 그래프). 우리 상전이 QF 곡선 차트(ux.md §5-4)와 동형 — 메인에도 도입 가치.
- ✓ **줌/디오라마로 스케일을 체감**(Cell). 우리 r게이지 글로우 확대 + 동심원 소멸(art-direction §3)이 우리식 줌. **거리감(다음 층까지)을 추가하면 강화.**
- ✓ **큰 수는 과학표기/접두사 + 색 코딩**(전 게임 공통, 우리 이미 함).

### 2-G. 데스크톱 vs 모바일 — "단일 컬럼이 모바일 기본, 데스크톱은 컬럼 펼침"

- ✓ **모바일 우선 게임은 태생이 단일 세로 컬럼**(Exponential, A Dark Room) → 그대로 모바일 이식.
- ✓ **데스크톱 3컬럼은 모바일에서 탭/아코디언으로 접힘**(Cookie, Kittens, Melvor). 우리 모바일 하단 탭바 + 단일 컬럼이 정석.
- ✓ **헤드라인 수치는 양 플랫폼 공통 상주.**

## 3. ★우리에 적용 (핵심) — Micro Idle 재배치 변경안

우리 3패널·r게이지·체인·층·도감을 위 패턴에 구체 매핑. 우선순위 P0(배치 "별로"를 당장 고침) → P1(강화) → P2(미래).

### 목표 레이아웃 (데스크톱) — "느슨한 3컬럼 → 응집된 계기판"

### P0 — 당장 바꿀 것 (배치 "별로"의 직접 원인)

**P0-1. 좌측 자원 "카드"를 "원장 행"으로 압축** (해결: D1)
- 현: 카드 4개, 카드당 큰 패딩/높이 → 헐겁고 정보밀도 낮음.
- 변경: **Kittens Game식 컴팩트 원장 행**. 행당 `[아이콘] 기호 값 / +생산율`을 1~2줄로. 카드 테두리/그림자 제거하고 얇은 구분선만. 4개 자원이 위쪽 1/3에 모이고, 그 아래 배율·상태로그가 자연히 따라옴.
- 근거: Kittens(자원당 1행 수십 개 상주), Exponential(좌우 2열 key:val 리드아웃). "카드"는 위젯 4개로 보이고 "원장"은 하나의 계기판으로 보인다.

**P0-2. 8단 체인 행을 AD 패턴으로 재설계 + 포컬에 붙이기** (해결: D2/D4/D6)
- 현: 중앙 하단 뜬 블록, 평면 행, 작은 "구매" 알약, 비용 별도 컬럼.
- 변경:
  - **비용을 구매 버튼 안으로**: `[Buy · 438.96 E]` 한 버튼. (Exponential/AD)
  - **버튼 색=상태**: 구매가능=녹(--col-glow-core 계열), E부족=암적(--col-reset 계열), 잠금=회색 "Locked". 텍스트 흐림만으로 구분하던 것 제거. (AD)
  - **행 배경 틴트**: 보유>0 또는 활성 티어=옅은 층악센트 밴드(opacity 0.04~0.06), 잠금=배경 거의 투명. (AD)
  - **행간 압축**: 현재 8행이 듬성한데 행 높이/간격을 줄여 **한 덩어리 리스트**로. 잠금 티어도 살짝 보이되 촘촘히.
  - **체인을 r게이지와 더 가깝게**: 압축 버튼 바로 아래 붙여 "게이지→버튼→체인"이 하나의 수직 흐름이 되게. 중앙 컬럼이 위→아래 단일 내러티브.
- 근거: AD `ad-full.png`(8행 풀폭, 비용 버튼 내장, 색 상태, 배경 틴트), Exponential Variables 표.

**P0-3. r게이지를 키워 진짜 포컬로** (해결: D5)
- 현: 게이지 직경이 중앙 컬럼 폭을 다 안 씀. 주변 크롬 대비 작음.
- 변경: 게이지 직경을 중앙 컬럼 가용 폭의 **70~80%까지 확대**. 게이지 안/주변에 **좌우 2열 미니 리드아웃**(Exponential식: 좌 `C 3.07K / +1.08`, 우 `배율 ×1.08 / dec 2.27`)을 박아 "계기 패널"로. r 수치는 게이지 바로 아래 더 크게.
- 근거: Cookie(거대 쿠키가 1/3 점유), Exponential(수식 패널이 컬럼 폭 전체 + 내부 리드아웃), 교차패턴 §2-B.

**P0-4. 우측 패널 빈 공간 해소 — 콘텐츠로 채우거나 컬럼을 좁히기** (해결: D3)
- 현: 저장 버튼 아래 우측 컬럼이 통째로 비어 "사고난 여백"처럼 보임.
- 변경(둘 중 택1, 권장은 A):
  - **A. 콘텐츠로 채움**: 층 카드 + 메커니즘 위젯 아래에 **도감 미니 진행(층별 n/전체 작은 바)**, **연구 가지 현황 1줄**, **상전이 힌트**, 그리고 하단 고정 "더 작은 것이 있다."를 배치(ui-flow.md §2-A에 이미 설계됨 — 구현이 비어있는 것). 저장은 작은 아이콘 버튼으로 탭바/설정에 흡수.
  - **B. 컬럼 폭 축소**: 우측을 160px로 좁히고 남는 폭을 중앙 포컬에 양보(Cell to Singularity 교훈: 중앙에 공간 몰아주기).
- 근거: Cell to Singularity 1차 소스(좌측 패널 과대 → 중앙에 공간 양보로 재설계), Paperclips(빈 공간은 "다음이 온다"로 읽히려면 나머지가 꽉 차야 함).

**P0-5. 헤드라인 수치 상주 강화**
- 현: 우상단 "원자층 dec 2.27"은 있으나 "지금 핵심 수치"가 헤드라인으로 안 잡힘.
- 변경: 탭바 우측 또는 r게이지 영역에 **현재 r 또는 C를 헤드라인급으로 상주**. 어느 탭(연구/도감)에 가도 "지금 얼마/어디"가 보이게. AD/Paperclips/Cookie 전부 이 원칙.
- 근거: 교차패턴 §2-A(헤드라인 수치 항상 상주).

### P1 — 강화 (포컬·진행감·테마 정합)

**P1-1. 스케일 "거리감" 시각화 — Cell to Singularity 차용**
- decade 진행 바를 넘어, r게이지 아래/주변에 **"다음 층으로 가는 통로/거리"**를 표현. 예: 다음 층 악센트 컬러의 희미한 점/링이 게이지 바깥에 있고, decade 진행에 따라 가까워짐. "Cell to Singularity의 빈 원처럼" 다음 스케일을 공간적으로 당겨 보여준다.
- 근거: Cell to Singularity(빈 원+연결선 거리감), 교차패턴 §2-F.

**P1-2. 메인에 미니 성장 그래프(스파크라인)**
- Exponential Idle처럼 r게이지 영역 또는 C 자원 옆에 최근 생산율 스파크라인. 방치 중에도 "성장하고 있음"을 한눈에. (이미 상전이 화면엔 QF 곡선 있음 — 메인으로 확장.)
- 근거: Exponential Idle, 교차패턴 §2-F.

**P1-3. 연구/도감을 "노드 트리"로 시각화**
- 현 ux.md 연구는 가지 사이드+노드 리스트. AD Time Study / Cell Tech Tree처럼 **노드+연결선 트리**로 격상하면 진행감·"다음 해금 미끼"가 강해짐. 잠금=흐림+연결선, 구매=층악센트 채움.
- 근거: AD Time Study, Cell to Singularity Tech Tree of Life, 교차패턴 §2-D.

**P1-4. 앰비언트 모션으로 "살아있는 방치 화면"**
- Idle Slayer(움직이는 아바타)·Cell(회전 디오라마)처럼 방치 중에도 포컬이 움직여야 매력적. 우리 art-direction §5-C(글로우 4s 맥동, 층별 파티클)가 정확히 이것 — **구현 우선순위를 올릴 가치 확인**. r게이지가 우리의 "살아있는 아바타".
- 근거: Idle Slayer, Cell to Singularity, art-direction §5-C.

### P2 — 미래 (콘텐츠 확장 시)

**P2-1. 콘텐츠 폭증 대비 — 사이드바 내비 또는 필터 서브탭**
- 11층×메커니즘×87입자 도감×52연구 노드가 다 열리면 상단 탭 4개로 부족. **Melvor식 좌측 사이드바 내비** 또는 **Kittens식 필터 서브탭(전체/구매가능/활성)**을 예약.
- 근거: Melvor(20+스킬 사이드바), Kittens(필터 서브탭), NGU 안티패턴(탭 폭증 회피).

**P2-2. 위젯 축소 보존 확장**
- 이전 층 메커니즘 위젯을 우측 하단에 "축소 행"으로 보존(ux.md에 이미 설계). 우측 패널이 길어지면 아코디언화.

## 6. 추천 1순위 — 당장 적용할 단일 레퍼런스

### Antimatter Dimensions의 "생성기 행" + Exponential Idle의 "다크 계기판 톤"

이 둘의 조합이 우리 "배치가 별로"를 가장 직접 고친다:

1. **AD 8행 패턴**(`ad-full.png`)을 우리 8단 압축 체인에 1:1 이식 — 비용 버튼 내장, 버튼 색=상태, 행 배경 틴트, 행간 압축. 우리 체인이 메인 루프인데 현재 가장 약하게 보이는 부분(D2/D4/D6)을 한 번에 해결. AD는 우리 구조의 직계 조상이라 차용 마찰이 가장 적다(open-source-games.md에서 이미 코드 골격으로 채택 권장).

2. **Exponential Idle 톤**(`expidle.png`)으로 전체 시각 언어 확정 — 다크 모노크롬 + 수식/수치 타이포 포컬 + 절제 여백. 우리 art-direction과 이미 정합하며, "과학 노트북" 룩이 실제로 매력적임을 라이브로 검증했다. r게이지를 이 게임의 수식 패널처럼 **내부 리드아웃을 품은 큰 계기 패널**로 키우면 포컬이 산다(P0-3).

3. **테마 깊이는 Cell to Singularity에서** — 단, 그들의 비주얼(3D)이 아니라 **스케일을 공간 거리로 당겨 보여주는 IA**(빈 원+연결선)만 차용(P1-1). 그리고 그들이 6주 재설계로 고친 "사이드 패널 과대" 실수를 우리는 처음부터 피한다(P0-4).

한 줄 요약: **AD처럼 체인 행을 짜고, Exponential Idle처럼 칠하고, Cell to Singularity처럼 다음 스케일을 당겨 보여주되, 중앙 포컬에 공간을 몰아준다.**
