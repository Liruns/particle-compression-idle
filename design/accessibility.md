## 개요 및 접근 원칙

이 게임은 11개 층을 **색으로 구별**한다(art-direction §2 층 정체성 = 색). 장시간 응시하는 방치형이다. 파티클·스케일줌·VFX가 많다. 이 세 가지가 접근성의 핵심 리스크를 결정한다.

접근 원칙 두 가지:
1. **색에만 의존하는 정보 전달 금지.** 색은 항상 형태·패턴·텍스트 라벨과 함께 쓴다.
2. **접근성 옵션은 후처리가 아니다.** 층 색이 확정되는 수직슬라이스(정거장 1)부터 이중 인코딩을 설계에 내장한다. 나중에 끼우면 UI 구조를 다시 짠다.

### 1-B. 이중 인코딩 원칙 — 색 + 형태/패턴/라벨

색만으로 전달하던 정보를 반드시 두 번째 채널로 보강한다.

**층 구분:**
- 층 카드 악센트 컬러 + **층 고유 아이콘 기호**(분자=⬡, 원자=◎, 핵=⬤, 핵자=△▲, 쿼크=─, 프리온=≈, 끈=~, 루프=⬡⬡, 거품=○, 정보=∥, 플랑크=▪)
- 층 이름 텍스트 라벨은 항상 병기(색만으로 층을 판단할 수 없는 상황 없도록)
- 탭바 층 표시: 색 + 텍스트("원자층 — dec 1.42")로 이미 이중 인코딩됨. 유지.

**자원 구분(좌측 패널):**
- E(노란 ⚡), C(파란 ◎), D(보라 ▣), QF(녹 ◆) — 이미 색 + 기호 이중 인코딩. 유지.
- 뱃지 시스템(리셋/보존): `리셋` 텍스트 라벨을 뱃지 색(`--col-reset` 붉은색) 외에 항상 표시. 색만으로 판단 불가 상황 제거.

**핵자층 색전하(최우선 이중 인코딩):**
```
기본(색 사용):            색맹 보조 채널:
적 [████████░░] 82%       ← 수치(82%) + "R" 라벨
청 [██████████] 100%      ← 수치(100%) + "G" 라벨  (※ 물리 RGB, 게임내 기호 유지)
녹 [█████░░░░░] 50%       ← 수치(50%) + "B" 라벨

불균형 표시: 붉은 테두리 → 테두리 + ▼ 화살표 아이콘 + "불균형" 텍스트
퍼펙트: 녹 강조 → "합일 달성" 텍스트 레이블 항상 표시
```

**상전이 탭 ● 점등:**
- 현재 색 점등만 → **● 점 + "상전이 준비" 텍스트** 탭 레이블 변경 병행(ux.md §5-1 탭 구조에 반영)
- 첫 상전이 "미지 진입 ●": 색 펄스 + 레이블 텍스트 변경으로 이미 이중 인코딩됨. 유지.

**도감 카드 등급:**
- 단일/이중/발광/애니메이션 테두리(형태 이중 인코딩) — 이미 설계. 유지.
- 뱃지 색(회색/파랑/층색/금색): 뱃지 내부에 "알려진 입자" "가설 입자" "미지 입자" "전설" 텍스트 축약형(알/가/미/전) 추가.

### 1-D. 색 단독 의존 체크리스트

구현 전 검증 항목. 각 항목은 "색 없이도 정보가 전달되는가?"로 판단:

- [ ] 층 전환이 색 변화만으로 구분되는가? → 아이콘 + 라벨 병행 확인
- [ ] 핵자층 색전하 불균형이 색만으로 전달되는가? → 수치 + 텍스트 라벨 확인
- [ ] 상전이 점등이 색 점만인가? → 레이블 텍스트 변경 확인
- [ ] 도감 미발견 카드가 색(투명도 감소)만으로 구분되는가? → "미발견" 라벨 또는 아이콘 확인
- [ ] 리셋/보존 뱃지가 색만인가? → 텍스트 라벨 내장 확인
- [ ] QF 미니 차트가 색으로만 현재/예측 점을 구분하는가? → 현재점 ● 실선, 예측점 ○ 점선으로 형태 구분

## 2. 모션 감소 (prefers-reduced-motion)

### 2-A. 리스크 목록

이 게임의 모션 집약 요소(art-direction §3·§5):
- 층 전환 페이드+스케일 파열(§3-B)
- 상전이 화이트 플래시(§3-C)
- 빅 크런치 3초 블랙아웃 + 링 이펙트(§3-D)
- 앰비언트 파티클 상시 움직임(§4-A)
- r 게이지 느린 맥동(§5-C)
- 발견 시 reveal wipe(§4-B)
- 구매 버튼 스쿼시(§5-B)
- 공명 슬롯 깜빡임(ux.md §4-B, §5-7)

전정 민감자(모션 멀미)와 광과민성 발작 위험자 모두에게 해당한다.

### 2-B. prefers-reduced-motion 대응

CSS `@media (prefers-reduced-motion: reduce)` 자동 감지 + 설정 탭 수동 토글("모션 감소" 옵션).

**자동 감지가 설정 토글을 덮어쓰는 것이 원칙.** 시스템 수준 설정은 의료적 필요일 수 있다.

> **구현(2026-07-08, M2.4b):** `ui/stores/reduced-motion.ts`(OS 미디어쿼리 readable) + `ui/stores/prefs.ts`
> `effectiveReducedMotion = OS감소 OR 사용자'reduce'`. 캔버스 렌더러(`CanvasRenderer.setReducedMotion`)가
> 이 파생 스토어를 구독해 파티클·맥동·드리프트를 정지한다(CSS `@media`는 DOM만 커버 → 비트맵은 이 경로).
> 설정 패널(`SettingsView`) 모션 옵션은 **자동/감소 2단**뿐 — 위 원칙상 사용자가 OS 감소를 전체 모션으로
> *되돌릴* 수단은 두지 않았다(감소를 *더* 켜는 것만 허용). CSS `@media` 분기(DOM 애니메이션)는 tokens.css가 담당.

| 요소 | 기본 동작 | 감소 모드 동작 |
|---|---|---|
| 층 전환 페이드 | 0.3~0.5s 페이드+스케일 파열 | 즉시 전환 (페이드 없음, 스케일 파열 없음) |
| 상전이 화이트 플래시 | 0.05s 플래시 투명도 0.15 | 플래시 제거. 새 층 악센트 컬러 즉시 전환 |
| 빅 크런치 플래시 | 화이트 플래시 0.3 + 링 방사 | 플래시 제거. 링은 1회 정적 표시 후 페이드 |
| 빅 크런치 블랙아웃 | 3초 암전 | 유지(점멸 없음, 천천히 어두워지는 것은 안전) |
| 앰비언트 파티클 | 상시 이동 | 매우 느린 드리프트(속도 10% 이하) 또는 정지 |
| r 게이지 맥동 | 4초 주기 글로우 펄스 | 정적 글로우 (맥동 없음) |
| reveal wipe | 0.8s 위→아래 걷힘 | 즉시 reveal (페이드 전환) |
| 구매 버튼 스쿼시 | 0.95 scale (80ms) | 투명도 잠깐 감소로 대체 (scale 없음) |
| 공명 슬롯 깜빡임 | CSS animation 반복 | 정적 테두리 강조 (깜빡임 없음) |
| dec2~3 험프 깜빡임 가속 | 깜빡임 속도 20% 증가 | 깜빡임 없음 → 테두리 굵기 증가로 대체 |
| QF 카운터 fade-cross | 0.6s | 즉시 전환 |
| 전설 카드 맥동 | 2s 주기 금 맥동 | 정적 금 테두리 + 별도 ★ 아이콘 |

**광과민성 경고 기준(W3C WCAG 2.1 §2.3):** 초당 3회 이상 플래시, 특히 붉은 계열 플래시. 이 게임의 플래시는 백색 기반이며 0.05~0.3s 단발. WCAG 기준을 넘는 플래시 없음. 단, 감소 모드에서는 모든 플래시를 제거한다.

**도입 정거장:** 프로토타입(정거장 1) 구현 시 CSS 변수와 `prefers-reduced-motion` 분기를 처음부터 반영. 빅 크런치 연출 구현 전(정거장 2~3)에 감소 모드 시퀀스도 함께 설계.

## 3. 텍스트 접근성

### 3-A. 크기 조절

설정 탭에 **텍스트 크기** 슬라이더(80% / 100% / 120% / 150%). CSS `font-size` root 배율 변경.

핵심 제약: 수치 표기법(과학 표기, 모노스페이스)은 크기를 올려도 줄바꿈되지 않아야 한다. 150% 크기에서도 `3.81×10⁻¹¹ m` 한 줄 유지 → 레이아웃 테스트 필수.

art-direction §1-C 폰트 크기 기준점:
- 수치(primary): 13~20px → 최소 13px 아래로 내려가지 않음
- 레이블: 11~14px → 최소 11px 기준, 120% 이상 모드에서 13px 이상 보장

### 3-B. 명도 대비

**WCAG AA 기준(본 게임의 목표):** 일반 텍스트 4.5:1, 큰 텍스트(18px 이상 또는 굵은 14px 이상) 3:1.

배경(#0a0c0f 기준) 대비 현황:
- 텍스트 강조 `#e8eef3`: 대비비 약 13.5:1 — AA/AAA 통과
- 텍스트 보조 `#7a8f9e`: 대비비 약 4.9:1 — AA 통과
- 텍스트 희미 `#3d5162`: 대비비 약 2.1:1 — **미해금 텍스트 의도적으로 낮음** → 접근성 예외로 명시(비활성 요소)

- `#7c4dff` L6: 대비비 약 5.6:1 — AA 통과
- `#ff4081` L7: 대비비 약 5.2:1 — AA 통과
- `#3d5afe` L8: 대비비 약 5.0:1 — AA 통과
- `#00e5ff` L10: 대비비 약 10.9:1 — AA 통과

*대비비는 설계 추정값. 구현 시 실제 렌더 색상으로 재측정 필수(색맹 모드 대체 팔레트 포함).*

### 3-C. 가독성 — 장시간 응시

방치형 = 수 시간~수 주 동안 같은 화면을 반복해서 본다. 피로 관리:
- 수치 업데이트 주기: 1~2초(§5-C). 너무 빠른 갱신은 깜빡임으로 인지됨. 이 주기 유지.
- 배경 딥 다크 베이스: 어두운 화면은 야간 사용에 눈 피로 감소. 단, 밝은 환경(낮, 외부)에서 대비가 충분한지 확인 필요.
- 폰트: JetBrains Mono / IBM Plex Mono — x-height가 높아 소형 크기에서도 숫자 식별 우수. 유지.

## 4. 입력 접근성

### 4-A. 클릭 타깃 최소 크기

WCAG 2.5.5 (AAA): 44×44px. WCAG 2.5.8 (AA, WCAG 2.2): 24×24px.

본 게임 목표: **AA(24px) 최소 보장, 핵심 버튼 44px 권장.**

핵심 버튼별 크기 지침:
- [압축] 버튼: 최소 44×44px (핵심 반복 액션)
- 구매 버튼(T1~T8): 최소 32px 높이 × 전체 행 너비(200px 이상)
- 오비탈 공명 슬롯 ○ ○ ○: 각 36px 원형 — 44px 권장(작은 원형 클릭 어려움)
- 위상 토글 버튼([응집][분산][공명]): 최소 32×32px
- 스핀 네트워크 노드(L8): 미니캔버스 내 — 최소 16px, 클릭 인식 반경 확대(hit-area ≥24px)
- 탭바 탭: 최소 44px 높이

### 4-B. 키보드 접근

이 게임은 마우스 클릭 기반이다. 완전한 키보드 전용 플레이는 설계 범위 밖이다. 단, 기본 탐색은 키보드로 가능해야 한다.

**구현 대상:**
- Tab / Shift+Tab: 탭바 포커스 이동
- Enter / Space: 포커스된 버튼 활성화([압축], 구매, 공명 슬롯, 상전이 확인)
- 숫자키 1~5: 탭 직접 이동(압축=1, 연구=2, 도감=3, 상전이=4, 설정=5) — 방치형 핵심 액션 단축키
- Escape: 모달(오프라인 복귀, 빅 크런치 특수 화면) 닫기

**포커스 표시:** 기본 브라우저 outline 제거하지 않는다. 층별 악센트 컬러로 커스텀 포커스 링 표시(2px solid, offset 2px).

**구현 제외:** 스핀 네트워크 캔버스(L8) 내 그래프 인터랙션 — 마우스 전용 명시. 설정 탭에 "루프층 스핀 네트워크는 마우스 필요" 안내 표시.

### 4-C. 마우스 대안

마우스 없는 환경(터치 스크린 — Steam Deck 포함):
- Steam Deck 우선 지원 고려: 게임패드 포인터 모드로 클릭 동작. 타깃 크기(§4-A) 44px 충족 시 Steam Deck 호환성 확보.
- 터치 이벤트 기본 지원(웹 기반이므로 자동 처리 가능).
- 핀치/줌 비활성화: r 게이지 줌 연출과 충돌. 뷰포트 `user-scalable=no` 설정.

## 5. 스크린리더 지원 (가능 범위 한정)

### 5-A. 인크리멘탈 장르의 구조적 한계

인크리멘탈 게임은 매 초 수십~수백 개 수치가 갱신되는 구조다. 모든 변화를 스크린리더가 읽으면 사용 불가 수준의 음성 스트림이 발생한다. 완전한 스크린리더 지원은 이 장르에서 현실적으로 불가능하다.

본 게임의 목표: **핵심 진행 상태를 요약 형태로 접근 가능하게.** 실시간 매 틱 읽기가 아닌, 플레이어가 명시적으로 확인할 때 읽히는 구조.

### 5-B. 구현 대상

**정적 구조 (최우선):**
- 탭바: `<nav role="tablist">`, 각 탭 `<button role="tab" aria-selected>` — 탭 전환이 스크린리더에서 인식됨
- 층 카드: `aria-label="현재 층: 원자층, decade 1.42"` 정적 레이블
- 도감 카드: 미발견 카드 `aria-label="미발견 입자, 원자층"`. 발견 시 `aria-label="전자, 발견됨, 영구 보너스 +3%"`
- 리셋/보존 매트릭스: 테이블 구조 `<table>` + `aria-label`로 읽힘

**이벤트 기반 알림 (선택 공개):**
- 입자 발견 토스트: `role="status" aria-live="polite"` — 토스트 텍스트("[입자명] — 도감에 기록됨")가 자동 읽힘
- 상전이 탭 점등: `aria-label` 변경("상전이 가능") — 탭에 포커스 시 읽힘
- 오프라인 복귀 모달: `role="dialog" aria-modal="true"` + 자원 획득 수치 텍스트 접근 가능

**의도적 제외:**
- 자원 수치(E/C/D/QF) 실시간 갱신 — `aria-live` 연결하지 않음(소음 방지)
- 파티클 시스템 — 시각 전용, aria 불필요
- r 게이지 매 틱 갱신 — 포커스 시 현재 r 값 1회 읽기만 가능

**수치 읽기 접근점:**
- 자원 패널 E/C/D/QF 수치 영역: `aria-label="압축 에너지: 4.71e8, 생산율 2.34e6/초"` 포커스 시 1회 읽기
- r 게이지: `aria-label="현재 반경: 3.81×10⁻¹¹ m, decade: 1.42"` 포커스 시 1회 읽기
- 설정 탭에 "접근성 읽기" 버튼 — 클릭 시 현재 진행 요약을 `aria-live="assertive"`로 1회 읽어줌

### 5-C. 도입 범위 정리

| 기능 | 도입 여부 | 근거 |
|---|---|---|
| 탭바 ARIA | O | 기본 브라우저 접근성 — 비용 없음 |
| 정적 라벨(층 카드, 도감 카드) | O | 정적 요소, 구현 단순 |
| 발견 토스트 aria-live | O | 이벤트 기반, 소음 없음 |
| 자원 수치 포커스 읽기 | O | 탭 키로 한 번에 확인 가능 |
| 자원 수치 실시간 aria-live | X | 음성 스트림 과부하로 오히려 해로움 |
| 파티클/VFX ARIA | X | 장식 요소 |
| 스핀 네트워크 그래프 ARIA | X | 시각 캔버스 — 대안 없음, 설정에 명시 |
