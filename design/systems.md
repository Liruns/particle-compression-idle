## 1. 층 구조 전체 지도

### 1-1. 핵심 구분 원칙

**"층 진입 ≠ 상전이"**: 알려진 물리 5층(+쿼크 꼬리)은 상전이 없는 온보딩 구간이다. 층 진입 = 스케일 진입 + 도감 항목 + 내러티브 비트. 미지 서브층 6개만 상전이와 1:1 대응한다. **총 상전이 7회(빅 크런치 포함).**

**"서사 스케일(측정) ≠ 압축 장벽(상전이)"**: 미지 서브층의 서사 스케일(1e-19, 1e-21 등)은 codex·narrative 참조용이다. 실제 게임플레이 경계(상전이·체류 벽)는 economy가 검증한 decade 벽(dec19~26)을 따른다. 플레이어는 두 정보를 서로 다른 레이어로 읽는다.

### 1-2. 층 지도

| 층 이름 | 서사 스케일(m) | 게임플레이 decade 벽 | 상태 | 새 메커니즘 | 상전이 |
|---|---|---|---|---|---|
| 분자층 | 1e-9 | dec0 (온보딩 시작) | 알려진 물리 | 수동 압축 클릭 (§3) | 없음 |
| 원자층 | 1e-10 | dec1 | 알려진 물리 | 전자 오비탈 공명 (§2-A) | 없음 |
| 핵층 | 1e-14 | dec5 | 알려진 물리 | 핵력 게이지 (§2-B) | 없음 |
| 핵자층 | 1e-15 | dec6 | 알려진 물리 | 쿼크 색전하 삼원합일 (§2-C) | 없음 |
| 쿼크층 + 꼬리 | 1e-18 ~ 1e-19 | dec9 ~ dec19 | 알려진 물리(꼬리 포함) | 점근 자유 가속 (§2-D) | 없음 — 미지 진입 램프 |
| [미지] 프리온층 | 1e-19 ~ 1e-21 | **dec19** (첫 벽 4.3h) | 미지 서브층 1 | 위상 겹침 (§2-E) | **상전이 1** (쿼크→프리온) |
| [미지] 끈층 | 1e-21 ~ 1e-24 | **~dec20.5** | 미지 서브층 2 | 진동 하모닉스 (§2-F) | 상전이 2 |
| [미지] 루프층 | 1e-24 ~ 1e-27 | **~dec22** | 미지 서브층 3 | 스핀 네트워크 (§2-G) | 상전이 3 |
| [미지] 거품층 | 1e-27 ~ 1e-30 | **~dec23.5** | 미지 서브층 4 | 양자 거품 불확정 (§2-H) | 상전이 4 |
| [미지] 정보층 | 1e-30 ~ 1e-33 | **~dec25** | 미지 서브층 5 | 홀로그래픽 인코딩 (§2-I) | 상전이 5 |
| [미지] 플랑크층 | 1e-33 ~ 1.6e-35 | **dec26** (캡, 플랑크 한계) | 미지 서브층 6 | 시공간 픽셀 붕괴 (§2-J) | 상전이 6 + **빅 크런치** |

> 알려진 물리 5층 + 쿼크 꼬리(dec9~19) + 미지 서브층 6개 = 총 11층 구조 유지.
> 미지 서브층 게임플레이 벽은 dec19~26(7 decade) 대역에 6개 분포. 벽 간격 정확 수치는 economy-designer 재시뮬로 확정 [ECONOMY]. 각 서브층 단일런 체류 목표 ≥ 4h.
> decade 캡 = 26(플랑크). dec27 이상 벽 없음. 추가 상전이는 빅 크런치 재하강으로 공급.

### 1-3. 상전이 7회 전체 맵

| 상전이 | 경계 | 게임플레이 벽 | 비고 |
|---|---|---|---|
| 상전이 1 | 쿼크층 → 프리온층 | dec19 (4.3h, 첫 상전이 앵커) | **첫 미지 진입 — 서사 정점** |
| 상전이 2 | 프리온층 → 끈층 | ~dec20.5 [ECONOMY] | |
| 상전이 3 | 끈층 → 루프층 | ~dec22 [ECONOMY] | |
| 상전이 4 | 루프층 → 거품층 | ~dec23.5 [ECONOMY] | |
| 상전이 5 | 거품층 → 정보층 | ~dec25 [ECONOMY] | |
| 상전이 6 | 정보층 → 플랑크층 | dec26 (캡) [ECONOMY] | |

> 빅 크런치 이후 새 런은 QF 부스트로 미지 서브층을 빠르게 재통과 (§2-J, §5-5 참조). 장기 리텐션은 재하강 루프가 담당.

## 2. 층별 메커니즘 상세

### 2-A. 전자 오비탈 공명 (원자층 / decade 1)

**배경 물리:** 전자는 특정 에너지 준위(오비탈)에만 존재할 수 있다. 광자를 흡수·방출하며 껍질 간 전이.

**이름:** 오비탈 공명 (Orbital Resonance)

**입력·출력:**
- 입력: 압축 에너지 `E` 소비
- 출력: **공명 배율** — 체인 전체 생산에 곱해지는 단기 부스트 (수십 초 지속)
- 부산물: `D` (발견 데이터) 소량 — 공명 성공 시 "전자 전이 데이터" 로그

**동작 (의사코드):**
```
// 주기적으로 오비탈 슬롯이 열림 (능동 또는 자동 타이머)
orbital_slots = [s1, s2, s3]  // 현재 열린 오비탈 슬롯
resonance_window = 3초        // 슬롯 유효 시간

if player_clicks_slot(s) within resonance_window:
    resonance_bonus *= 1.5    // 클릭 시 30% 추가
    D += small_amount
else:
    resonance_bonus = base_resonance  // 자동 체인이 기본 공명 처리
    // 놓쳐도 진행은 계속됨

production_this_tick *= resonance_bonus
resonance_bonus decays to 1.0 over 30 seconds
```

**기존 시스템 상호작용:**
- 8단 체인에 곱해지는 배율이므로 체인이 강할수록 공명 효과가 커진다 → "작아짐=강해짐" 강화
- `D` 공급원으로 연구 트리 진입을 유도
- 방치 시 기본 공명이 자동 적용 (60/40: 클릭 시 1.5× 추가)

**왜 재미있는가:** 첫 "눌러야 할 이유"가 생기는 층. 슬롯이 깜빡이고 클릭하면 피드백이 터진다. 온보딩에서 능동 개입의 맛을 가르친다. 방치해도 진행되지만 "아까워서" 자꾸 들여다보게 만드는 유인.

### 2-B. 핵력 게이지 (핵층 / decade 5)

**배경 물리:** 강한 핵력(강력)은 극히 짧은 거리에서만 작용. 양성자와 중성자를 묶는 잔류 강력.

**이름:** 핵력 게이지 (Strong Gauge)

**입력·출력:**
- 입력: 체인 생산량 누적 → 게이지 충전
- 출력: **게이지 만충 시 핵결합 이벤트** — 순간적으로 C 생산량 [ECONOMY]× 폭증, 이후 쿨다운
- 부산물: 이벤트 발생 시 `QF` 미세 적립 (상전이 전이지만 복리로 쌓임)

**동작 (의사코드):**
```
// 게이지는 체인 생산에 비례해 자동 충전
gauge += production_rate * dt * gauge_fill_factor  // [ECONOMY]

if gauge >= gauge_max:
    trigger nuclear_binding_event()
    gauge = 0
    cooldown_timer = [ECONOMY] seconds

nuclear_binding_event():
    C_production *= binding_spike_mult   // [ECONOMY]× 지속 [ECONOMY]초
    QF += micro_QF_amount               // [ECONOMY]
    log_to_codex("핵결합 에너지 방출")   // 도감 항목 증가

// 연구로 게이지 충전 속도, 스파이크 배율, 쿨다운 단축 가능
```

**기존 시스템 상호작용:**
- 도감: 핵결합 이벤트마다 "핵종" 항목 추가 → 수집 동기
- 연구 트리: "게이지 과충전", "연속 결합" 같은 노드가 이 메커니즘을 강화
- 상전이: QF 미리 쌓기 시작 → 다음 상전이 준비를 자연스럽게 유도

**왜 재미있는가:** 게이지 바라는 것 자체가 긴장감. 충전 직전 "끝까지 기다릴까 업글 먼저 살까" 딜레마가 생긴다. 스파이크 이벤트는 숫자가 폭발하는 짧은 도파민 주사.

### 2-C. 쿼크 색전하 삼원합일 (핵자층 / decade 6)

**배경 물리:** 쿼크는 적·청·녹 색전하를 가지며, 세 개가 합쳐질 때만 색 중성인 하드론 형성.

**이름:** 색전하 삼원합일 (Color Charge Trinity)

**입력·출력:**
- 입력: 3종 색전하 `Cr`, `Cg`, `Cb` (각각 압축 깊이 C의 1/3씩 자동 분배)
- 출력: 세 색전하가 균형을 이룰 때 **하드론 합성 보너스** 활성화 (체인 전체 배율)
- 세 색전하가 불균형하면 배율 저하 패널티

**동작 (의사코드):**
```
// C 생산이 자동으로 3색으로 분배됨 (기본 균등, 연구로 조정 가능)
Cr, Cg, Cb = distribute(C_production, ratios)

// 균형 측정: 가장 작은 값 / 가장 큰 값
balance_ratio = min(Cr, Cg, Cb) / max(Cr, Cg, Cb)  // 0~1

if balance_ratio >= threshold_perfect:    // [ECONOMY]
    hadron_bonus = max_hadron_mult         // [ECONOMY]×
    synthesize_hadron()                    // 도감 항목 추가
elif balance_ratio >= threshold_ok:
    hadron_bonus = partial_mult            // [ECONOMY]×
else:
    hadron_bonus = penalty_mult            // [ECONOMY]× (페널티)

production *= hadron_bonus

// 연구로: 분배 비율 조정, 균형 임계값 완화, 추가 색전하 타입 해금
```

**기존 시스템 상호작용:**
- 압축 깊이 `C`에 색 속성이 붙음 — 단순 수치에서 3차원 벡터로 진화 (새로움)
- 연구 트리에서 색 분배 비율을 조작하는 것이 주요 업그레이드 카테고리가 됨
- 도감: 합성된 하드론 종류를 수집 (π 메손, 중성자, 양성자 등)

**왜 재미있는가:** 처음으로 "자원이 하나가 아니다"라는 감각. 세 막대를 맞추는 퍼즐감. 연구로 균형 임계값이 올라가면서 점점 정교하게 튜닝하는 재미로 발전.

### 2-D. 점근 자유 가속 (쿼크층 / decade 9)

**배경 물리:** 점근 자유(Asymptotic Freedom) — 쿼크가 서로 가까울수록 강한 핵력이 약해져 자유롭게 움직인다.

**이름:** 점근 자유 가속 (Asymptotic Freedom Burst)

**입력·출력:**
- 입력: 압축 깊이 C가 특정 임계값에 도달할 때마다 (층 내 마일스톤)
- 출력: **역설적 가속** — C가 클수록 압축 저항이 줄어드는 단기 구간 (GDD §7의 "로그 저항" 역전)
- 유지: [ECONOMY]초 동안 로그 계수 α 감소 → r 더 빠르게 감소

**동작 (의사코드):**
```
// 정상 상태: r = r₀ · 10^(-α·log₁₀(C)), α = 0.65 (저항 있음)
// AF 발동 시
if C > AF_threshold_k:                 // 마일스톤마다 임계값 증가
    alpha_effective = alpha * AF_factor // [ECONOMY] < 1.0, 저항 감소
    AF_duration_remaining = AF_duration // [ECONOMY]초

// 지속 중
if AF_duration_remaining > 0:
    r 감소가 일시적으로 빨라짐
    AF_duration_remaining -= dt
    // 만료 시 α가 원래 값으로 복원
    // 단, 연구 "점근 고정"으로 AF 지속 시간 영구 증가 가능

// 도감: AF 발동 횟수 기록 → "자유 쿼크 관측" 항목 해금
```

**기존 시스템 상호작용:**
- GDD 핵심 수식 `r = r₀ · 10^(-α·log₁₀(C))`의 α를 직접 건드리는 유일한 메커니즘
- 방치 중에도 자동 발동하지만, 발동 타이밍에 맞춰 체인 업그레이드를 사두면 더 큰 이득 (개입 보상)
- 미지 영역 진입 직전의 "도약대" 역할 — 플레이어에게 "이제부터 다른 세계"를 체감시킴

**왜 재미있는가:** "작아질수록 저항이 커진다"는 GDD의 물리 원칙을 잠깐 뒤집는 예외. 물리적으로 실제 점근 자유와 메타포가 완벽하게 맞아떨어진다. 역설적 가속의 맛.

---

### 2-E. 위상 겹침 (프리온층 / 게임플레이 벽 dec19, 서사 스케일 1e-19~1e-21)

**배경 물리:** 프리온(Preon) — 쿼크보다 작은 가상의 하위 구성 요소. 아직 실험적으로 미발견.

**새 입자:** 프리온 (Preon) — `P+`, `P-`, `P0` 세 타입. 색전하 대신 "위상" 속성을 가짐.

**이름:** 위상 겹침 (Phase Superposition)

**입력·출력:**
- 입력: 프리온 층에 진입하면 C가 "위상 C"로 재정의됨. 위상 C는 세 상태(응집/분산/공명) 중 하나를 오감
- 출력: 현재 상태에 따라 다른 자원 생산 배율. **상태 전환 타이밍이 핵심 플레이.**
  - 응집(Coherent): 체인 배율 최대, 하지만 D 생산 0
  - 분산(Dispersed): 체인 배율 낮음, D 생산 최대
  - 공명(Resonant): 중간 배율, QF 미적립 지속

**동작 (의사코드):**
```
phase_state ∈ {COHERENT, DISPERSED, RESONANT}

// 상태는 자동으로 순환 (방치 시 랜덤 분포로 수렴)
auto_cycle_timer -= dt
if auto_cycle_timer <= 0:
    phase_state = next_auto_state()    // 가중 랜덤
    auto_cycle_timer = [ECONOMY]

// 능동 개입: 플레이어가 상태를 수동으로 고정 가능
if player_pins_state(s):
    phase_locked = True
    phase_state = s
    pin_cost_E = [ECONOMY]             // E 소모로 고정 유지
    // 연구로 고정 비용 감소, 고정 지속 시간 증가

// 상태별 출력
production_mult = state_mult[phase_state]
D_rate = state_D[phase_state]
QF_trickle = state_QF[phase_state]
```

**기존 시스템 상호작용:**
- 연구 트리에 "위상 전문화" 가지가 생김: 응집 전문(체인 몬스터), 분산 전문(연구 러셔), 공명 전문(QF 축적가)
- 도감: 각 위상 상태에서 발견되는 프리온 타입이 다름 → 도감 수집 요소
- 방치 시 랜덤 분포(평균 효율)로 진행되지만, 상태 고정으로 전략적 특화 가능

**왜 재미있는가:** 처음으로 "어떤 자원을 우선할지" 선택이 의미 있어짐. 양자역학의 중첩 상태를 메커니즘으로 표현. 방치 시 균형 잡힌 진행, 개입 시 전문화 보너스.

### 2-F. 진동 하모닉스 (끈층 / 게임플레이 벽 ~dec20.5, 서사 스케일 1e-21~1e-24)

**배경 물리:** 끈이론 — 모든 입자는 1차원 끈의 진동 모드로 설명. 진동수가 입자의 성질을 결정.

**새 입자:** 개방끈 (Open String)·폐쇄끈 (Closed String). 각기 다른 진동 모드.

**동작 (의사코드):**

```
    V_remainder = V % harmonic_interval

// 연구: 하모닉 간격 단축, 특정 티어 고정 공명, V 저장 용량 확장
// 도감: 공명 패턴 기록 → 끈 진동 모드 도감 항목
```

**기존 시스템 상호작용:**
- 8단 체인과 직접 연결: 어느 티어가 공명할지 예측 가능 → 미리 해당 티어에 투자
- 연구 트리 "하모닉 조율" 가지: 공명 스케줄을 플레이어가 조작 가능
- 방치 시 8단 체인이 순차적으로 모두 혜택을 받음 (자동 밸런싱)

**왜 재미있는가:** "지금 어느 티어를 사면 가장 이득인가" 계산의 재미. 하모닉 공명 타이밍에 맞춰 대량 구매를 준비하는 전술 플레이. 끈이론의 핵심 개념과 메커니즘이 완전히 일치.

### 2-I. 홀로그래픽 인코딩 (정보층 / 게임플레이 벽 ~dec25, 서사 스케일 1e-30~1e-33)

**배경 물리:** 홀로그래픽 원리(Holographic Principle) — 3차원 공간의 정보는 2차원 경계면에 완전히 인코딩된다. 블랙홀 엔트로피 = 사건 지평선 넓이.

**새 입자:** 홀론 (Holon) — 정보 단위 자체가 입자화.

**이름:** 홀로그래픽 인코딩 (Holographic Encoding)

**입력·출력:**
- 입력: 누적 발견 데이터 `D_total` (상전이를 거쳐도 일부 보존)
- 출력: `D_total`에 비례하는 **경계 면적 배율** — 이 층에서 체인 효율이 총 발견량에 연동됨
- 의미: 도감을 많이 채운 플레이어가 더 빠르게 진행 → 컬렉션 동기 폭발

**동작 (의사코드):**
```
// holographic_mult 구성 — 두 항의 합, 상한 ≤ ×1.35 (economy §7.1·director §7.2 확정)
// ① D 항: 총 발견 데이터에 비례하는 기본 배율
//    holo_factor = 0.008, D_total이 클수록 증가하되 기여 상한 +5.8% (≤+0.058×)
D_total = D_current_run + D_lifetime      // D_lifetime 100% 보존 (§5-2)
D_holo_bonus = log₁₀(D_total + 1) * 0.008  // 기여 상한 +5.8%

// ② codex² 항: 도감 완성도 제곱 형태의 수집 보너스
//    완성도 1.0(만렙)에서 최대 +0.35× → holographic_mult 총 상한 ×1.35
//    B_HOLO(research-tree B_HOLO 노드)의 도감 비례 시너지 배율은 이 항에 흡수됨
//    B_HOLO를 별도 holographic 가산항으로 추가하면 ×1.454 → dec26 −31.2% FAIL
codex_completion = collected_particles / total_particles  // 0~1
codex_bonus = codex_completion^2 * 0.35  // 0 → +0.35× (완성도 제곱, 후반 집중 보상)

holographic_mult = 1.0 + D_holo_bonus + codex_bonus  // ≤ 1.0 + 0.058 + 0.35 = ×1.408
// 실제 상한: D항 + codex항 합산 ≤ +0.35× 유지 (D항이 작으므로 구조적 충족)
// economy §7.1 검증: holographic_mult ≤ ×1.35, dec26 단축 ≤ −25.9%

production_total *= holographic_mult
```

> **B_HOLO 흡수 규칙 (director-review §3.4 / economy §7.5):** B_HOLO 노드(research-tree.md)는 holographic_mult에 별도 가산항을 추가하지 않는다. B_HOLO의 효과는 codex²항 계수 강화(즉 위 codex_bonus 곱 증폭)로 표현하며, 전체 holographic_mult ≤ ×1.35 상한 내에서 관리된다. 별도 holographic 가산항 신설 금지.

**기존 시스템 상호작용:**
- 도감과 연구 트리가 처음으로 **생산 효율에 직접** 연결됨
- 상전이를 많이 할수록 `D_lifetime`이 쌓여 이 층에서 혜택이 폭발적으로 커짐
- "컬렉터" 플레이 스타일에 메커니즘 보상 — 도감을 꼼꼼히 채운 플레이어가 유리

**왜 재미있는가:** "과거에 발견한 것들이 지금 나를 돕는다"는 레트로 보상감. 도감이 단순 수집을 넘어 게임 후반의 핵심 자원이 됨. 홀로그래픽 원리라는 실제 물리 개념과 메커니즘이 일치.

## 3. 연구 트리 구조

### 3-1. 개요

`D` (발견 데이터)로 구매하는 영구 업그레이드 트리. 상전이 후에도 연구는 유지.
트리는 4개 가지(Branch)로 구성되며, 각 가지가 코어 루프의 다른 측면을 강화한다.

### 3-2. 4대 가지 (Branch)

```
[연구 트리 루트]
      │
      ├── A. 체인 증폭 (Chain Amplification)
      ├── B. 시너지 엔진 (Synergy Engine)
      ├── C. 자동화 확장 (Automation Expansion)
      └── D. 위상 심화 (Phase Deepening)
```

#### A. 체인 증폭 Branch
**목적:** 8단 체인의 기본 생산 배율을 직접 강화.

| 노드 카테고리 | 효과 | 게이팅 조건 |
|---|---|---|
| 티어 배율 노드 | 특정 티어 k의 생산 × [ECONOMY] | D [ECONOMY] + 해당 층 진입 |
| 마일스톤 확장 | 마일스톤 임계값(25/50/100/200)에 추가 구간 추가 | D [ECONOMY] |
| 체인 연속 강화 | 직전 티어 보유 수에 따라 다음 티어 배율 증가 | D [ECONOMY] + 체인 노드 3개 이후 |
| 로그 저항 감소 | α 계수를 영구적으로 소폭 감소 | D [ECONOMY] + 쿼크층 진입 |

#### B. 시너지 엔진 Branch
**목적:** 층별 메커니즘끼리의 상호작용 강화.

| 노드 카테고리 | 효과 | 게이팅 조건 |
|---|---|---|
| 공명-핵력 시너지 | 오비탈 공명 발동 시 핵력 게이지 추가 충전 | D [ECONOMY] + 핵층 진입 |
| 색전하-하모닉 연결 | 삼원합일 완성 시 V(진동 에너지) 추가 충전 | D [ECONOMY] + 끈층 진입 |
| 위상-요동 안정화 | 응집 상태에서 불확정 요동 하한 상승 | D [ECONOMY] + 거품층 진입 |
| 홀로그래픽 증폭 | 도감 항목 수에 따라 모든 시너지 효과 추가 배율 | D [ECONOMY] + 정보층 진입 |

#### C. 자동화 확장 Branch
**목적:** 방치 효율 향상 및 새 메커니즘의 자동화 해금.

| 노드 카테고리 | 효과 | 게이팅 조건 |
|---|---|---|
| 오비탈 자동 클릭 | 공명 슬롯을 자동으로 클릭 (인터벌 [ECONOMY]) | D [ECONOMY] + 원자층 + 수동 클릭 50회 이상 |
| 위상 자동 고정 | 최적 위상 상태를 자동 선택·고정 | D [ECONOMY] + 프리온층 진입 |
| 네트워크 자동 확장 | 스핀 네트워크에 노드 자동 추가 (느린 속도) | D [ECONOMY] + 루프층 진입 |
| 오프라인 효율 강화 | offline_modifier 상승 | D [ECONOMY] + 2회 이상 상전이 |

> **자동화 해금 원칙:** 수동으로 30~50회 능동 개입 후 해금. "노력해서 얻은" 느낌 보장. 너무 빠른 자동화는 개입 동기를 제거하므로 층 진입 후 [ECONOMY]분 이후에만 해금 가능.

#### D. 위상 심화 Branch
**목적:** 각 층의 새 메커니즘을 심화·변형. 층에 따라 잠금 해제됨.

| 노드 카테고리 | 효과 | 게이팅 조건 |
|---|---|---|
| 오비탈 슬롯 확장 | 동시 공명 슬롯 수 증가 | D [ECONOMY] + 원자층 |
| 핵 과충전 | 핵력 게이지 만충 전에도 부분 방출 가능 | D [ECONOMY] + 핵층 |
| 색 전문화 | 삼원합일 완성 시 추가 "색 전문 자원" 해금 | D [ECONOMY] + 핵자층 |
| 하모닉 고정 | 특정 티어 하모닉 공명 우선도 증가 | D [ECONOMY] + 끈층 |
| 스핀 구조 잠금 | 상전이 후 스핀 네트워크 부분 보존 | D [ECONOMY] + 루프층 |
| 요동 패턴 기억 | 이전 요동 이벤트 기록, 다음 요동 예측 힌트 | D [ECONOMY] + 거품층 |

### 3-3. D 게이팅 규칙

```
// D 획득 경로
D += particle_discovery_bonus     // 새 입자 발견 (일회성, 큰 값)
D += compression_depth_trickle    // 압축 깊이 비례 상시 생산 (작은 값)
D += resonance_event_bonus        // 메커니즘 이벤트 발동 시 보너스

// 연구 노드 비용 구조
node_cost = base_cost_D * tier_factor * depth_in_branch  // [ECONOMY]

// 가지 잠금 해제 조건
branch_A_unlock: 원자층 진입 (자동)
branch_B_unlock: 핵층 진입 + D [ECONOMY] 누적
branch_C_unlock: 상전이 1회 완료
branch_D_unlock: 해당 층 진입 시 자동 해금 (층마다 1개 노드씩 추가)
```

### 3-4. 층 진행과의 맞물림

- **층 진입 시:** 해당 층의 위상 심화 노드 1개 자동 해금 (발견의 느낌)
- **새 입자 발견 시:** `D` 큰 보너스 → 연구 속도 순간 가속
- **상전이 후:** 연구는 유지, 하지만 새 층의 노드는 다시 `D` 필요 → "리셋감" 없이 자연스러운 목표 이동
- **미지 서브층:** 각 서브층 진입마다 위상 심화 Branch에 2~3개 노드 추가 → 트리가 점점 깊어짐

## 5. 상전이 루프의 층 관통 구조

### 5-1. 상전이(Phase Transition) 정의

**상전이는 미지 서브층 6개 + 빅 크런치 = 총 7회만 발생한다.** 알려진 물리 5층(분자/원자/핵/핵자/쿼크, dec0~9)과 쿼크 꼬리(dec9~19)는 상전이 없는 온보딩 구간이다. 층 진입 = 스케일 진입 + 도감 + 내러티브 비트이며, 상전이 리셋·QF 획득은 발생하지 않는다.

상전이 = 미지 서브층 경계(게임플레이 벽, dec19~26)에 도달하면 현재 층을 리셋하고 QF를 획득하며 다음 서브층으로 진입. 첫 상전이(쿼크→프리온, dec19)가 미지 영역 진입의 서사적 정점이다.

상전이 7회 전체 구조는 §1-3 참조.

### 5-2. 리셋/보존 매트릭스

| 항목 | 층 내 상전이 | 미지 서브층 간 상전이 | 빅 크런치 (플랑크층 완료) |
|---|---|---|---|
| 압축 에너지 `E` | 리셋 | 리셋 | 리셋 |
| 압축 깊이 `C` | 리셋 | 리셋 | 리셋 |
| 8단 체인 압축기 개수 | 리셋 | 리셋 | 리셋 |
| 발견 데이터 `D` (현재 런) | 일부 보존 [ECONOMY]% | 일부 보존 [ECONOMY]% | **D_current 보존율 (회차 곡선)**: 2회차 65% / 3회차 50% / 4~5회차 40% / 6회차 38% / 7회차+ 35%. D_lifetime은 항상 100% 보존 (§5-5 참조). |
| 양자 거품 `QF` | 영구 보존 + 누적 | 영구 보존 + 누적 | 영구 보존 + 누적 |
| 입자 도감 | 완전 보존 | 완전 보존 | 완전 보존 |
| 연구 트리 구매 | 완전 보존 | 완전 보존 | 완전 보존 |
| 층별 메커니즘 상태 (위상·스핀 네트워크 등) | 리셋 | 리셋 | 리셋 (연구로 부분 보존 가능) |
| 홀로그래픽 `D_lifetime` 누적 | 보존 | 보존 | 보존 |

### 5-3. QF 획득 공식 구조

**QF는 economy의 `lifetime_C` 단일 모델이 지배한다.** `layer_QF_factor`(별도 층별 배율)는 삭제됨 — economy의 `QF = K·(lifetime_C/D_norm)^0.5` 수식에서 압축 깊이가 깊을수록 `lifetime_C`가 자연히 커지므로 "deeper = more QF"가 자동 달성된다. 이중 가중 없음.

```
// 상전이 시 QF 계산 (수식 기반, 수치는 economy에 위임)
// economy 모델: QF = K · (lifetime_C / D_norm)^0.5
// → 깊은 층일수록 lifetime_C 누적이 커져 QF 자동 증가
// → layer_QF_factor 별도 곱 불필요, 삭제

delta_QF = economy_QF_formula(lifetime_C, D_norm)   // [ECONOMY] 수식 전체 위임

// QF 활용
production_mult = 1 + QF_total * QF_base_rate    // [ECONOMY]
// 7회 상전이(빅 크런치 포함) 시 최종 mult 2~3× 밴드 유지 목표
// base_rate=0.25 기준, economy 재시뮬로 확인 [ECONOMY]
```

### 5-4. 층 진입과 상전이의 관계

```
// 층 진입 흐름
decade_current = α * log₁₀(C + 1)

if decade_current >= layer_threshold[next_layer]:
    // 알려진 물리 5층(dec0~19): 층 진입 = 스케일 진입 + 도감 + 내러티브 비트만.
    // 상전이 UI 없음. QF 획득 없음. 리셋 없음.
    if next_layer in KNOWN_PHYSICS_LAYERS:   // 분자/원자/핵/핵자/쿼크+꼬리
        auto_enter_layer(next_layer)
        trigger_codex_and_narrative_bit(next_layer)
    
    // 미지 서브층(dec19~26): 상전이 1:1 대응.
    else:
        phase_transition_available = True
        // UI: "상전이 가능 — 미지 영역 진입" + QF 획득 예정량 미리보기

// 플레이어 선택 (미지 서브층에서만):
// A. 지금 상전이 → QF 획득, 리셋, 다음 서브층 메커니즘 해금
// B. 계속 압축 → 더 큰 QF를 위해 현재 서브층을 더 파기

// 최적 리셋 타이밍 힌트 (UX)
// "다음 상전이에서 QF 2× 달성까지 남은 시간: [ECONOMY]분" 표시
// → 플레이어가 타이밍을 스스로 판단할 수 있게
```

### 5-5. 미지 서브층 간 상전이 특징

미지 서브층은 알려진 층과 달리 **서브층 내에서 중간 상전이 없음** — 각 서브층은 게임플레이 벽(economy decade 벽) 하나로 진입하고, 벽을 넘으면 상전이 1회가 발생한다. 서브층 간격은 균등 3 decade가 아니라 economy가 검증한 후반 가중 배치를 따른다(dec19~26 대역, 각 체류 ≥ 4h).

대신 서브층 내 "마일스톤 이벤트"가 중간 목표를 제공:

```
// 서브층 내 마일스톤 (예: 프리온층, 게임플레이 벽 dec19~다음 벽)
milestone_진입직후: 첫 프리온 발견 → D 보너스 + 도감 항목
milestone_중간:    위상 응집 100회 → "안정 프리온" 도감 항목 + 연구 노드 해금
milestone_벽도달:  상전이 가능 → 끈층 진입

// 마일스톤 보상이 빠를 경우
// → 마일스톤 완료 후 바로 상전이 or 마일스톤 수익을 더 파기의 선택
```

**빅 크런치 재하강 연결(§2-J):** 플랑크층 완료(dec26) 후 빅 크런치가 발동하면 QF 영구 누적 후 dec0 재시작. 이후 런은 QF 부스트로 미지 서브층을 빠르게 재통과하며(§2-J 의도), 장기 리텐션은 재하강 루프가 담당한다. dec27 이상 벽은 존재하지 않는다.

## 6. economy-designer / content-designer와의 조율 필요 항목

### 미해결 질문 (이번 설계 범위 밖)

1. **QF 수식 정확 수치:** economy `lifetime_C` 모델의 `K`, `D_norm`, `QF_base_rate` 값. 목표: 첫 상전이(dec19, 4~6h) 후 7회 상전이(빅 크런치 포함) 시 최종 mult 2~3× 밴드. (`layer_QF_factor`는 삭제됨 — §5-3 참조.)

2. **D 보존율:** 상전이 시 D가 몇 % 이월되는가. 높으면 연구 러싱 약화, 낮으면 매 런 연구 재투자 필요 — 밸런싱 레버.

3. **위상 겹침 상태별 배율:** 응집/분산/공명의 각 배율이 얼마나 차별화되어야 방치와 개입 간 40% 격차가 유지되는가.

4. **불확정 요동 범위:** 하향 요동이 얼마나 낮을 수 있는가 (0.5×? 0.3×?). 너무 낮으면 불쾌, 너무 높으면 긴장감 소실.

5. **스핀 네트워크 노드 비용:** D와 E를 어떤 비율로 소모. D 소모가 너무 크면 연구 트리와 충돌.

6. **하모닉 공명 인터벌:** V 임계값 간격이 너무 빠르면 지루 (어떤 티어든 자주 공명), 너무 느리면 무관심.

7. **층별 체류 시간 배분:** 미지 서브층 6개가 각각 몇 % 시간을 차지할지. GDD는 미지 영역 전체 = 95%. 각 서브층의 배분이 필요 (균등 vs 후반 집중).

8. **클릭 파워 곡선:** 수동 클릭 1회의 C 기여가 체인 대비 몇 %인가. 초반엔 높아야 클릭이 의미 있지만, 너무 높으면 클릭 게임이 됨.

9. **도감 완성도 보너스 강도:** 홀로그래픽 인코딩의 `codex_bonus_factor`. 도감 100% 완성 시 몇 %의 추가 배율인가.

10. **오프라인 시 메커니즘 처리:** 위상 겹침, 스핀 네트워크 등 능동 메커니즘이 오프라인 중 어떻게 동작하는가 (평균값? 최악값? 별도 공식?).
