## 1. 전체 화면 목록 & 상태 전이 개요

### 1-A. 화면 목록

| ID | 화면 이름 | 탭/모달 | 진입 조건 |
|---|---|---|---|
| SCR-01 | 압축 메인 | 탭 (기본) | 항상 |
| SCR-02 | 연구 | 탭 | 원자층(L2) 진입 + D 첫 획득 |
| SCR-03 | 도감 | 탭 | 첫 입자 발견 (게임 시작 즉시) |
| SCR-04 | 상전이 | 탭 (● 점등 시) | 미지 서브층 6벽(dec19/20.5/22/23.5/25/25.5) 또는 빅 크런치(dec26) 도달 |
| SCR-05 | 설정 | 탭 | 항상 |
| SCR-06 | 오프라인 복귀 모달 | 모달 | 로드 시 elapsed > 60초 |
| SCR-07 | 빅 크런치 시퀀스 | 전체화면 오버레이 | dec26 도달 (자동 진입) |
| SCR-08 | 재하강 RUN_START | 전체화면 | SCR-07 종료 후 |
| SCR-09 | 집중 서브층 선택 (FOCUS_SELECT) | 전체화면 오버레이 | run_index >= 2, RUN_START 확인 후 |

### 1-B. 최상위 상태 전이 다이어그램

```
[초기 로드]
    │
    ├─ elapsed > 60s ──→ [SCR-06 오프라인 모달] ──→ 확인 클릭 ──→ [SCR-01 메인]
    │
    └─ elapsed <= 60s ──→ [SCR-01 메인]
                                │
            ┌───────────────────┼───────────────────────────┐
            │                   │                           │
        탭 클릭             탭 클릭                     탭 클릭
            │                   │                           │
        [SCR-02]            [SCR-03]                   [SCR-05]
         연구                도감                        설정
            │
        탭 클릭 (● 점등 후)
            │
        [SCR-04 상전이]
            │
            ├─ 일반 상전이: "다음 층으로" 클릭 ──→ [SCR-01 메인 (새 층)]
            │
            └─ dec26 도달: 자동 ──→ [SCR-07 빅 크런치]
                                         │
                                   시퀀스 완료
                                         │
                                   [SCR-08 RUN_START]
                                         │
                               "새 압축 시퀀스 준비" 클릭
                                         │
                                ┌────────┴────────┐
                                │                  │
                           run_index=1         run_index>=2
                                │                  │
                          [SCR-01 메인]    [SCR-09 FOCUS_SELECT]
                                                   │
                                             하강 시작 클릭
                                                   │
                                           [SCR-01 메인]
```

### 1-C. 탭 점등 조건 완전표

| 이벤트 | 탭 변화 | 연출 |
|---|---|---|
| 알려진 물리 5층 진입 (dec0~19) | 비점등 | 우측 패널 층 카드 슬라이드인만 |
| dec15 도달 (쿼크층 꼬리) | 비점등 | 층 카드에 "알려진 입자 경계 근접" 1줄 추가 |
| 첫 상전이 (dec19, 프리온 벽) | 점등 — 특별 강조 | "미지 진입 ●" 레이블, 1.5s 주기 글로우 펄스, 틱 사운드 1회 |
| 상전이 2~6 (dec20.5~25.5) | 점등 — 일반 | 정적 ● 표시, 레이블 "상전이 ●" |
| 빅 크런치 (dec26) | 점등 후 자동 시퀀스 진입 | ● 0.3s 표시 후 SCR-07 자동 전환 |
| 재하강 후 알려진 물리 재진입 (2회차+) | 비점등 | 탭바 배지 "N회차 · 집중: {층}"만 표시 |
| 재하강 후 미지 서브층 벽 재도달 (2회차+) | 점등 — 일반 | 정적 ●. 첫 상전이 특별 연출은 1회차에만 |

### 2-E. 메커니즘 위젯 — RIGHT PANEL

층마다 다른 위젯이 우측 패널에 표시된다. ux.md §4에 따른 스펙 요약:

| 층 | 위젯 이름 | 핵심 요소 | 상호작용 |
|---|---|---|---|
| L1 분자 | 없음 (층 카드만) | — | — |
| L2 원자 | 오비탈 공명 슬롯 | ○ ○ ○ 원형 버튼 + 카운트다운 링 | 클릭 |
| L3 핵 | 핵력 게이지 | 수평 진행 바 + 충전율 | 수동 없음 (자동 충전) |
| L4 핵자 | 색전하 삼원합일 | 적/청/녹 3색 막대 + 균형 비율 | 수동 없음 |
| L5 쿼크 | 점근 자유 | 상태 + 조건 진행 바 | 수동 없음 |

### 7-C. dec2~3 험프 넛지 (CONCERN-3)

dec2~3 구간은 자동 생산이 클릭을 완전히 대체하기 전, 공명 타이밍이 진행속도를 결정하는 텐션 구간이다.

**넛지 1 — 공명 슬롯 깜빡임 가속:**
- 평소 깜빡임 속도 대비 20% 증가
- 카운트다운 링 3초 → 2.5초로 단축
- 감소 모드(prefers-reduced-motion): 깜빡임 없음 → 슬롯 테두리 굵기 증가로 대체

**넛지 2 — 상태 로그 힌트:**
```
[힌트] 공명 클릭 시 현재보다 ×1.4 빠름     ← 회색 중립 텍스트, 강요 아님
```
- 실시간 계산: 현재 C 속도 대비 공명 활성 시 속도 비율
- 비활성 조건: dec4 진입 또는 자동화 연구 노드 구매 시 사라짐

**넛지 3 — 체인 최적 티어 ▶ 표시:**
- 체인 테이블의 "현재 가장 유리한 구매 티어" 행에 ▶ 아이콘 표시
- 계산: (해당 티어 구매 후 T1 생산 증가율) / 비용의 기대값으로 실시간 최적 티어 판단

## 10. 오프라인 복귀 모달 (SCR-06)

### 10-A. 진입 조건

- 게임 로드 시 `elapsed > 60초` 자동 표시
- 모달 방식: 배경 어둡게 dim, 모달이 아래에서 슬라이드업 (300ms)

### 10-B. 레이아웃

```
┌────────────────────────────────────┐
│  오프라인 압축 진행                 │
│  8시간 12분 동안 압축이 진행되었습니다. │
│  유효 온라인 환산: 5h 20m (×0.65)  │
├────────────────────────────────────┤
│  오프라인 효율 ──────────── 65%   │
│                                    │
│  E   +4.48e12                     │
│  C   +2.81e16                     │
│  D   +2.38e5                      │
│                                    │
│  반경 변화: 3.81→3.04 ×10⁻¹¹m    │
│  decade: 1.42 → 1.58              │
│  입자 발견: +0 (공명 미활성)       │
├────────────────────────────────────┤
│  [안내] 오비탈 공명은 기본 배율만   │
│  오프라인에서 적용됩니다.          │
├────────────────────────────────────┤
│          [ 확인 ]                  │
└────────────────────────────────────┘
```

**자원 수치:** 카운트업 없음. 최종값 즉시 표시 + "(+{오프라인 증가량})" 한 줄.

### 10-C. 특수 케이스

| 케이스 | 표시 변화 |
|---|---|
| 상전이 조건이 오프라인 중 충족 | "상전이 조건이 충족되었습니다." + 확인 후 탭 ● 즉시 점등 |
| 24h 캡 초과 | "24시간 상한 도달 — 이후 진행은 적용되지 않았습니다." 명시 |
| 오프라인 효율 100% (상전이 직후 첫 오프라인) | "복귀 보너스 활성. 생산 효율 100%." 토스트 |
| 오프라인 시간 >= 12h | 모달 배경에 현재 층 악센트 컬러 미세 틴트 |
| elapsed <= 60s | 모달 없음, 바로 메인 화면 |

**Escape 키:** 모달 닫기 ([확인] 클릭과 동일 효과)

## 11. 점진 공개 규칙 & 알림/배지

### 11-A. 전체 점진 공개 규칙 (정보 과부하 관리)

새 시스템 해금 시 일시에 모두 보여주지 않는다. 다음 규칙을 따른다:

| 원칙 | 구현 |
|---|---|
| 자원 순차 공개 | E,C → D → QF. 각 자원은 관련 메커니즘과 함께 등장 |
| 탭 순차 해금 | 도감(시작) → 연구(D 획득) → 상전이(dec19 도달). 기본 탭(압축, 설정) 항상 존재 |
| 체인 티어 점진 노출 | T1 구매 시 T2 등장, T2 구매 시 T3 등장 (+1 노출 방식) |
| 위젯 축소 보존 | 이전 층 메커니즘 위젯은 축소 상태로 유지 (정보 손실 없음) |
| 연구 가지 잠금 | 각 가지의 해금 조건 충족 전 흐림 + 잠금 아이콘 |
| 도감 미지 섹션 | dec15 이전 숨김, dec15 흐릿 등장, dec19 이후 정상 표시 |

### 11-B. 알림/배지 시스템

| 이벤트 | 알림 형태 | 지속 시간 |
|---|---|---|
| 입자 발견 | 토스트 (우상단): "[입자명] — 도감에 기록됨" + 도감 탭 잠깐 밝아짐 | 토스트 2s |
| 연구 탭 해금 | 상태 로그: "발견 데이터 축적. 연구 가능." + 탭 레이블 활성화 | 영구 |
| 상전이 조건 충족 | 탭 ● 점등 + 우측 패널 "상전이 준비됨" 힌트 | ● 유지 (클릭 전까지) |
| 첫 상전이 (dec19) | "미지 진입 ●" 레이블 + 글로우 펄스 1.5s 주기 + 틱 사운드 | 클릭 전까지 |
| D 첫 획득 | D 행 패널 페이드인 + 연구 탭 활성화 | 영구 |
| QF 첫 획득 | QF 행 패널 페이드인 + ◆ 아이콘 펄스 | 영구 |
| 집중 서브층 진입 | 우측 패널 층 카드 강조 (3초) + 탭바 배지 색상 → #3ecf8e | 카드 3s, 배지 서브층 체류 중 |
| 전설 입자 발견 | reveal wipe + 황금 플래시 (0.08 opacity, 1s) + 전용 토스트 | 토스트 3s |
| 도감 완성도 milestone | 정보층 배율 증가 시 홀로그래픽 배율 바 flash | 0.5s |

### 11-C. 토스트 큐 관리

- 동시 최대 2개 (거품층 요동 토스트 캡)
- 표시 위치: 화면 우하단 (요동 토스트) / 우상단 (발견 토스트)
- 우선순위: 전설 입자 > 첫 상전이 > 일반 발견 > 요동 이벤트
- 큐 쌓임 시: 최신 대기, 기존 토스트 지속 후 순차 표시

## 12. 플랫폼 차이 (웹 ↔ 데스크톱 NW.js)

### 12-A. 창 크기 & 반응형

| 항목 | 웹 브라우저 | NW.js 데스크톱 |
|---|---|---|
| 최소 폭 | 640px (LEFT PANEL 접힘) | 900px 강제 (최소창 크기 설정) |
| 최소 높이 | 480px | 600px 강제 |
| 최적 해상도 | 1280×720 이상 | 1280×720 이상 (Steam 기본) |
| 전체화면 | F11 / 브라우저 전체화면 | NW.js win.enterFullscreen() |
| 확대/축소 | 브라우저 줌 (제한 없음) | 설정 탭 "텍스트 크기" 옵션만 |
| 핀치 줌 | user-scalable=no (비활성화) | 해당 없음 |

**640px 미만 (웹 전용) LEFT PANEL 접힘 상태:**
```
[접힘 상태 — 좌측 세로 바]
  ⚡        ← E 아이콘만
  ◎         ← C 아이콘만
  ▣         ← D 아이콘만
  ◆         ← QF 아이콘만
  ─ (구분선)
  [로그 아이콘]
```
hover 시 툴팁으로 수치 표시.

### 12-B. Steam 전용 기능 (NW.js)

| 기능 | 구현 방식 | 웹 대응 |
|---|---|---|
| Steam 오버레이 (Shift+Tab) | NW.js + steamworks.js 자동 지원 | 해당 없음 |
| Steam Cloud 세이브 | lz-string 압축 → Steam Cloud 경로 기록 | localStorage 폴백 |
| Steam 업적 | steamworks.js `activateAchievement()` | 해당 없음 |
| 창 크기 저장/복원 | NW.js win 설정 저장 | localStorage 폴백 |
| 시스템 트레이 | NW.js tray API (선택, 방치형 적합) | 해당 없음 |

### 12-C. 접근성 플랫폼별

| 기능 | 웹 | NW.js |
|---|---|---|
| prefers-reduced-motion | CSS 미디어 쿼리 자동 감지 | 동일 (Chromium 기반) |
| 키보드 Tab 이동 | 표준 브라우저 동작 | 동일 |
| 스크린리더 | NVDA/JAWS (Windows), VoiceOver (Mac) | 제한적 (NW.js ARIA 지원 확인 필요) |
| Steam Deck 터치 | 해당 없음 | 44px 타깃 + 터치 이벤트 |

### 12-D. 해상도 스케일 (accessibility.md §3-A 연계)

CSS root font-size 배율 기준점:
- 수치(primary): 최소 13px 보장 (150% 모드에서도)
- 레이블: 최소 11px, 120%+ 모드에서 13px 이상
- 150% 모드에서 `3.81×10⁻¹¹ m` 한 줄 유지 필수 → 구현 후 레이아웃 테스트
