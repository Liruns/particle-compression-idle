### N-1. 빅 크런치 시퀀스 카피

> 이 카피는 narrative.md §5-B "빅 크런치 상전이 메시지"의 **확장판이자 교체본**이다.
> §5-B의 4줄 시퀀스를 다음 구조로 대체한다.

**트리거 조건**: dec26 완료(플랑크 길이 도달) → 빅 크런치 화면 진입.

---

#### (a) 플랑크 도달 / 진입 비트

narrative.md 비트 11(플랑크층 내부)이 선행 표시된 상태.

> 1.616×10⁻³⁵ m.  
> 시공간의 격자가 끝난다.  
> 더 이하의 거리는 측정할 공간이 없다.

*[비트 11은 그대로 유지. 이 카피는 비트 11 직후, 빅 크런치 화면 전환 전에 추가로 삽입되는 전환 텍스트. 0.8초 표시 후 화면 전환.]*

---

#### (b) 최종 QF 폭발 순간

*한 줄씩 순차 등장. 각 줄 사이 0.4초.*

> 압축 한계 도달.  
> 양자 거품 +{N}.  
> 임계점.

*0.7초 정지.*

> 붕괴가 아니다.  
> 재압축이다.

*[설계 노트: "붕괴가 아니다. 재압축이다." — 이 두 줄이 핵심이다. 플랑크 도달을 종결이 아니라 다른 종류의 시작으로 명명한다. 느낌표 없음. 선언하되 소리 높이지 않는다.]*

---

#### (c) 재하강 안내 — "처음으로 돌아가지만 같지 않다"

*전환 후 새 화면 정착. 3초 암전 후 표시.*

> dec 0.  
> 분자층.  
> 압축기: 없음.  
> 양자 거품: {누적 QF}.  
> 이전 런의 관측이 남아있다.

*2초 정지.*

> 다시 내려간다.  
> 이번엔 어디까지 갈 수 있는지 알고 있다.

*[설계 노트: "이전 런의 관측이 남아있다" — QF와 도감이 유지된다는 사실을 시스템 언어로 표현. "어디까지 갈 수 있는지 알고 있다" — 2회차 이후의 재하강이 1회차 복사본이 아니라는 암시. 플레이어는 이미 플랑크를 봤다. 이번엔 다른 목표로 내려간다.]*

### N-2. 재하강 회차별 톤 변주 방향

> 이 섹션은 회차 구분 UI 문구·층 비트 보이스에 적용된다.
> 2회차+가 1회차의 복사본이 아니라 "다른 결의 회차"로 읽히게 하는 톤 가이드.

---

#### 1회차 보이스: 발견의 긴장

1회차 플레이어는 처음 내려가는 중이다. 아직 무엇이 있는지 모른다.

- 톤 키워드: **조심스러운 기록. 처음 보는 것의 무게.**
- 층 진입 비트(narrative.md §4)는 그대로 적용.
- 시그니처 로그(§6)는 발견의 순간으로 읽힌다.
- 연구소장의 목소리에 여백이 있다. 확신이 없다.

예시 톤 —
> "표준 모델에 없는 신호가 감지된다. 프리온 — 가설로만 존재하던 이름. 로그에 기록한다."

---

#### 2회차 보이스: 검증의 집중

2회차 플레이어는 이미 플랑크층을 봤다. 되돌아온 이유가 있다.

- 톤 키워드: **의도된 재하강. 검증. 이번엔 다른 것을 보러 왔다.**
- 층 진입 비트는 **짧은 변형 텍스트** 1줄을 앞에 추가한다.
  - (변형 형식): `"[n]회차. {층 이름}."` 1줄 단독 표시 후 원래 비트 표시.
  - 예: `"2회차. 프리온층."` → 원래 비트 5 표시.
  - *이 변형 텍스트는 층 비트를 덮지 않는다. 정보 레이어 1줄 추가.*
- 시그니처 로그는 짧아진다. 이미 알고 있는 것은 새로 놀라지 않는다.
  - 1회차 로그: `"[LOG] 표준 모델 외부 신호 감지. 프리온 — 가설이 아닐 수 있다."`
  - 2회차 로그: `"[LOG] 프리온 — 재확인. 이번엔 끈층 아래를 더 파고든다."`
- 연구소장의 목소리는 빠르고 건조하다. 이미 아는 것은 길게 쓰지 않는다.

---

#### 3회차+ 보이스: 심화의 침묵

3회차 이후 플레이어는 루틴을 알고 있다. 목표는 특정 서브층의 심화다.

- 톤 키워드: **극도로 간결. 말이 줄어든다. 숫자와 측정값만.**
- 층 비트 변형: `"[n]회차. {층 이름}."` 만 표시하고 원래 비트 **생략 가능**.
  - 이미 여러 번 본 층의 진입 비트는 플레이어에게 정보가 아니라 잡음이 된다.
  - 생략 여부는 ux-designer가 "첫 진입만/매번" 토글 설계로 결정.
- 시그니처 로그는 최소화된다:
  - `"[LOG] {n}회차. 이전 관측과 정합."` 또는 표시 생략.
- 연구소장의 목소리는 도달 목표 서브층에서만 팽창한다.
  - 해당 서브층(2회차와 다른 심화 대상)에서만 1회차 수준의 발견 텍스트 부활.
  - 나머지 서브층: 측정값 확인 로그 1줄.

---

#### 회차별 요약

| 회차 | 보이스 밀도 | 비트 처리 | 시그니처 로그 |
|---|---|---|---|
| 1회차 | 보통 — 발견의 긴장 | 전체 표시 | 전체 6개 발현 |
| 2회차 | 간결 — 검증의 집중 | 변형 1줄 추가 후 원본 | 단축 변형 |
| 3회차+ | 최소 — 심화의 침묵 | 선택적 생략 | 심화 대상층만 전체, 나머지 최소 또는 생략 |

### N-3. "시작점" 프레이밍 원칙

> UI 카피·연출이 따라야 할 narrative 가이드.
> ux-designer는 이 원칙을 받아 화면 구성과 버튼 언어를 설계한다.

---

#### 원칙 1. 빅 크런치는 종착점 언어를 사용하지 않는다

금지 표현:
- "게임 클리어", "완료", "엔딩", "축하", "모든 것을 발견했습니다"
- "다시 시작하시겠습니까?" (재시작을 결정으로 만들지 않는다)
- "새 게임", "초기화" (초기화가 아니다)

허용 언어:
- "재하강", "재압축", "다음 런", "새 런"
- "양자 거품 {N} 누적"
- "이전 관측: 유지"

*연구소장은 플랑크층에 도달했다. 실패하지 않았다. 그러나 끝나지도 않았다. 다시 내려갈 이유가 생겼다.*

---

#### 원칙 2. 재하강은 손실이 아닌 선택이어야 한다

압축기를 잃는다. 에너지를 잃는다. 그러나 이것은 **더 강하게 내려가기 위한 선택**이다.

리셋 화면 언어:
- "압축 초기화. QF 누적 유지."가 아니라 → **"이전 런 완료. 양자 거품 {+N} 적립."**
- 잃는 것을 먼저 말하지 않는다. 얻는 것을 먼저 말한다.
- "압축기 리셋됩니다" → "새 압축 시퀀스 준비."

*이 언어 선택이 플레이어가 빅 크런치를 "초기화"가 아니라 "다음 단계"로 읽게 한다.*

---

#### 원칙 3. 새로움은 서브층 심화에서 온다, 복사본 반복에서가 아니다

2회차 이후의 "새로움"을 narrative가 명시적으로 안내한다.

재하강 시작 직후 1회성 텍스트:
> "이전 런에서 닿지 못한 서브층이 있다.  
> 이번엔 거기까지 파고든다."

*이 텍스트는 2회차 첫 시작 시 단 1회만 표시된다. 이후에는 반복하지 않는다.*
*3회차 이후는 생략하거나 더 짧게: "다음 목표를 정한다."*

목적: 플레이어에게 "같은 것을 다시 한다"가 아니라 "다른 목표로 내려간다"는 프레임을 재하강 첫 순간에 심는다. systems의 "이번엔 어느 서브층을 깊이 파는가" 메타를 narrative가 언어로 뒷받침한다.

---

#### 원칙 4. 첫 빅 크런치의 서사적 무게는 단 한 번만 허용한다

1회차 빅 크런치는 이 게임에서 가장 큰 서사적 이벤트다.
2회차 빅 크런치부터는 **같은 연출을 반복하지 않는다.**

| 회차 | 빅 크런치 연출 |
|---|---|
| 1회차 | N-1 전체 시퀀스 — 완전한 카피와 정지 |
| 2회차 | 단축 버전 — QF 수치 + "재압축." + 재하강 안내 |
| 3회차+ | 최소 버전 — QF 수치 + 재하강 안내만 |

연출 반복은 무게를 줄인다. 첫 빅 크런치의 경이는 보존하고, 이후 크런치는 루틴으로 정착시킨다.

---

#### 원칙 5. "작아질수록 거대한 진실에 다가간다" — 역설은 재하강에서도 유지된다

재하강은 물리적으로 dec0으로 돌아가지만, 연구소장은 더 깊은 이해를 가지고 내려간다.
이 역설이 보이스의 중심이다.

재하강의 핵심 카피 문구(UI 어딘가에 고정 표시, 재하강 내내):
> "더 작은 것이 있다."

*이 문구는 처음 재하강 시작 시부터 다음 빅 크런치 때까지 화면 어딘가에 존재한다.
정확히 어디에 표시할지는 ux-designer가 결정한다.
이 문구는 설명하지 않는다. 다음 목적지를 암시한다.*

---

### N-4. narrative.md §5-B와의 정합

이 문서의 빅 크런치 카피(N-1)는 narrative.md §5-B의 기존 4줄 시퀀스를 **교체**한다.

교체 전 (narrative.md §5-B):
```
플랑크 길이 도달.
양자 거품 +{N}.
시공간 재편 중.
새 런을 시작합니다.
```

교체 후 (이 문서 N-1 전체):
- (a) 플랑크 도달 전환 텍스트 (비트 11 후 삽입)
- (b) 최종 QF 폭발 시퀀스 5줄
- (c) 재하강 안내 시퀀스

*교체 이유: §5-B의 "새 런을 시작합니다"는 재시작의 언어다. "이번엔 어디까지 갈 수 있는지 알고 있다"는 재하강의 언어다.*

*narrative.md 본문은 수정하지 않는다. 이 문서가 §5-B 빅 크런치 항목의 교체본임을 gdd-sync 시 반영.*

### U-1. 빅 크런치 시퀀스 — 화면 연출 타임라인

> N-1 카피의 타이밍 지시(0.8s/0.4s/0.7s/3s 암전)를 화면 전환·상태 명세로 변환한다.
> game-programmer는 이 타임라인을 구현 스펙으로 사용한다.

---

#### 상태 머신 개요

```
[일반 압축 화면]
       ↓ dec26 완료 트리거
[PLANCK_HIT]    — 플랑크 도달 전환 텍스트 (0.8s)
       ↓
[QF_BURST]      — 최종 QF 폭발 시퀀스 (2.7s + 0.7s 정지 = 3.4s)
       ↓
[BLACKOUT]      — 3s 암전
       ↓
[RUN_START]     — 재하강 안내 (1회차 풀 / 2회차 단축 / 3회차+ 최소)
       ↓
[FOCUS_SELECT]  — 집중 서브층 선택 (2회차+ 전용)
       ↓
[MAIN_RESET]    — 압축 메인 화면 (dec0 재시작)
```

---

#### 타임라인 A — 1회차 빅 크런치 (N-4: 풀 시퀀스)

```
T+0.0s   [화면 전환 잠금] 일반 압축 UI 전체 입력 차단
          — 배경: 현재 화면을 0.3s에 걸쳐 dim(opacity 0.0으로 페이드)

T+0.0s   [PLANCK_HIT 화면 등장]
          배경: 완전 검정
          중앙 텍스트 (mono 폰트, 소형):
            "1.616×10⁻³⁵ m."
            (0.2s 뒤) "시공간의 격자가 끝난다."
            (0.2s 뒤) "더 이하의 거리는 측정할 공간이 없다."

T+0.8s   [화면 전환 — QF_BURST 화면]
          배경: 완전 검정 유지
          중앙 텍스트 순차 등장 (각 줄 사이 0.4s):
            "압축 한계 도달."
            (0.4s 뒤) "양자 거품 +{N}."
            (0.4s 뒤) "임계점."

T+2.0s   [0.7s 정지] — 화면 변화 없음

T+2.7s   [두 줄 추가 등장, 줄 사이 0.3s]
            "붕괴가 아니다."
            (0.3s 뒤) "재압축이다."

          [시각 이펙트] "재압축이다." 등장 시:
            — 중심에서 바깥으로 퍼지는 흰 링 1회 (0.4s 지속, opacity 0→1→0)
            — 화면 전체 0.15s 흰 플래시(opacity 0.12 max)
            — QF 수치 레이블 "{+N} QF 획득"이 화면 우상단에 0.5s fade-in

T+3.4s   [BLACKOUT — 3s 암전]
          화면 완전 검정, 텍스트 없음
          — 배경음악 볼륨 0으로 페이드 (2s)
          — 3s 경과 후 RUN_START로 전환

T+6.4s   [RUN_START 화면 등장]
          (아래 U-2에서 상세)
```

#### 타임라인 B — 2회차 빅 크런치 (N-4: 단축 버전)

```
T+0.0s   [화면 전환 잠금 + dim]

T+0.0s   [단축 QF_BURST]
          텍스트 2줄만 (줄 사이 0.3s):
            "재압축."
            (0.3s 뒤) "양자 거품 +{N}. 누적 {총 QF}."
          — 링 이펙트 없음 / 플래시 없음
```

#### 구현 노트 (game-programmer)

- `run_index`(1 / 2 / 3+)를 저장소에서 읽어 타임라인 분기.
- 텍스트 등장은 CSS `opacity` 트랜지션(linear). JS `setTimeout` 체인으로 구현.

### U-4. 회차별 차별화 정보 표시

> 방치형 정보밀도 원칙: 한눈에 읽힘. 과부하 금지.
> 현재 회차·집중 서브층·D 보존율을 ux.md §2 메인 화면 레이아웃 안에서 처리한다.

---

#### 4-A. 회차·집중 서브층 배지 (메인 압축 화면 상단)

기존 탭바 우측에 현재 층·decade를 표시하는 영역(ux.md §2-A)에 회차 정보를 추가한다:

```
[탭바 우측 고정 영역]
  원자층 — dec 1.42    [2회차  ·  집중: 끈층]
```

- 회차 배지: 작은 폰트, 회색. "2회차"
- 집중 서브층 배지: 같은 줄, 구분점(·) 뒤에 "집중: {층 이름}"
- 1회차: 배지 없음 (첫 런은 자유 탐색, 불필요한 메타 정보 차단)
- 집중 서브층에 도달 시: 배지 색상이 QF 녹(#3ecf8e)으로 변환되어 "지금 집중 구간"을 시각 신호

---

#### 4-B. D 보존율 표시 — 재하강 직후 1회만

RUN_START 화면의 왼쪽 정보 블록(1회차 풀 버전)에 추가:

```
   dec 0.   분자층.
   압축기: 없음.
   양자 거품: {누적 QF}.
   이전 런의 관측이 남아있다.
   D 이월: {이월 D 수치} ({보존율}%)    ← 2회차+ 전용 추가 행
```

- 메인 압축 화면에서는 표시하지 않는다. RUN_START 화면에서 1회 확인 후 진행.
- 비율은 원칙2(얻는 것 먼저) 준수: "{이월 D 수치} ({보존율}%)" 순서(수치 먼저, 비율은 괄호 안).

---

#### 4-C. 비용 감소율 — 암묵 표시

비용 감소율(systems 산출물3 §3-2 레버iii)은 별도 UI로 표시하지 않는다.
플레이어가 "이번 런이 더 빠르다"는 체감으로 인지하는 것이 목적이기 때문이다.
단, 압축기 구매 화면(ux.md §2-C 체인 테이블)에서 비용 수치가 이미 감소된 값으로 표시되면 충분하다.

명시적 "이번 회차 비용 -5%" 표시는 메타 정보 과부하다. 숫자가 직접 말하게 한다.

---

#### 4-D. 집중 서브층 진입 시 강조 연출

집중 서브층(현재 회차 선택 서브층)에 처음 진입할 때 1회성 알림:

```
[우측 패널 층 카드 — 집중 서브층 진입 순간]
  ─────────────────────
  {층 이름}
  [집중 서브층 — 심화 콘텐츠 해금]
  ─────────────────────
  입자 {X}개 · 연구 노드 {Y}개 추가
  ─────────────────────
  (3초 후 일반 층 카드로 전환)
```

- 팝업·오버레이 없이 우측 패널 층 카드 내부에서만 처리한다.
- 3초 후 일반 층 카드(층 이름 + 짧은 설명 1줄)로 자동 전환.
- 이후 해당 서브층 체류 동안 탭바 배지의 "집중: {층 이름}" 텍스트가 QF 녹으로 유지된다.

### U-5. 상전이 탭 점등 조건 확정

#### 5-A. 점등 조건 완전 표

| 빅 크런치 도달 — dec26 캡 | **점등** — 빅 크런치 특별 | 탭 점등 후 자동 PLANCK_HIT 시퀀스 진입(U-1 타임라인 A/B/C 분기) |
| 재하강 후 알려진 물리 재진입 (dec0~19, 2회차+) | 비점등 | 탭바 배지 "2회차 · 집중: {층}" 표시만. 탭 ● 없음 |
| 재하강 후 미지 서브층 벽 재도달 (dec19~26, 2회차+) | **점등** — 일반 점등 | 정적 ● 표시. 첫 상전이 특별 연출은 1회차에만 |

---

#### 5-B. 빅 크런치 상전이 탭 점등 특이사항

빅 크런치(dec26)는 일반 상전이 탭 점등 후 바로 시퀀스로 진입하므로, 상전이 탭 화면이 중간 화면으로 작동하지 않는다.

```
흐름:
  dec26 도달
  → 상전이 탭 ● 점등 (0.3s 표시)
  → 자동으로 PLANCK_HIT 화면 전환 (U-1 타임라인)
  → 시퀀스 종료 후 FOCUS_SELECT (2회차+) 또는 MAIN_RESET (1회차 복귀)
```

"플레이어가 상전이 탭을 클릭해야 진행"하는 일반 상전이와 달리, 빅 크런치는 자동 시퀀스 진입이다. 이는 필러 ①("작아짐=강해짐")의 정점 연출이 플레이어 클릭 타이밍에 의존하지 않도록 보호한다.

#### 5-C. game-programmer 구현 명세

```
// 상전이 탭 점등 조건 (Svelte 스토어 또는 게임 상태 기반)
function shouldLightPhaseTab(current_dec: number, run_index: number): boolean {
  const UNKNOWN_WALLS = [];           // 알려진 물리 5층 — 점등 없음
  const KNOWN_WALLS = [19, 20.5, 22, 23.5, 25, 25.5, 26]; // 미지 벽 6 + 빅 크런치
  return KNOWN_WALLS.some(wall => Math.abs(current_dec - wall) < EPSILON);
}

// 첫 상전이 특별 점등 (dec19, 1회차만)
function isFirstPhaseTransition(current_dec: number, run_index: number): boolean {
  return run_index === 1 && Math.abs(current_dec - 19) < EPSILON;
}

// 빅 크런치 자동 시퀀스 (dec26)
function isBigCrunch(current_dec: number): boolean {
  return Math.abs(current_dec - 26) < EPSILON;
}
// isBigCrunch → true 시 자동으로 U-1 타임라인 실행 (run_index 분기)
```

### U-6. 화면 흐름 전체 요약 (핵심 3~5줄)

```
1. dec26 도달 → 자동 빅 크런치 시퀀스 (run_index에 따라 타임라인 A/B/C 분기,
   1회차 6.4s 풀 / 2회차 2.1s 단축 / 3회차+ 0.5s 최소)

2. 시퀀스 종료 → RUN_START 화면 (1회차 풀/2회차 단축/3회차+ 최소,
   N-3 원칙 엄수: 종착점 언어 금지, 얻는 것 먼저, "새 압축 시퀀스 준비" 버튼)

3. "새 압축 시퀀스 준비" 클릭 → 2회차+이면 FOCUS_SELECT (6개 서브층 카드 선택,
   이전 선택 이력 표시, 심화 콘텐츠 힌트 툴팁), 1회차는 바로 메인으로 복귀

4. 메인 압축 화면 복귀 → 탭바에 "N회차 · 집중: {층}" 배지,
   우하단 "더 작은 것이 있다." 고정 문구 (다음 빅 크런치까지 유지)

5. 상전이 탭 점등: 미지 6벽(dec19·20.5·22·23.5·25·25.5) + 빅 크런치(dec26)만.
   알려진 물리 5층(dec0~19) 및 재하강 후 알려진 물리 재진입은 비점등.
```
