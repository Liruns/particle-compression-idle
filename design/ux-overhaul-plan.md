## 0. 한 줄 결론

설계 문서(`ui-flow.md §2-A`)는 **데스크톱 3패널 레이아웃**(LEFT 160–180px / CENTER 가변 / RIGHT 180–200px)을 명시했는데, 구현은 **전 컴포넌트 `max-width:460px` 단일 세로 컬럼**으로 끝났다. 의도와 구현이 정면으로 어긋났고, 그 결과 데스크톱에선 검은 여백 속 좁은 기둥, 모바일에선 미검증 상태다. **이건 "덜 다듬어진 UI"가 아니라 "설계된 레이아웃이 구현되지 않은" 상태다.**

## 1. 비판적 감사 — 현 UX의 구체적 결함

### 1-1. [치명] 460px 기둥 — 설계된 레이아웃의 미구현

**사실 (grep 검증):** `max-width:460px`가 10곳에 하드코딩됨 — `App.svelte`(.tabs / .resources / .ftue-hint), `ChainTable`, `LayerCard`, `PhaseWidget`, `ResonanceWidget`, `HarmonicsWidget`, `PrestigeView`. 레이아웃 적응용 `min-width`/`@media` 브레이크포인트는 **0개**. 존재하는 `@media`는 전부 `prefers-reduced-motion`(모션 감쇠)과 `CodexView`의 단일 `max-width:560px`(사이드 접힘)뿐.

**증상 (스크린샷):**
- `station2-after-L1.png`: 폭 ~945px 캡처인데 콘텐츠는 중앙 ~460px에만 갇히고 좌우로 거대한 검은 띠. 1080p·1440p 모니터로 갈수록 낭비가 비례 확대. **PC 게임의 화면이 아니다.**
- 모든 정보(게이지·층카드·공명·자원 4행·압축 버튼·체인 8행)가 **한 줄로 세로 적층** → 분자층 첫 화면에서 이미 스크롤 발생(`station2-fresh-L1-mol.png` 우측 스크롤바·하단 잘린 압축 버튼). 방치형의 핵심 가치인 "한눈에 상태 파악"이 깨졌다.

**의도와의 어긋남:** `ui-flow.md §2-A`는 3패널 ASCII 와이어프레임을 그려놨고, `§2-A 반응형 분기점`에 "화면 폭 < 900px: LEFT PANEL 접힘"까지 적어놨다. **즉 ">= 900px일 때 3패널"이 설계 전제인데 그 분기가 코드에 없다.** `§12-A`는 "데스크톱(Tauri) 최소폭 900px 강제 / 최적 1280×720"이라고 못박았다. 1280px를 깔고 460px만 쓰는 건 설계 위반.

### 1-2. [치명] 모바일 미최적화 — itch.io 타깃인데 터치 동선 부재

`index.html`의 뷰포트 메타(`width=device-width, initial-scale=1.0`)는 토대로 있으나 레이아웃이 안 따라간다. 모바일에서 실제로 깨지는 것:

- **세로 스크롤 지옥:** 460px 단일 컬럼이 그대로 모바일 폭으로 줄면, 게이지→층카드→공명→자원4행→버튼→체인8행이 전부 세로로 쌓여 첫 화면에 핵심 액션([압축])이 안 보인다. 방치형은 "열자마자 진행도 + 핵심 버튼"이 보여야 하는데 둘 다 fold 아래로 밀린다.
- **하단 탭바 부재:** 탭바가 `<header>` 바로 아래 상단에 있다(`App.svelte:276`). 모바일에서 탭 전환은 엄지로 가장 자주 누르는 동작인데 화면 최상단에 둬서 엄지 도달성(thumb-zone)이 최악. 한 손 조작 불가.
- **터치 타깃 점검 결과 (혼재):**
  - 양호: [압축] 버튼 `min-height:44px`(App.svelte:665), 공명 슬롯 64px(ResonanceWidget:124), 상전이 버튼 `min-height:44px`.
  - **미달:** 체인 구매 버튼 `min-height:28px`(ChainTable:183) — WCAG AA 24px는 넘지만 44px 권장 미달, 모바일에서 8행이 28px로 빽빽하면 오터치 빈발. 대량구매 모드 버튼(×1/×10/×100/Max) `padding:3px 8px`(ChainTable:124) → 실측 높이 ~22px, **AA 24px 미달**. 도감 층 사이드 버튼 `padding:5px`(CodexView), 저장 버튼 등도 44px 미확보.
  - `accessibility.md §4-A`가 "탭바 최소 44px 높이, 구매 버튼 32px"를 명시했는데 탭바엔 명시적 min-height가 없고(`.tab` padding만), 구매 버튼은 28px다. **자기 문서 기준 미준수.**

### 1-3. [높음] 정보 위계 붕괴 — "지금 중요한 것"이 안 보인다

`station2-after-L1.png`를 정보 위계로 읽으면:
- 화면에서 시각적으로 가장 강한 건 "Micro Idle" h1(20px, 화면 폭 중앙 최상단) — **게임 타이틀이 매 순간 가장 큰 요소다.** 방치형에서 타이틀은 한 번 보면 되는 정보. 매번 1순위로 노출될 이유가 없다(`art-director` 레인과 겸침: 위계는 내 레인, 글자 크기 값은 art).
- r 게이지(핵심 "작아짐" 비주얼)와 자원 readout이 **같은 세로축에서 동등한 시각 무게**로 나열 → 무엇이 주이고 무엇이 보조인지 위계 신호가 없다.
- 자원 4행(C/E/D/배율)이 게이지와 압축 버튼 사이에 끼어 있어, **핵심 액션 동선(게이지 보고 → 압축 누르고 → 체인 산다)이 자원 행에 의해 끊긴다.** 압축 버튼이 자원 블록 *아래*에 있어 게이지에서 버튼까지 시선 이동이 길다.
- `ux.md §1`이 천명한 "지금 중요한 것 하나를 크게, 나머지는 수축"이 단일 컬럼에선 물리적으로 불가능 — 모든 게 같은 폭, 같은 흐름이라 "수축"할 자리가 없다.

### 1-4. [높음] 빈 공간 활용 0 + 정보 밀도 역설

데스크톱에서 좌우 검은 띠(화면의 50%+)가 완전히 죽어있다. 그런데 동시에 **세로로는 스크롤이 나도록 빽빽하다.** 공간이 남는데 정보는 넘치는 모순 — 횡으로 펼치면 해결될 것을 종으로만 쌓은 결과. 방치형은 정보 밀도가 무기다(체인 8행 + 자원 + 게이지 + 메커니즘 위젯을 동시에 보며 "다음 살 것"을 판단). 단일 컬럼은 이 동시성을 스크롤로 분해해버린다.

### 1-5. [중간] 화면 흐름 — 탭 전환 시 컨텍스트 상실

- 압축↔연구↔도감↔상전이 탭은 **상호 배타적 전체 교체**(App.svelte:299–386, `{#if tab===...}`). 연구 탭을 열면 게이지·자원·체인이 전부 사라진다. 방치형에서 "연구 사면서 자원이 충분한지" 같은 교차 확인이 불가 → 탭을 왕복해야 한다. 데스크톱 넓은 화면에선 자원 readout을 상주시키면 해결될 문제다(`ux.md §2` LEFT PANEL이 정확히 이 의도였다).
- 메커니즘 위젯(공명·위상·하모닉스)이 압축 탭 본문 **세로 흐름 중간**에 들어가 게이지와 자원 사이를 더 벌린다. `ux.md §4`·`ui-flow.md §2-E`는 이걸 **RIGHT PANEL**에 두기로 설계했다 — 즉 게이지 옆 보조 영역. 현재는 메인 흐름에 끼어 흐름을 끊는다.

### 1-6. [중간] 천문학적 수치 — 표기 토대는 있으나 맥락 부족

`formatNumber`/`formatRadius`로 과학표기 자체는 처리됨(스크린샷의 `7.620×10⁻¹³ m`, `1.58×10⁶ E`). 다만:
- `ux.md §2-B`·`ui-flow.md §2-C`가 약속한 **접두사↔과학 표기 토글**(설정)이 화면에 노출 안 됨 — 캐주얼/하드코어 양쪽 가독성을 위한 옵션인데 빠졌다(설정 탭 자체가 미구현).
- decade 진행 바(`ui-flow.md §2-B`의 `dec N ──[■■■]── dec N+1`)가 없다. 현재 `dec 3.118` 숫자만 → "이 층에서 얼마나 왔나"를 한눈에 못 읽는다. 방치형 진행감의 핵심 시각 요소 누락. (바 자체 비주얼 디테일은 art지만, "진행 바가 IA상 필요"는 내 레인.)

### 1-7. [낮음·기록] 잘 된 점 (변호 아님, 재사용 자산)

- 토큰 시스템(`tokens.css`)이 `--space-*`/`--text-*`/층 팔레트로 잘 잡혀 있어 **반응형 재배치는 토큰 위에서 가능** — 처음부터 다시 안 짜도 된다.
- `CodexView`만 유일하게 2열 그리드 + `@media(max-width:560px)` 단일 컬럼 폴백을 갖춤 → **이 컴포넌트가 반응형 패턴의 참조 모델**이 된다.
- 단방향 스냅샷 구독 구조라 레이아웃을 바꿔도 게임 로직 불변(요구사항 충족 용이).

## 2. PC/Mobile 반응형 전략 (핵심)

### 2-0. 설계 원칙

> **하나의 컴포넌트 트리, 세 개의 레이아웃.** 컴포넌트는 그대로 두고, 셸(App.svelte)의 그리드와 컨테이너 폭만 브레이크포인트로 재배치한다. 컴포넌트 내부 `max-width:460px`는 전부 제거하고 `width:100%`로 바꿔 **부모 그리드 셀이 폭을 결정**하게 한다(현재는 컴포넌트가 자기 폭을 우겨서 그리드를 못 짠다).

### 2-1. 브레이크포인트 정의

| 토큰 | 범위 | 폼팩터 | 레이아웃 |
|---|---|---|---|
| `--bp-narrow` | < 720px | 모바일·세로 (itch.io, Steam Deck 세로) | **단일 컬럼 + 하단 탭바** |
| `--bp-mid` | 720–1079px | 좁은 데스크톱·태블릿·세로 모니터 | **2컬럼** (메인 + 우측 보조) |
| `--bp-wide` | >= 1080px | 데스크톱·Steam (1280×720 기준) | **3컬럼** (좌 자원 / 중 게이지·체인 / 우 메커니즘·층) |

근거: `ui-flow.md §2-A`가 900px를 LEFT 접힘 분기로, `§12-A`가 데스크톱 최소 900·최적 1280을 명시. 720을 모바일 상한으로 잡는 이유 — 460px 컬럼 + 좌우 최소 여백 + 우측 보조 패널(180px)이 들어가려면 ~700px가 필요하므로 그 아래는 단일 컬럼이 정답. 1080을 wide 기준으로 올리는 이유 — 3패널(180+가변+200 = 최소 ~700에 여백)이 답답하지 않으려면 중앙 CENTER가 충분히 넓어야 함(1280 기준 CENTER ~640px 확보).

CSS 구현은 `min-width` 미디어쿼리(mobile-first) 또는 컨테이너 쿼리. 셸은 단순하므로 **`@media (min-width: 720px)` / `(min-width: 1080px)` 2단**으로 충분.

### 2-2. 데스크톱 (wide, >= 1080px) — 3패널 적응형

```
┌──────────────────────────────────────────────────────────────────────────┐
│  [상단 바]  Micro Idle(소형)   |  탭: 압축·연구·도감·상전이●·설정  | 원자층 dec3.12 [N회차] │
├────────────────┬─────────────────────────────────────┬─────────────────────┤
│ LEFT (200px)   │  CENTER (가변, 최소 560px)          │  RIGHT (240px)      │
│ [상주]         │                                     │  [탭 종속]          │
│                │   ┌───────────────────────────┐     │                     │
│ 자원 Readout   │   │   r 게이지 (원형·글로우)    │     │  현재 층 카드       │
│  ⚡ E +rate    │   │   r = 7.62×10⁻¹³ m         │     │  층명·dec범위·메커  │
│  ◎ C +rate    │   │   dec [■■■□□] 진행 바      │     │  ─────────          │
│  ▣ D          │   └───────────────────────────┘     │  메커니즘 위젯       │
│  ◆ QF         │                                     │  (공명/위상/하모닉)  │
│  × 배율        │   [ 압축 ]  ← 게이지 바로 아래       │  ─────────          │
│ ───────────   │                                     │  도감 진행 N/전체    │
│ 생산 배율      │   ── 8단 체인 테이블 ──             │  연구 가지 현황      │
│ QF 부스트      │   T1 ▶ / T2 / … / T8               │  ─────────          │
│ 공명 배율      │   (대량구매 ×1/10/100/Max)         │  상전이 힌트         │
│ ───────────   │                                     │  (조건 충족 시)      │
│ 상태 로그 4줄  │                                     │  "더 작은 것이 있다" │
└────────────────┴─────────────────────────────────────┴─────────────────────┘
```

**핵심 결정 — 정보 밀도 vs 집중의 균형:**
- **LEFT(자원·로그)는 모든 탭에서 상주.** 연구·도감을 열어도 좌측에 자원이 남아 "D 충분한가"를 왕복 없이 확인(1-5 결함 해소). 이게 `ux.md §2`의 원래 의도.
- **CENTER는 탭 콘텐츠.** 압축 탭=게이지+체인, 연구 탭=연구 트리(2–3열 그리드로 횡 확장), 도감 탭=도감(이미 그리드 보유). 상전이 탭=상전이 화면. **즉 탭은 CENTER만 교체**하고 LEFT/RIGHT 골격은 유지 → 컨텍스트 상실 제거.
- **RIGHT는 메커니즘+층 컨텍스트.** 압축 탭에서 메커니즘 위젯이 게이지 *옆*(흐름 안 끊김). 연구/도감 탭에선 RIGHT를 접거나 층 카드만 축약.
- **빈 공간 해소:** 좌우 패널이 검은 띠를 정보로 채움. CENTER 최대폭 캡(예: 720px)을 줘서 초광폭(21:9)에서 게이지가 과도하게 안 퍼지게 — 남는 폭은 좌우 패널 gutter로.

**게이지 = 마케팅 후킹:** CENTER 상단에 게이지를 크게 두면 스토어 GIF의 "r 줄고 글로우 커짐" 루프가 화면 주인공이 된다(`ux.md §2-A`·`§7-1`). 현재는 게이지가 세로 흐름 한 칸일 뿐이라 후킹이 약하다.

### 2-3. 모바일 (narrow, < 720px, 터치) — 단일 컬럼 + 하단 탭바

```
┌─────────────────────────────┐
│ [상단 슬림] 원자층 dec3.12   │  ← 타이틀 축소/제거, 층·dec만 (1-3 위계 교정)
├─────────────────────────────┤
│                             │
│     r 게이지 (원형)         │  ← fold 최상단: 핵심 비주얼
│     r = 7.62×10⁻¹³ m       │
│     dec [■■■□□]            │
│                             │
│   ┌───────────────────┐     │
│   │   [ 압  축 ]       │     │  ← fold 안에 핵심 액션 (큰 버튼, 폭 100%)
│   └───────────────────┘     │
│                             │
│   자원 칩 (가로 스크롤/2×2)  │  ← 4행 세로적층 대신 컴팩트 칩
│   ⚡E  ◎C  ▣D  ×배율       │
│                             │
```

**모바일 핵심 결정:**
- **탭바를 하단 고정**(`position:sticky/fixed; bottom:0`). `< 720px`에서만. 엄지로 가장 자주 누르는 탭 전환을 thumb-zone에 둠(1-2 해소). 데스크톱에선 상단 유지(마우스는 상단도 동등).
- **fold 규칙:** 첫 화면(스크롤 전)에 반드시 (1) 게이지+r+진행바, (2) [압축] 버튼이 보이게. 자원/체인은 그 아래 스크롤. → "열자마자 진행도+핵심 액션"(방치형 동선) 충족.
- **자원 4행 → 컴팩트 칩:** 세로 4행(각 full-width)은 모바일에서 과도. 2×2 그리드 또는 가로 칩으로 압축해 fold 절약. 상세 rate는 탭/롱프레스로.
- **터치 타깃 일괄 44px:** 체인 구매 버튼 28→44, 대량구매 모드 버튼 22→`min-height:32px`(AA 충족)+간격 확대, 탭바 탭 `min-height:48px`, 도감 사이드 버튼 44. (`accessibility.md §4-A` 자기 기준 준수.)
- **메커니즘 위젯 접이식:** 현재 층 위젯만 펼치고 과거 층 위젯은 `ui-flow.md §2-E`의 "축소 상태(제목+핵심 수치 1줄)"로. 모바일 세로 절약.

### 2-4. 태블릿/중간 (mid, 720–1079px) — 2컬럼

- **CENTER + RIGHT** 2컬럼(LEFT 자원은 RIGHT 상단으로 합치거나 CENTER 상단 가로 자원 바로). 게이지·체인은 CENTER, 메커니즘·층은 RIGHT.

## 3. 정보설계(IA) 개선

### 3-1. 탭 구조 — 현 4탭 → 5탭 + 상주 영역 분리

- **현재:** 압축/연구(조건)/도감(조건)/상전이(조건). 설정 탭 **없음**(저장 버튼만 압축 탭에). → `ux.md §1`의 5탭(압축/연구/도감/상전이/**설정**) 미완.
- **개선:**
  1. **설정 탭 추가**(`ui-flow.md §6` 스펙 존재). 표기법 토글·접근성(모션/텍스트크기/색맹)·세이브가 들어갈 자리. 현재 압축 탭에 떠 있는 [저장]은 설정으로 이관(압축 탭 정리 + 1-3 위계 개선).
  2. **상주 vs 탭 분리(데스크톱):** 자원·로그·층 컨텍스트는 "탭"이 아니라 "상주 패널". 탭은 압축/연구/도감/상전이/설정의 **주작업 영역(CENTER)만** 전환. → 교차 확인 가능(1-5 해소).
  3. **탭 상태 배지 강화:** `ux.md §1` "탭을 안 열어도 상태를 안다" 원칙. 도감 배지(수집 수)는 있음. 연구에 "구매 가능 노드 N" 배지, 상전이에 ● + "준비" 텍스트(색맹 대응, `accessibility.md §1-B`) 추가.

### 3-2. 우선 정보 노출 (방치형 "한눈에 상태")

화면 어디서든 **항상 보여야 할 4종**(데스크톱 상주 / 모바일 fold+탭바):
1. **진행 위치** — 현재 층 + dec + decade 진행 바(어디까지 왔나).
2. **핵심 자원 속도** — C·E의 +/s(방치 생산이 도는가).
3. **다음 액션 후보** — 체인 최적 티어 ▶(이미 구현됨, ChainTable) + 살 수 있는지(affordable 색).
4. **임박 이벤트** — 상전이 준비 ● / 공명 슬롯 열림(개입 타이밍).

현재는 탭 전환 시 1·2가 사라진다 → 상주화로 해결.

### 3-3. 점진 공개(progressive disclosure) — 유지하되 레이아웃 인지

`ui-flow.md §11-A`의 점진 공개(자원 E,C→D→QF / 탭 순차 / 체인 +1 노출)는 **잘 설계됐고 구현도 됨**(App의 `showResearchTab`/`showCodexTab`/`showPrestigeTab`/`ftue.*`). 오버홀이 깨면 안 되는 자산. **단** 패널 레이아웃 전환 시 빈 패널 처리 주의 — FTUE 0~1분엔 RIGHT 패널(메커니즘)이 비어있다. 그 구간엔 RIGHT를 숨기고 CENTER를 넓게(게이지 집중) → 온보딩 단순성 유지. 패널은 **콘텐츠가 생길 때 등장**(자원처럼 페이드인).

### 3-4. 온보딩(FTUE) IA — 레이아웃 관점 보강

`ui-flow.md §7` FTUE 흐름(클릭→T1→공명→D→탭 해금)은 탄탄. 레이아웃 측면 2가지만 추가:
- **0~2분은 의도적으로 단일 컬럼처럼:** 데스크톱이라도 FTUE 초반엔 패널을 늦게 펼쳐 정보 과부하 방지. 게이지+압축 버튼만 크게 → 첫 인터랙션 명확(요구사항 "방치형이라도 첫 인터랙션은 명확"). 패널은 해당 시스템 해금과 함께 슬라이드인.
- **모바일 FTUE:** 하단 탭바는 탭이 1개(압축)일 때 숨기거나 비활성 → 첫 화면 깔끔. D 획득으로 연구/도감 해금되는 순간 탭바 등장(점진 공개를 탭바 출현으로 시각화).

## 4. 우선순위 실행목록 (P0/P1/P2)

> 게임 메커니즘 불변. 아래는 전부 표현·레이아웃·CSS·셸 구조. 구현은 game-programmer.

### P0 — 레이아웃 골격 (반드시 먼저, 나머지의 토대)

- **P0-1. 컴포넌트 폭 해방.** 9개 컴포넌트의 `max-width:460px`(및 ChainTable/Research/Prestige 등) → `width:100%` 제거하고 폭은 부모 그리드가 결정. *대상:* `App.svelte`(.tabs/.resources/.ftue-hint), `ChainTable`, `LayerCard`, `PhaseWidget`, `ResonanceWidget`, `HarmonicsWidget`, `PrestigeView`. *주의:* CENTER 최대폭 캡은 **셸 컨테이너**에서 1회만(예: `.center{max-width:720px}`).
- **P0-2. 셸 그리드 도입 (App.svelte).** `main`을 `display:flex column`(현재)에서 **반응형 그리드 셸**로 교체:
  - `>= 1080px`: `grid-template-columns: 200px minmax(0,1fr) 240px`(LEFT/CENTER/RIGHT), 영역 `grid-template-areas`.
  - `720–1079px`: 2컬럼(CENTER/RIGHT 또는 자원 상단바 + CENTER).
  - `< 720px`: 1컬럼.
  - CSS 변수로 `--bp-narrow:720px`, `--bp-wide:1080px` 정의(tokens.css 또는 App).
- **P0-3. 상주/탭 영역 분리 (App.svelte 마크업).** 자원·층 컨텍스트를 `{#if tab==='compress'}` **밖으로** 빼서 LEFT/RIGHT 패널에 상주(데스크톱). 탭 `{#if}`는 CENTER 콘텐츠만 감싸도록 리팩터. *모바일:* 미디어쿼리로 자원을 압축 탭 본문에 인라인(상주 패널 숨김) — 또는 동일 마크업을 CSS `order`/`grid-area`로 재배치.
- **P0-4. 하단 탭바 (모바일).** `.tabs`에 `< 720px`일 때 `position:sticky; bottom:0; z-index` + `justify-content:space-around` + 탭 `min-height:48px`. 데스크톱은 현 상단 유지. (art-director와 겸침: 바 배경·구분선 색은 art, 위치·도달성은 ux.)
- **P0-5. 터치 타깃 일괄 점검.** 체인 구매 28→44px, 대량구매 모드 버튼 `min-height:32px`+gap 확대, 탭바 48px, 도감 사이드/저장 버튼 ≥44px. (`accessibility.md §4-A` 준수.)

### P1 — 정보 위계·동선·핵심 누락 요소

- **P1-1. 압축 탭 흐름 재정렬.** [압축] 버튼을 자원 블록 위(게이지 직하)로 이동 → 게이지→버튼 동선 단축, 자원이 흐름을 안 끊게(1-3·1-5). *대상:* `App.svelte` `.actions`를 `.gauge` 바로 뒤로.
- **P1-2. 메커니즘 위젯을 RIGHT로.** 공명/위상/하모닉 위젯을 압축 탭 본문 흐름에서 RIGHT 패널(데스크톱)로. 모바일은 게이지+버튼+자원 **다음**, 체인 앞 접이식. *대상:* `App.svelte` 위젯 3개의 배치 위치.
- **P1-3. decade 진행 바 추가.** 게이지 직하에 `dec N ──[■■■□□]── dec N+1` 진행 바(`ui-flow.md §2-B`). 층 내 진행감 = 방치형 핵심 시각. (바 채움 색·글로우는 art, 막대 존재·위치·라벨은 ux. 데이터: 현재 dec의 층 내 정규화 — game-programmer가 layer.decadeRange로 산출.)
- **P1-4. 설정 탭 + 표기법 토글.** `ui-flow.md §6` 최소 구현: 세이브/로드(압축 탭 [저장] 이관), 과학↔접두사 토글(`format`에 모드 인자), 모션감소/텍스트크기. 탭바에 5번째 탭.
- **P1-5. 자원 컴팩트 칩 (모바일).** `.resources` 세로 4행을 `< 720px`에서 2×2 또는 가로 칩 그리드로(`grid-template-columns` 미디어쿼리). fold 절약.
- **P1-6. 타이틀 위계 강등.** 상단 "Micro Idle" h1을 데스크톱 상단 바 좌측 소형 로고로, 모바일은 층·dec만 남기고 타이틀 축소/숨김. (글자 크기 값은 art와 협의 — 위계 의도는 ux.)

### P2 — 폴리시·적응 심화

- **P2-1. 중간(mid) 2컬럼 정밀화.** 720–1079px에서 LEFT 아이콘 접힘 바(`ui-flow.md §2-A` "<900px LEFT 접힘") vs 2컬럼 중 택1 실측 후 확정.
- **P2-2. 초광폭 대응.** >= 1600px에서 CENTER 캡 유지 + 좌우 패널 폭 소폭 확대 or gutter, 게이지 과확산 방지.
- **P2-3. 탭 종속 RIGHT 패널 컨텍스트화.** 연구 탭일 때 RIGHT에 "선택 가지 요약", 도감 탭일 때 "완성도/홀로 배율"을 상주(현재 각 뷰 내부에 있는 걸 패널로 승격) — 탭 내부 스크롤 절약.
- **P2-4. 키보드 단축키 1~5 탭 이동**(`accessibility.md §4-B`) — 데스크톱 방치형 핵심 단축. 패널 상주 구조와 함께면 자연스러움.
- **P2-5. 빈 패널 FTUE 처리.** 0~2분 RIGHT 패널 숨김→해금 시 슬라이드인 로직(3-3).
