## 산출물 1 — 연구 시너지 중첩 상한 + ×1.06 헤드룸 해소

### 1-1. 문제 정의

director-review §3.4의 30% 가드레일 적용 결과:
- dec26 −30% 한도 ↔ QF 기본 부스트 위에 곱해지는 추가곱 H ≤ ×1.43(1 − 1/1.43 = 0.300)
- 도감 만렙(×1.35) 단독으로 −25.9%를 소진 → 잔여 예산: 1.43/1.35 = **×1.059 ≈ ×1.06**
- "완전 시너지" B12와 "전체 티어 연쇄" A13이라는 이름값에 비해 ×1.06 헤드룸은 체감상 빈약

대상 노드 4개: A13(전체 티어 연쇄), B8(3중 메커니즘 시너지), B12(완전 시너지), B_HOLO(홀로그래픽 증폭). 이 4개가 research_mult에 동시 곱해질 때의 worst-case 분석이 핵심 과제.

### 1-2. 노드 해금 타이밍 분석 (중첩·활성 타이밍 모델)

각 노드가 단일런 dec26 도달 "시점"에 실제로 활성 상태인지, 도감 만렙과 동시에 곱해지는지를 해금 조건으로 추적한다.

|---|---|---|---|---|

#### 결론: worst-case 동시곱 시나리오

**B8은 조기 해금(dec6~9, ~1.2h 내)이다.** B8 활성 조건(공명 활성 + 게이지 50% + 균형 달성)은 알려진 물리 층에서 이미 구성 가능하므로, B8은 단일런 전 구간에 걸쳐 "잠재적 활성" 상태다. 단, B8의 배율은 세 메커니즘 동시 활성 "때만" 발생하는 조건부 보너스(이벤트 배율)이며, 항상 곱해지는 상수배율이 아니다.

**A13·B12·B_HOLO는 후기 해금(dec19~25+)이다.** 이 세 노드가 모두 활성 상태가 되는 최초 시점은 "B12 + B_HOLO 해금 시점" = 정보층(dec25) 진입 후 도감 80% 달성. 단일런 기준 최소 50.76h.

**도감 만렙(×1.35)은 더욱 후기다.** 87입자 전부 수집은 플랑크온(L11 전용 LEGENDARY) 해금을 포함하므로 dec26 도달 직전~도달 시점. 단일런 62.66h 부근.

**따라서 worst-case "A13 + B8 + B12 + B_HOLO + 도감 ×1.35" 5중 동시 곱은:**
- dec25 진입 후 도감 80% 달성 + B12 해금 시점 이후에만 가능
- 단일런 mult=1 기준 50h 이상 진행된 후반부
- **dec26 도달 시점(62.66h)에서의 잔여 진행 거리가 짧은 만큼** 실제 race 영향은 이미 소폭

#### 타이밍 분리로 예산 재해석 (핵심 논거)

§3.4의 "×1.43 상한"은 도감 ×1.35가 **런 전체에 걸쳐 상수배율로 곱해진다고 가정**한 계산이다(economy §7.1의 codex_race.py 전제). 그러나:

1. **도감 보너스는 램프업한다.** 완성도 0%에서 시작해 입자 발견에 따라 선형(또는 비선형)으로 증가. 런 시작 시 0, 중반 도달 시점 부근에야 만렙.
2. **A13·B12·B_HOLO는 dec25+ 이후에만 활성.** dec26까지 남은 단일런 진행 시간은 약 11.9h(플랑크층 체류).
3. **B8은 조건부 이벤트 배율.** 세 메커니즘 동시 활성 기간 동안만 적용. 평균 활성율이 상수배율보다 낮다.

**결론:** 5중 동시곱의 "실제 런타임 단축" 영향은 worst-case(모두 런 시작부터 상수배율로 가정)의 절반 이하다. **dec26 종점 시각의 절대 단축보다 "dec25~26 구간의 국소 단축"이 관련 지표**이며, 이 구간은 이미 전체 런의 마지막 19%다.

**이것이 ×1.06 헤드룸 재해석의 핵심 논거:** worst-case 동시곱이 실제 플레이에서 동시 달성 불가능(도감 만렙은 dec26 직전, A13은 dec19+, B12는 dec25+)이라는 점에서, §3.4의 "×1.43 총 상한"을 **A13+B12+B_HOLO+B8의 "후기 국소 추가곱" 상한으로 재해석**하고 각 노드별 효과를 이 헤드룸 내에서 배분한다.

단, economy-designer가 python sim으로 "도달 가능 최악 시나리오(도감 50% 완성도 상태에서 A13+B12+B_HOLO 동시 활성)" 기준으로 실제 dec26 단축률을 검증해야 한다.

### 1-3. 노드별 effect_value 후보 표

아래 표의 설계 원칙:
- A13·B12: 이름값에 걸맞은 체감을 주되 production_mult 상수배율이 아닌 방식으로 설계
- B8: 조건부 이벤트 배율이므로 상한 계산에서 가중 평균 활성율로 환산
- B_HOLO: 도감 입자 수에 비례하는 누적형 배율 → 만렙 시 상한 ×1.06 예산 안에서 기여

#### 권장안 A — "국소 배분" (effect_value가 research_mult에 상수곱이 아닌 경우)

| 노드 | 효과 종류 | effect_value 후보 | research_mult 상수곱 환산(상한) | 비고 |
|---|---|---|---|---|
| **A13** 전체 티어 연쇄 | 연속 강화 보너스(Tk 보유 수 × 계수 → Tk+1 배율) | 각 티어 계수 = **+0.008/개** (T1 50개 보유 시 T2에 +0.4× 추가) | 간접 체인 강화, production_mult 직접 곱 없음 | 체인 내 배율 증폭으로 설계 → §3.4 상수곱 회피 가능 |
| **B8** 3중 메커니즘 시너지 | 조건부 이벤트 배율 (세 메커니즘 동시 활성 시 체인 ×[조건부 배율]) | 동시 활성 시 체인 전체 **×1.18** | 평균 활성율 40% 가정 → 시간 평균 ≈ ×1.07 | 조건부 배율이므로 상수곱보다 실효 낮음 |
| **B12** 완전 시너지 | B 가지 시너지 효과 추가 배율 | B 가지 각 시너지 효과 배율 × **+15%** (B1~B11 효과 증폭) | 시너지 효과가 production_mult 직접 곱이 아닌 경우 간접 배율 | 이름값에 걸맞게 "B 가지 전체 강화" 체감 제공 |
| **B_HOLO** 홀로그래픽 증폭 | 도감 입자 수 비례 B 가지 시너지 추가 배율 | codex_count × **0.001** (87개 만렙 시 +0.087 = +8.7%) | 만렙 기준 ×1.087 직접 기여 | 87개 만렙 시 8.7% → 도감 완성 직전~플랑크층에서만 풀 효과 |

**A안 총 예산 점검:**
- B_HOLO 만렙 시 직접 research_mult 기여: ×1.087
- B8 시간 평균 기여: ×1.07(조건부 → 가중)
- A13·B12는 production_mult 직접 상수곱 아님 → 예산 외 처리
- **실질 추가곱 ≈ ×1.087 × 1.07 ≈ ×1.163**

이는 ×1.06 예산을 초과한다. 도달 타이밍(dec25~26에만 B_HOLO·B12 활성)을 감안하면 전체 dec26 단축률에 대한 기여는 dec25 이후 국소 구간에 한정되나, economy sim에서 이를 명시적으로 검증해야 한다.

#### 권장안 B — "엄격 예산" (×1.06을 4개 노드가 나눔, 도달 가능 타이밍 무시)

| 노드 | effect_value 후보 (research_mult 기여) | 예산 배분 |
|---|---|---|
| A13 | 연속 강화 계수 +0.003/개 (간접 체인 → research_mult 직접 0 처리) | 예산 외(체인 내부 배율) |
| B8 | 동시 활성 시 체인 ×1.05(이벤트, 조건부) | 시간 평균 ×1.02 |
| B12 | B 가지 효과 +8% 추가 | ×1.02 |
| B_HOLO | codex_count × 0.0005(87개 만렙 시 +4.35%) | ×1.0435 |
| **합산** | | ×1.02 × 1.02 × 1.0435 ≈ **×1.087** |

B안도 ×1.06 초과이지만 도달 타이밍 보정 전 수치. 타이밍 가중(dec25~26 국소) 적용 시 전체 dec26 단축 기여는 §3.4 범위 내로 떨어질 가능성이 높다.

#### 권장안 C — "구조 분리" (가장 권장, 시스템 관점)

A13과 B12의 "이름값" 체감을 유지하면서 §3.4를 지키는 방법: **이 두 노드를 production_mult 상수 곱셈이 아니라 체인·시너지 내부 증폭으로 설계**하고, research_mult에 직접 기여하는 노드를 B_HOLO + B8(조건부)만으로 제한한다.

- **A13**: 연속 강화 계수를 높여 체인 자체 생산량을 늘림. production_mult 외부에서 곱해지는 상수배율 아님.
  - 설계: Tk 보유 수 × 계수 → Tk+1 생산 추가 보너스. 계수 = **+0.01/개**. T1 30개 보유 시 T2에 +0.30× 추가 생산.
  - research_mult 직접 기여: 0. production_mult 간접 효과는 경로가 다름(체인 수량 → 생산량 증가 → C 누적 → 더 빠른 dec 도달).
- **B12**: B 가지 시너지의 "기간 합산 효과" 강화. 시너지 이벤트 발생 시 D, QF 추가 트리클 증가로 표현.
  - 설계: B 가지 시너지 발동마다 D + [B12_D_bonus], QF + [B12_QF_bonus]. production_mult 상수곱 아님.
- **B_HOLO**: research_mult에 직접 기여하는 유일한 후기 노드.
  - 설계: holographic_mult += codex_count × 0.0012 (87개 만렙 시 +10.4%, 즉 홀로그래픽 계층에 +0.104× 추가).
  - 단, holographic_mult 전체(도감 완성도 보너스 ×1.35 포함)가 economy §7.1 상한을 공유한다면, B_HOLO 기여는 ×1.35 내에서 할당해야 한다.
- **B8**: 조건부 체인 배율. research_mult 상수곱 아님.
  - 설계: 세 메커니즘 동시 활성 시 dC/dt × 1.20 (체인 생산량 직접 증폭). 이 구간 연속 지속 5초 초과 시 D + 0.5(연구 가속 부산물).

**C안 §3.4 적합성:**
- research_mult 직접 상수곱 노드: B_HOLO만 (도감 완성도 보너스 ×1.35 풀에 내포)
- production_mult = QF 부스트 × holographic_mult(≤×1.35, B_HOLO 포함) × research_mult
- A13·B8·B12는 production_mult 외부에서 체인 수량·생산량 증폭 → §3.4 상수곱 상한 계산 범위 외
- §3.4 규칙 충족: holographic_mult ≤ ×1.35 유지 시 −25.9% 내(기존 도감 상한 그대로).

**C안이 "×1.06 헤드룸 빈약" 문제를 해결하는 방식:** A13·B12를 production_mult 외부로 분리하면 이름값 체감을 유지하면서 §3.4를 준수할 수 있다. "완전 시너지"라는 체감은 B 가지 이벤트의 D·QF 증폭 + B_HOLO 누적 배율의 조합으로 제공한다.

### 1-4. 힉스 보너스 처리

요청 문서에는 "힉스 보너스(전 티어 배율)"가 언급되어 있으나 research-tree.md에는 힉스 보너스라는 이름의 노드가 명시되어 있지 않다. 현재 research-tree.md에서 "전 티어 배율"에 해당하는 가장 근접한 노드는 A12(체인 공명, 전 티어 짧은 공명 배율)다. "힉스 보너스"는 미래 추가 예정 노드로 처리한다.

힉스 보너스가 production_mult 상수곱으로 설계될 경우: 기존 ×1.06 예산을 이 노드와 나눠야 하므로 노드당 효과는 더 작아진다. 구조 분리(C안) 방식으로 체인 내부 배율 또는 조건부 배율로 설계할 것을 권장한다.

## 산출물 2 — D(발견 데이터) 보존율

### 2-1. 설계 공간 정의

D 보존율은 빅 크런치 재하강 시 D_current_run 중 다음 런으로 이월되는 비율이다(systems §5-2 리셋 매트릭스). D_lifetime은 별도로 완전 보존된다(§2-I holographic_mult 입력).

D의 역할:
- 연구 트리 노드 구매 화폐
- 홀로그래픽 배율(D_total = D_current + D_lifetime × preservation_rate) 입력
- 재하강 시 재투자 압력의 손잡이

| 보존율 | 체감 효과 | 위험 |
|---|---|---|
| 0% | 매 런 연구 완전 재투자 | 2회차+ 권태 (이미 구매한 노드 재구매 반복) |
| 25% | 연구 비용의 75% 재투자 압박 | 연구 속도 느림, 재하강 인센티브 약함 |
| 50% | 절반 이월, 절반 재투자 | 균형 후보 |
| 75% | 연구 대부분 이월, 소량 재투자 | 재투자 압박 거의 없음 → 권태(빠른 반복) |
| 100% | 재투자 없음 | trivial, 연구 트리 후기 의미 소멸 |

### 2-2. 보존율 권장값 결정 논거

**권장 기본값: 50%**

근거:
1. **"재하강마다 약간의 재투자 + 진행감 유지"** (economy §7.3 잠정 권장과 일치). 50%는 재투자 비용이 줄되 "D를 다시 벌어야 하는" 동기를 유지한다.
2. **연구 트리 52노드의 누적 비용 구조에서 50% 이월의 의미:** 1회차에서 전체 노드를 구매한 경우, 2회차 재투자 D는 전체 비용의 50%다. 후기 노드(B12, D_I2 등)는 비용이 크므로 재구매 압박이 실제로 체감된다. 단, C 가지(자동화) 노드처럼 방치 효율 관련 노드는 재하강 초기에 다시 구매하는 것이 루틴이 되어 "재투자 행동"에 패턴감이 생긴다.
3. **D_lifetime 완전 보존과 분리:** D_lifetime은 holographic_mult에 입력되므로 재하강 후에도 홀로그래픽 배율 기여는 유지된다. 이 구조 하에서 D_current 50% 보존은 "연구 재투자"와 "배율 유지"를 분리하는 깔끔한 설계다.
4. **재하강 차별화(산출물 3)의 손잡이 연결:** 회차별로 보존율을 변주하면 재하강 차별화의 레버로 활용된다(아래 산출물 3 참조).

**0% 거부 이유:** 52노드 전체를 매 런 재구매하면 1~3회차 재투자 시간이 단조롭다. 특히 C 가지(자동화)를 매번 처음부터 재구매하면 "이미 배운 것을 다시 배우는" 체감이 강해진다.

**100% 거부 이유:** 연구 트리가 첫 런에서 완성된 후 재하강에서 D를 벌 의미가 없다. D 생산원(입자 발견, 공명 이벤트)이 자원으로서 무의미해진다.

### 2-3. 보존율 세분 설계 (기본 + 회차 변주)

기본 원칙을 단일값으로 두기보다 **노드 카테고리별 차등 보존**이 더 세밀한 설계다.

| D 보존 카테고리 | 권장 보존율 | 근거 |
|---|---|---|
| A 가지(체인 증폭) 구매 이력 → D 환급 | 60% | 체인 업그레이드는 재하강 후 빠른 가속에 필요, 재진입 장벽 낮춤 |
| B 가지(시너지 엔진) 구매 이력 → D 환급 | 40% | 시너지는 층 진입 후 새로운 메커니즘과 연동, 재탐색 동기 |
| C 가지(자동화 확장) 구매 이력 → D 환급 | 70% | 자동화는 "이미 잘 쓰던 것" — 재구매 반복이 가장 지루한 영역 |
| D 가지(위상 심화) 구매 이력 → D 환급 | 30% | 층별 심화 노드는 해당 층 재진입 시 재발견 체감 강화 |
| **단순 단일값 권장 (economy 시뮬용)** | **50% 단일** | 위 차등 없이 단일 보존율 사용 시 50% 권장 |

**economy 재시뮬 파라미터:**
```
D_preservation_rate = 0.50 (기본 시뮬)

시뮬 검증 항목:
  1. 2회차 런 시작 시 이월 D로 즉시 구매 가능한 노드 수 (목표: 체인 가속 노드 일부 즉시 재구매 가능)
  2. 2회차 런에서 D 생산으로 전체 노드 재구매까지 걸리는 시간 (목표: 1회차의 50~70% 이내)
  3. holographic_mult에 대한 D_total 기여:
     D_total = D_2nd_run_current + D_lifetime × 1.0 (D_lifetime은 100% 보존)
     → D_lifetime이 크면 D_current 보존율이 holographic_mult에 미치는 영향은 상대적으로 작음

민감도 체크:
  - preservation_rate = 0.30: 재투자 압박 강, 2회차 초반 진행 느림 → 권태 위험
  - preservation_rate = 0.50: 균형 (권장)
  - preservation_rate = 0.70: 재투자 부담 경, 2회차 초반 빠름 → 재하강 차별화 약화
```

**필러 정합 (산출물 2):** "방치도 보상, 개입도 보상"(필러 ③) — D 50% 보존은 방치 플레이어가 D_lifetime 홀로그래픽 배율을 유지하면서 재하강하고, 개입 플레이어가 연구 재투자 선택에서 전략을 발휘하게 한다. 산출물 3의 재하강 차별화 레버와 연동 시 회차별 보존율 변주로 "방치 vs 개입" 페이싱을 조절할 수 있다.

## 산출물 3 — 재하강 회차 차별화 메커니즘

### 3-1. 문제 정의

현재 economy §5.3 가정: 재하강 2회차+ 비용 불변 → 매 회차 ~5.77일 동일 사이클. director-review §7.1(후속-A1)이 이를 재하강 루프 구현 전 선결 과제로 격상했다.

목표: 2회차가 1회차의 단순 가속 복사본이 아니라 "다른 결의 회차"로 체감되게.
필러 ④("한 층=한 새로움")가 재하강 루프 안에서도 살아있어야 한다.

### 3-2. 레버 설계 — "서브층 심화 선택" (핵심 레버)

**레버 (i): 회차별 서브층 심화 해금 (추천 핵심 레버)**

메커니즘: 빅 크런치 리셋 후 재하강을 시작할 때, 6개 미지 서브층 중 1개를 "이번 런의 집중 서브층"으로 선택한다. 선택된 서브층에서만 활성화되는 추가 해금 콘텐츠(심화 입자·연구 노드·배율)가 존재하며, 다른 서브층은 빠르게 통과한다.

```
빅 크런치 후 재하강 시작 시 UI 표시:
  "이번 하강에서 집중할 서브층을 선택하세요."
  [프리온층] [끈층] [루프층] [거품층] [정보층] [플랑크층]
  (이미 이전 런에서 선택한 서브층은 잠금 해제 아이콘 표시)

선택된 집중 서브층에서:
  - 해당 서브층 체류 시간 +30~50% (비용 저항 완화 X, 해금 콘텐츠 추가)
  - 집중 서브층 전용 "심화 입자" 2~3개 해금 (도감 추가 항목)
  - 해당 서브층 메커니즘 연구 노드 "심화 가지" 1~2개 해금 (D 가지 확장)
  - 해당 서브층 클리어 시 "집중 보너스 QF" 추가 획득

선택되지 않은 서브층: QF 부스트로 빠르게 통과 (기존 재하강 속도 유지)
```

**이것이 "다른 결"을 만드는 이유:**
- 1회차: 6 서브층을 처음 발견, 각 메커니즘 학습 (발견의 재미)
- 2회차: "프리온층" 집중 선택 → 프리온층 전용 심화 입자 + D 가지 확장 노드 (탐구의 재미)
- 3회차: "끈층" 집중 선택 → 끈층 전용 콘텐츠 (최적화의 재미)
- 7회차+: 6개 서브층 모두 집중 완료 후 "자유 선택" 모드 → 가장 좋아하는 서브층 반복 또는 신규 조합

서브층 심화 해금이 축적되면 도감 완성도가 올라가고 → holographic_mult 램프업 → 재하강 깊어질수록 배율 증가의 연결 고리가 생긴다.

**레버 (ii): D 보존율 회차 변주**

| 회차 | D 보존율 | 의도 |
|---|---|---|
| 1회차 | — (첫 런) | — |
| 2회차 | **65%** | 재하강 첫 경험, 연구 재투자 부담 완화 (진입 장벽 낮춤) |
| 3회차 | **50%** | 기본 권장값, "재투자 + 진행" 균형 |
| 4~5회차 | **40%** | 재투자 압박 강화, 선택 비중 증가 |
| 6회차+ | **35~40%** | 안정 구간, 최소 진입 장벽 유지 |

근거: 2회차는 재하강 개념 자체가 새로우므로 진입 장벽을 낮추는 것이 유리하다. 이후 회차에서 점진적으로 재투자 압박을 높여 "연구 트리 재선택"이라는 전략적 결정이 생긴다.

**레버 (iii): 2회차+ 비용 조정 곡선 (보조 레버)**

회차별 비용 불변 대신 경미한 비용 감소를 적용하여 "재하강이 약간 빠르다"는 가속 체감을 준다. 단, 페이싱 캘린더에 영향을 주므로 economy 시뮬 검증이 필요하다.

```
cost_multiplier(n) = 1 / (1 + 0.05 × (run_index - 1))
  run_index = 1: 비용 1.0 (기준)
  run_index = 2: 비용 × 0.952 (약 5% 감소)
  run_index = 3: 비용 × 0.909 (약 9% 감소)
  run_index = 5: 비용 × 0.833 (약 17% 감소, 하한)
  run_index = 6+: 비용 하한 = × 0.80 (20% 감소 고정)
```

이 비용 조정은 QF 부스트(mult 2.76×)와 겹치므로 경미한 추가 가속에 그쳐야 한다. economy 시뮬에서 "비용 감소 + QF 부스트 합산 후 캘린더" 재검증 필요.

```
  생산 mult = 1 + 0.25 × log10(1 + QF_run2) ≈ 2.759×

  → 기대: 1회차 5.77일 대비 5.2~5.5일 예상

시뮬 2 — D 보존율 65% 효과:
  D_start_run2 = 0.65 × D_end_run1
  → run2 시작 즉시 구매 가능한 연구 노드 수
  → run2에서 모든 노드 재구매까지 걸리는 시간
  → 홀로그래픽 배율(D_total = D_run2_current + D_lifetime) 계산

시뮬 3 — 집중 서브층 체류 효과:
  집중 서브층 진입 시 해당 벽 통과 C 임계값을 +10% 상향 (심화 콘텐츠 체류 확장)
  비집중 서브층은 변화 없음
  → 집중 서브층 체류 시간: 기본 대비 +15~20% 추정 (심화 콘텐츠 소화 시간 포함)

검증 목표:
  - 2회차 캘린더 5.0~5.6일 (1회차 대비 5~15% 단축, 가속 체감 O)
  - 집중 서브층 체류 기본 4h × 1.15 = 4.6h (체류 ≥4h 유지)
  - 회차별 캘린더: 2회(5.4) → 3회(5.2) → 4~5회(4.9) → 6회+(4.7~4.8일) 수렴
  - 3회차 누적: ~16일 (3회×평균 5.3일 ≈ 2.3주) → "수 주" 충족 유지

경계 조건:
  - 비용 감소 + QF 부스트 합산 시 dec26 도달이 지나치게 빠르지 않을 것 (1회차의 75% 이상 유지)
  - 집중 서브층 체류 ≥ 4h (FAIL-1 기준 유지)
  - race 단조성: 각 회차 내 상전이 시퀀스 단조 증가 유지
```

**필러 정합 (산출물 3):** "한 층=한 새로움"(필러 ④) — 집중 서브층 선택이 매 회차마다 "이번엔 어느 서브층을 탐구하는가"라는 새로운 결정을 만든다. 1회차가 "발견", 2~6회차가 "탐구", 7회차+가 "최적화"라는 3단계 재하강 아치를 형성하며, 각 회차의 집중 서브층이 다르므로 필러 ④가 재하강 루프 내에서도 구조적으로 강제된다.

## 핵심 권장값 요약

**헤드룸 해소 권장안:** A13·B8·B12를 production_mult 직접 상수곱이 아닌 체인 내부 배율·조건부 이벤트·시너지 트리클로 분리(C안). B_HOLO만 holographic_mult에 직접 기여하되 ×1.35 총 풀 안에서 도감 완성도 보너스와 합산 관리. 이렇게 하면 ×1.06이라는 수치 제약 없이 각 노드가 이름값 체감을 제공하면서 §3.4를 구조적으로 준수한다.

**D 보존율 권장값:** 50% 단일 기본값 + 2회차 65%/3회차 50%/4~5회차 40%/6회차+ 35~40%의 회차별 점감 곡선. D_lifetime은 항상 100% 보존.

**재하강 차별화 핵심 레버:** 집중 서브층 선택(회차당 1개 지정, 심화 입자·D 가지 확장 노드 해금)을 1순위, D 보존율 회차 점감 곡선을 2순위, 비용 경미 감소(최대 −20%)를 3순위로 조합. "발견→탐구→최적화"의 3단계 재하강 아치 구조.
