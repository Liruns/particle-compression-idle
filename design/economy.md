## 1. 상전이(Phase Transition) 보상 수식 ★

### 1.1 확정 수식

```
영구통화 (양자 거품):
    QF_total = floor( K · (lifetime_C / D_norm)^e )
    QF_gain  = QF_total − QF_claimed        (이미 받은 양 차감)
    K = 1,  D_norm = 1e26,  e = 0.5 (제곱근)
    lifetime_C = 모든 런 누적 압축깊이 C (AdCap식 lifetime)

영구 생산 부스트 (정준형 — QF가 힘이 되게 / 필러 "작아짐=강해짐"):
    production_mult = 1 + base_rate · log₁₀(1 + QF_total)
    base_rate = 0.25

영구 생산 부스트 (연구 스펙 단순형 — 위와 근사 등가, GDD 기재용):
    production_mult ≈ 1 + N_prestige · 0.194    (N = 상전이 횟수)
```

- **e=0.5 (제곱근) 채택 이유**: 연구 §1-C의 검증된 AdCap 형(`p=150·√(c_L/1e15)`)과 동형. 세제곱근(e=0.33)은 엔드게임 단일 런이 4.78일로 과도하게 길어짐(§1.4 비교) → 기각.
- **QF가 부스트를 직접 구동**: 연구 §2의 리터럴 스펙 `1+N·base_rate`는 QF를 코스메틱으로 만든다(누적 20억 QF가 무의미). log 형으로 QF→부스트 직결시켜 필러 충족 + 자기감쇠(후반 폭주 방지). 단순 카운트형도 거의 같은 곡선(아래 등가성).

### 1.3 Race 분석 — 부스트 vs 목표 인플레 ★ (qa CONCERN-2 정정 반영)

- **인플레 힘**: 벽 간격이 +1.5~2 decade로 넓어짐 = `C ×10^(2/0.65) ≈ ×1200`/벽 수준(구 11벽의 ×34.6/벽보다 큰 점프). 한 상전이 클리어에 필요한 C가 매번 크게 증가.
- **부스트 힘**: mult 증가분이 런 시간을 단축하지만, log 감쇠 + 선형 누적이라 인플레를 **추월하지 못함** → 런 시간 단조 증가.
- **단조성 — 정정**: 구 11벽에서는 L1→L2가 4.26h→4.19h로 **-1.6% 예외**가 있었다(부스트 1.406×가 +1 decade 인플레를 소폭 상쇄, qa CONCERN-2). **v0.2 6벽에서는 벽 점프가 +2.5 decade(dec19→21.5)로 더 커서 이 예외가 사라지고 PT1부터 완전 단조 증가**(4.26→7.18→9.60→15.36→20.60→23.60h). 즉 재스케줄이 CONCERN-2를 구조적으로 해소.
  - **표기 규칙**: (구 11벽 기준) "L1→L2 -1.6% 예외, 이후 완전 단조". (v0.2 6벽) "PT1부터 완전 단조, 예외 없음".
- **base_rate 안전 범위표 (명문화)**:

  | base_rate | L1→L2 거동 | 판정 |
  |---|---|---|
  | **0.25 (채택)** | 6벽: 완전 단조 (예외 0). 11벽: -1.6% 미미 | **안전** |

### 2.1 확정 파라미터

### 2.3 마일스톤 배율 — **벽 파괴 발견 (qa FAIL-2 정정 반영)** ★

연구 §2/§1-B: 25/50/100/200 보유 → 누적 ×2/×3/×5/×7. **우리 체인에서 이건 페이싱을 파괴함.**

**정정**: economy v0.1은 "약함 -57%"를 표기하며 "연구스펙이 가장 문제"라 서술해 수치 매핑이 어긋났다(qa FAIL-2). 독립 재현(qa) + 본 재시뮬이 일치하는 정확한 표:

| 세트 | 누적 배율(@25/50/100/200) | dec19 | dec26 | 기준 대비 | 판정 |
|---|---|---|---|---|---|
| 없음 (기준) | — | 4.26h | **2.611d** | — | 벽 정상 |
| **연구스펙** | **×2/×3/×5/×7** | 2.35h | **0.741d** | **−72%** | **붕괴** |
| 약함 | ×1.5/×2/×3 | 2.86h | 1.127d | −57% | 파괴 |
| 매우 약함 | ×1.25@25, ×1.5@100 | 3.66h | 1.935d | −26% | 한계치 |

- **이유**: 마일스톤 배율이 production_mult로 체인 전체에 곱 전파 → 후반 200+개 보유 시 누적 ×7이 `t^n/n!`를 폭발시켜 dec26 벽이 −72% 붕괴. 상전이 race 무력화(벽이 사라지면 상전이 타이밍 무의미). 마일스톤 효과는 **벽 위치와 무관**(생산 배율이라 곡선 전체를 스케일) → dec26 캡에서도 −72% 동일.
- **한계치 명문화 (디렉터 §3.3)**: `dec26 감소 < 30%` 조건에서 역산하면 **×1.25@25, ×1.5@100이 상한**(−26%). 그 이상은 금지.
- **확정 (디렉터 §3.3 / qa 수정2)**: **마일스톤은 플레이버 한정 ×1.1 이하** (또는 도감/연구 보너스로 대체). 강한 마일스톤 금지. 연구스펙 ×2/×3/×5/×7은 GDD §13 백로그에 "−72% 파괴, 사용 금지"로 명기.
- **최적 구매 티어 전환은 마일스톤 없이도 달성**(§2.4) → 연구가 마일스톤에 기대한 "시점마다 최적 티어 변화" 목적은 base 구조만으로 충족(기능 손실 없음).

### 2.4 최적 구매 티어 전환 (시뮬 실증)

`base_k=10^(1+1.3k)` + growth 내림차순의 구조만으로 최적 구매 티어가 자연 이동:

| 진행 | 다음 1개 최저가 티어 | 의미 |
|---|---|---|
| 초반(~5개 보유) | **T1** | 낮은 base가 지배 |
| 중반(~30개) | T1 (격차 축소) | 전이 시작 |
| 후반(~90개) | **T8** | growth^90이 고-base 티어를 상대적으로 싸게 만듦 + 고티어가 전 체인 견인 |

→ 연구 §2 "어느 티어를 살지 시점마다 바뀌게" **마일스톤 불필요로 달성**. 8단 전 티어가 게임 전체에서 의미 유지.

## 3. 오프라인 공식

### 3.1 확정 수식

```
유효_오프라인_초 = min(elapsed, MAX_DELTA) 에서 출발:
    elapsed_clamped = min(now − last_save, CAP · TAMPER_MULT)   ← 시계조작 방어
    capped          = min(elapsed_clamped, CAP)
    long_idle_bonus = 1 + LB · log₁₀(1 + max(0, elapsed_clamped − CAP)/CAP)
    유효_온라인_초  = capped · modifier · long_idle_bonus

파라미터:
    CAP = 24h            (하드코어 옵션 48h)
    modifier = 0.65      (온라인의 65%)
    상전이 직후 첫 런: modifier = 1.0   (복귀 임팩트)
    LB = 0.5             (장시간 방치 로그 보너스 계수)
    TAMPER_MULT = 2.0    (elapsed 상한 = 48h)
```

- **부여 방식**: 접속 끊긴 시점의 생산 rate × 유효시간 분량을 E·C에 일괄 지급. 생산이 C에 선형이므로 "rate×Δt×mod 일괄지급"은 "frozen rate로 mod·Δt만큼 온라인 진행"과 등가(보수적 하한 — 온라인은 복리로 더 빠르므로).

### 3.2 시뮬 실증 — 오프라인 크레딧

| 자리비움 | 유효 온라인 | 비율 | 비고 |
|---|---|---|---|
| 1h | 0.65h | 65% | 캡 이내 선형 |
| 8h | 5.20h | 65% | 하룻밤 |
| 12h | 7.80h | 65% | |
| 24h | 15.60h | 65% | 캡 도달 |
| 36h | 16.97h | 47% | 로그보너스 트리클(평주말) |
| 48h | 17.95h | 37% | **탬퍼 클램프 도달** |
| 72h | 17.95h | 25% | 클램프(추가 없음) |
| 168h(1주) | 17.95h | 11% | 클램프 |

- 24~48h 구간만 로그 보너스가 약하게 보상(주말 방치 배려), 48h에서 하드 클램프 → 무한 방치 익스플로잇 차단.
- **상전이 직후 modifier=1.0**: 8h 자리비움 시 5.2h→8.0h(+54%), 24h 시 15.6h→24.0h(+54%). 새 층 복귀를 강하게 보상 = 연구 §2 권장.

### 3.3 시계 조작 방어

- `elapsed = min(now − last_save, 48h)` 하드 상한. 클라이언트 시계를 +1년 돌려도 최대 48h(유효 18h)만 지급.
- Steam 빌드: 가능시 서버/Steam 타임스탬프 교차검증. 데스크톱(Tauri) 환경에선 last_save를 암호화/체크섬해 변조 탐지. (tech-architect 영역 — 본 문서는 수식만 확정.)

### 4.2 통합 캘린더 결과 — 단일 캠페인 (6벽, 첫 빅 크런치까지)

#### 7.2.3 holographic_mult 캡 아키텍처 정밀 정의 (오케스트레이터 핵심 요청) ★

> codex.md §13-3 / systems.md §2-I 공식:
> ```
> holographic_mult = [1 + log₁₀(D_total+1)·holo_factor]   ← D항 (홀로그래픽 인코딩 L10 메커니즘)
>                  + min(codex_completion²·0.35, 0.35)     ← codex항 (곡선 B, 캡 +0.35×)
> ```

- **캡이 걸리는 위치 (정밀)**:
  - **(1) codex항 캡 = +0.35×는 codex 서브풀에만 적용** (codex.md §13-3 `min(…, 0.35)`). → "codex 기여분만 ≤+0.35×"가 **YES**.
  - **(2) D항은 별도 서브풀**, 자체 서브예산(holo_factor로 통제). codex 캡과 무관.
  - **(3) §3.4 −30% 규칙은 전체 추가곱 H = (1 + D항 + codex항) × research_mult에 적용**. → "전체 holographic_mult가 race에 들어가는 곱"도 **YES** — 단 D항은 +0.35×의 일부가 아니라 별도 작은 서브예산.
- **§3.4 총예산 (정확)**: dec26 단축 <30% ↔ 전체 추가곱 **H < ×1.4286** (1−1/1.4286 = 30.0%). codex항 최대 ×1.35가 −25.9% 소진 → **D항이 ×1.35 위에 가질 헤드룸 = 1.4286/1.35 = ×1.058 (D항 ≤ +5.8%)**.
- **holo_factor = 0.008 결정 (`holographic_cap_def.py`)**: D는 느린 연구 화폐(발견·공명 트리클, C와 달리 작음). D_total ≲ 1e6 범위에서 D항 = log₁₀(D+1)×0.008:

  | D_total | D항 | (codex 만렙과) 전체 H | dec26 단축 (worst, 상수) |
  |---|---|---|---|
  | 1e4 | +3.2% | ×1.382 | −27.6% |
  | 1e5 | +4.0% | ×1.390 | −28.0% |
  | 1e6 | +4.8% | ×1.398 | −28.5% |
  | 1e7 | +5.6% | ×1.406 | −28.9% |

  - **worst-case(D항+codex항 둘 다 PT1부터 상수, D_total=1e6)에서도 −28.5% < 30% PASS**. holo_factor=0.008은 D_total ≲ 1e7까지 안전(−28.9%). D_total ≫ 1e7로 커지면 재튜닝 필요(§7.7 한계).
  - **reachable(현실) 단축은 훨씬 작음**: D항 메커니즘은 **L10(정보층, dec25+)에서만 활성**(systems §2-I) → dec25 이전엔 D항=0. 곡선 B codex도 램프업. 국소화 sim: D_total=1e6에서 dec26 단축 **−18.7%** (상수 worst −28.5% 대비 큰 여유, `holographic_cap_def.py`).

- **D_lifetime 100% 보존 / D_current 50% 보존 분리**: "배율 유지"(D_lifetime → holographic D항)와 "연구 재투자"(D_current 50% 재투자 압박)를 깔끔히 분리. 방치 플레이어는 D_lifetime 배율 유지하며 재하강, 개입 플레이어는 재투자 전략 발휘(필러 ③).
- **회차 점감 (2회차 65% → 6회차+ 35%)**: 2회차는 재하강 개념 자체가 새로우므로 진입 장벽 낮춤(65%), 이후 재투자 압박 점증으로 "연구 트리 재선택" 전략 결정 발생. 재하강 차별화(§7.5) 레버와 연동.
- **재투자 압박 (재현)**: run2 50%(mid) → run3+ 40~35%(high). D_lifetime이 크면 D_current 보존율이 holographic D항에 미치는 영향은 작음(D_total = D_current + D_lifetime, 후자 지배). → 보존율은 주로 **연구 재구매 캘린더**에 영향, **race/배율엔 미미**.
- **미검증 잔여**: "2회차 즉시 재구매 가능 노드 ≥5개" 기준은 연구트리 노드별 D 비용 확정 후 검증(현재 비용 미정 → systems 후속). 본 패스는 보존율 비율과 재투자 압박 정성 평가만.

### 7.5 재하강 차별화 경제 (신규) ★★

> **director §7.1 후속-A1 = 재하강 루프 구현 전 선결 과제.** systems 산출물 3 파라미터를 재시뮬. 재현: `redescent_diff.py`, `redescent_diag.py`.

#### 7.5.1 핵심 발견 — 비용 감소는 방치 캘린더를 못 움직인다 (negative result)

- **systems §3-5 목표 "2회차 캘린더 < 1회차 ×0.95 (비용 감소로)" 는 방치 플레이어에게 달성 불가**. `redescent_diag.py` 진단:
  - **방치 캘린더는 체크인 케이던스 + 오프라인 캡(24h×0.65=15.6h/일)이 결정** → 비용 감소가 못 넘는 바닥.
  - cost_mult **1.0→0.50** 으로 내려도 방치 캘린더는 **5.77일 고정(×1.000)**. PT6 단독 22h 능동이 항상 3 체크인 갭 필요 → 비용으로 안 줄어듦.
  - 능동 back-to-back 시간조차 cost x1.0→0.80에서 **64.91h→64.47h (−0.7%)** 만 변함 — 체인 `t^n/n!` 성장에서 base 비용 이동은 ~log₁₀(cm)/slope decade만큼만 곡선을 옮기기 때문.
- **캘린더는 능동시간에 계단형(step-like)** (`redescent_diag.py §A`): 능동 12h→1갭(0.59일), 18h→2갭(1.02일), 22h→3갭(1.62일). 22h→20h는 같은 갭 수 → 캘린더 불변.

#### 7.5.2 수정 모델 — 차별화는 정성(집중 서브층) + 방치/개입 분리

- **재하강 차별화 = 주로 정성적**(회차별 집중 서브층 선택 + 심화 입자·노드), **캘린더 압축 메커니즘 아님**.
- **집중 서브층 = 캘린더 EXTENDER**(심화 콘텐츠 +6h 블록, 플레이어가 선택해 소화) → 집중 런이 약간 길어지며 회차마다 다른 형태. **≥4h 능동 게이트가 아니라** 심화 콘텐츠 참여 시간(systems의 "+15% dwell" 재해석).
- **방치 vs 개입 분리 (확정, `redescent_diff.py §2b`)**:

  | 플레이 | 1회차 캘린더 | 2회차 재하강 | 비율 | 출처 |
  |---|---|---|---|---|
  | 방치 (2체크인/일) | 5.77일 | **5.77일** | ×1.00 | 오프라인 바운드 (안정) |
  | 개입 (3체크인/일) | 5.16일 | **4.12일** | **×0.80** | 누적 QF가 재통과 압축 |

  - **개입 플레이어는 누적 QF로 ×0.80 압축** → systems 목표의 정신 충족(가속 체감). **방치 플레이어는 의도적으로 ~5.8일 유지**(오프라인 바운드 = 올바름). 이 분리가 필러 ③(방치·개입 모두 보상)의 정확한 구현.
- **비용 감소 곡선의 실제 역할**: 방치 캘린더가 아니라 **개입 플레이어의 능동 back-to-back 시간**을 약간 줄이고, 개입 경로에서 초반 층 능동시간을 갭 경계 아래로 내리는 보조 레버. systems cost_mult_curve `[1.0,0.952,…,0.80]` 유지하되 **"캘린더 단축 약속"으로 표기하지 말 것**.
