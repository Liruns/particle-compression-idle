## 1. 상전이(Phase Transition) 보상 수식 ★

### 1.1 확정 수식

```
영구통화 (양자 거품):
    QF_total = floor( K · (lifetime_C / D_norm)^e )
    QF_gain  = QF_total − QF_claimed        (이미 받은 양 차감)
    K = 1,  D_norm = 1e26,  e = 0.5 (제곱근)
    lifetime_C = 모든 런 누적 압축깊이 C (AdCap식 lifetime)

영구 생산 부스트 (정준형 — QF가 힘이 되게 / 필러 "작아짐=강해짐"):
    production_mult = 1 + base_rate · log₁₀(1 + QF_total)
    base_rate = 0.25

영구 생산 부스트 (연구 스펙 단순형 — 위와 근사 등가, GDD 기재용):
    production_mult ≈ 1 + N_prestige · 0.194    (N = 상전이 횟수)
```

- **e=0.5 (제곱근) 채택 이유**: 연구 §1-C의 검증된 AdCap 형(`p=150·√(c_L/1e15)`)과 동형. 세제곱근(e=0.33)은 엔드게임 단일 런이 4.78일로 과도하게 길어짐(§1.4 비교) → 기각.
- **QF가 부스트를 직접 구동**: 연구 §2의 리터럴 스펙 `1+N·base_rate`는 QF를 코스메틱으로 만든다(누적 20억 QF가 무의미). log 형으로 QF→부스트 직결시켜 필러 충족 + 자기감쇠(후반 폭주 방지). 단순 카운트형도 거의 같은 곡선(아래 등가성).

### 1.3 Race 분석 — 부스트 vs 목표 인플레 ★ (qa CONCERN-2 정정 반영)

- **인플레 힘**: 벽 간격이 +1.5~2 decade로 넓어짐 = `C ×10^(2/0.65) ≈ ×1200`/벽 수준(구 11벽의 ×34.6/벽보다 큰 점프). 한 상전이 클리어에 필요한 C가 매번 크게 증가.
- **부스트 힘**: mult 증가분이 런 시간을 단축하지만, log 감쇠 + 선형 누적이라 인플레를 **추월하지 못함** → 런 시간 단조 증가.
- **단조성 — 정정**: 구 11벽에서는 L1→L2가 4.26h→4.19h로 **-1.6% 예외**가 있었다(부스트 1.406×가 +1 decade 인플레를 소폭 상쇄, qa CONCERN-2). **v0.2 6벽에서는 벽 점프가 +2.5 decade(dec19→21.5)로 더 커서 이 예외가 사라지고 PT1부터 완전 단조 증가**(4.26→7.18→9.60→15.36→20.60→23.60h). 즉 재스케줄이 CONCERN-2를 구조적으로 해소.
  - **표기 규칙**: (구 11벽 기준) "L1→L2 -1.6% 예외, 이후 완전 단조". (v0.2 6벽) "PT1부터 완전 단조, 예외 없음".
- **base_rate 안전 범위표 (명문화)**:

  | base_rate | L1→L2 거동 | 판정 |
  |---|---|---|
  | **0.25 (채택)** | 6벽: 완전 단조 (예외 0). 11벽: -1.6% 미미 | **안전** |
  | 0.30~0.35 | (7상전이 mult 미달 시 후보였으나 2.76×로 불필요) — 미검증, L1→L2 -5~10% 추정 | 미검증 |
  | 0.5 | 11벽: L1→L2 4.26h→3.25h (-24%), 이후 증가 = race 붕괴 경고 | **붕괴** |

- **추월 방지 확인**: 시뮬에서 런 시간이 모든 상전이에서 증가 → **붕괴 없음**. base_rate=0.25는 안전 구간(상향 불필요, 2.76×가 밴드 내).

## 2. 비용 곡선 튜닝 (8단 체인)

### 2.1 확정 파라미터

```
비용:   cost_k(n) = base_k · growth_k^n
base_k = 10^(1 + 1.3·k)             ← GDD의 10^(1+2k)에서 하향 (온보딩 정상화)
growth_k = 2.2 − (0.4/7)·(k−1)      ← 내림차순: T1=2.2 ... T8=1.8
```

### 2.3 마일스톤 배율 — **벽 파괴 발견 (qa FAIL-2 정정 반영)** ★

연구 §2/§1-B: 25/50/100/200 보유 → 누적 ×2/×3/×5/×7. **우리 체인에서 이건 페이싱을 파괴함.**

**정정**: economy v0.1은 "약함 -57%"를 표기하며 "연구스펙이 가장 문제"라 서술해 수치 매핑이 어긋났다(qa FAIL-2). 독립 재현(qa) + 본 재시뮬이 일치하는 정확한 표:

| 세트 | 누적 배율(@25/50/100/200) | dec19 | dec26 | 기준 대비 | 판정 |
|---|---|---|---|---|---|
| 없음 (기준) | — | 4.26h | **2.611d** | — | 벽 정상 |
| **연구스펙** | **×2/×3/×5/×7** | 2.35h | **0.741d** | **−72%** | **붕괴** |
| 약함 | ×1.5/×2/×3 | 2.86h | 1.127d | −57% | 파괴 |
| 매우 약함 | ×1.25@25, ×1.5@100 | 3.66h | 1.935d | −26% | 한계치 |

- **이유**: 마일스톤 배율이 production_mult로 체인 전체에 곱 전파 → 후반 200+개 보유 시 누적 ×7이 `t^n/n!`를 폭발시켜 dec26 벽이 −72% 붕괴. 상전이 race 무력화(벽이 사라지면 상전이 타이밍 무의미). 마일스톤 효과는 **벽 위치와 무관**(생산 배율이라 곡선 전체를 스케일) → dec26 캡에서도 −72% 동일.
- **한계치 명문화 (디렉터 §3.3)**: `dec26 감소 < 30%` 조건에서 역산하면 **×1.25@25, ×1.5@100이 상한**(−26%). 그 이상은 금지.
- **확정 (디렉터 §3.3 / qa 수정2)**: **마일스톤은 플레이버 한정 ×1.1 이하** (또는 도감/연구 보너스로 대체). 강한 마일스톤 금지. 연구스펙 ×2/×3/×5/×7은 GDD §13 백로그에 "−72% 파괴, 사용 금지"로 명기.
- **최적 구매 티어 전환은 마일스톤 없이도 달성**(§2.4) → 연구가 마일스톤에 기대한 "시점마다 최적 티어 변화" 목적은 base 구조만으로 충족(기능 손실 없음).

### 2.4 최적 구매 티어 전환 (시뮬 실증)

`base_k=10^(1+1.3k)` + growth 내림차순의 구조만으로 최적 구매 티어가 자연 이동:

| 진행 | 다음 1개 최저가 티어 | 의미 |
|---|---|---|
| 초반(~5개 보유) | **T1** | 낮은 base가 지배 |
| 중반(~30개) | T1 (격차 축소) | 전이 시작 |
| 후반(~90개) | **T8** | growth^90이 고-base 티어를 상대적으로 싸게 만듦 + 고티어가 전 체인 견인 |

→ 연구 §2 "어느 티어를 살지 시점마다 바뀌게" **마일스톤 불필요로 달성**. 8단 전 티어가 게임 전체에서 의미 유지.

## 3. 오프라인 공식

### 3.1 확정 수식

```
유효_오프라인_초 = min(elapsed, MAX_DELTA) 에서 출발:
    elapsed_clamped = min(now − last_save, CAP · TAMPER_MULT)   ← 시계조작 방어
    capped          = min(elapsed_clamped, CAP)
    long_idle_bonus = 1 + LB · log₁₀(1 + max(0, elapsed_clamped − CAP)/CAP)
    유효_온라인_초  = capped · modifier · long_idle_bonus

파라미터:
    CAP = 24h            (하드코어 옵션 48h)
    modifier = 0.65      (온라인의 65%)
    상전이 직후 첫 런: modifier = 1.0   (복귀 임팩트)
    LB = 0.5             (장시간 방치 로그 보너스 계수)
    TAMPER_MULT = 2.0    (elapsed 상한 = 48h)
```

- **부여 방식**: 접속 끊긴 시점의 생산 rate × 유효시간 분량을 E·C에 일괄 지급. 생산이 C에 선형이므로 "rate×Δt×mod 일괄지급"은 "frozen rate로 mod·Δt만큼 온라인 진행"과 등가(보수적 하한 — 온라인은 복리로 더 빠르므로).

### 3.2 시뮬 실증 — 오프라인 크레딧

| 자리비움 | 유효 온라인 | 비율 | 비고 |
|---|---|---|---|
| 1h | 0.65h | 65% | 캡 이내 선형 |
| 8h | 5.20h | 65% | 하룻밤 |
| 12h | 7.80h | 65% | |
| 24h | 15.60h | 65% | 캡 도달 |
| 36h | 16.97h | 47% | 로그보너스 트리클(평주말) |
| 48h | 17.95h | 37% | **탬퍼 클램프 도달** |
| 72h | 17.95h | 25% | 클램프(추가 없음) |
| 168h(1주) | 17.95h | 11% | 클램프 |

- 24~48h 구간만 로그 보너스가 약하게 보상(주말 방치 배려), 48h에서 하드 클램프 → 무한 방치 익스플로잇 차단.
- **상전이 직후 modifier=1.0**: 8h 자리비움 시 5.2h→8.0h(+54%), 24h 시 15.6h→24.0h(+54%). 새 층 복귀를 강하게 보상 = 연구 §2 권장.

### 3.3 시계 조작 방어

- `elapsed = min(now − last_save, 48h)` 하드 상한. 클라이언트 시계를 +1년 돌려도 최대 48h(유효 18h)만 지급.
- Steam 빌드: 가능시 서버/Steam 타임스탬프 교차검증. NW.js 환경에선 last_save를 암호화/체크섬해 변조 탐지. (tech-architect 영역 — 본 문서는 수식만 확정.)

## 5. 가정 · 한계 (미검증 항목 명시)

### 5.1 검증됨 (시뮬 실증)
- 상전이 수식·첫 상전이 타이밍(4.26h)·부스트 race 단조성·상전이 7회 민감도(2.76× 밴드).
- **6벽 재스케줄**: 각 미지 서브층 체류 ≥4h (mult=1 및 캠페인 부스트 양쪽) = FAIL-1 해소.
- 단일런 페이싱·최적티어 전환·마일스톤 영향(연구스펙 −72% 재현, qa 일치).
- 오프라인 크레딧 곡선·탬퍼 클램프·통합 캘린더(첫 크런치 0.82주)·**빅 크런치 재하강 멀티위크(3회=2.47주)**.
- 도감 보너스 race 단조성·상한 ×1.35 확정(상수배율 H는 런타임을 정확히 1/H로 균일 스케일 → 단조 구조적 보존, dec26 −25.9%). §7.1, qa CONCERN-NEW-2 해소.

### 5.2 가정 (재검증 권장)
- **구매 휴리스틱 = 고티어 우선 전액 투입**. 실제 플레이어 전략과 다를 수 있음 → 프로토타입에서 실측 필요(skill 경고). 다른 전략이면 페이싱이 ±20% 흔들릴 수 있음.
- **층 리셋 = C·압축기 카운트 완전 리셋, QF·mult 유지** 가정. 부분 리셋(일부 자동화 유지)이면 런 시간 단축 → 벽 재튜닝 필요.
- **lifetime_C 누적이 층 간 이어짐**(AdCap식). since-reset 기준으로 바꾸면 QF_gain 공식 재설계 필요.
- **체크인 모델**(2회/일, 15분)은 합리적 추정. 실제 텔레메트리로 교체 권장.
- **오프라인 등가-시간 근사는 하한**. 온라인 복리를 무시하므로 실제 오프라인 가치는 약간 더 높을 수 있음.

### 5.3 미검증 / 후속
- **연구 트리 경제 = 후속 패스** (§7). 트리 부스트가 mult에 곱해지면 본 race 재튜닝 필요.
- **입자 도감 영구 보너스** 상한 +0.35×(×1.35) 확정(디렉터 §7.2 R2, CONCERN-NEW-2 해소, §7.1). 실제 램프 곡선만 §7 후속 패스(content 확정 후 재시뮬).
- **서브층 수 = 6 확정** (systems C-refined). 상전이 7회 민감도는 §1.5에서 강건 확인(2.76× 밴드 내).
- **break_eternity.js 실수 정밀도**: 시뮬은 native float(dec26에서 C~1.2e40, 안전). 실제 게임은 재하강 누적 lifetime_C가 더 커지므로 BigNumber 필수(GDD §11 확정).
- **재하강 캘린더 차별화**: 현재 2회차+ 비용 불변 가정 → 매 회차 ~5.77일 동일. 디렉터 §2.3 #4 "2회차 비용 조정" 레버로 차별화 가능(후속).

### 7.1 도감 완성도 보너스 — 상한 +0.35×(×1.35) ★ (디렉터 §7.2 R2 확정 / qa CONCERN-NEW-2 해소)

> **2026-06-25 갱신**: 상한을 §3.2 초기값 +0.5×(×1.5)에서 **+0.35×(×1.35)**로 하향. 근거 — qa-report-v2 **CONCERN-NEW-2**: 도감 ×1.5 상수배율 단독으로 dec26 단일런을 **62.65h→41.77h = −33.3% 단축**하여 디렉터 §3.4 "단일런 dec26 벽 30% 이상 단축 금지"를 위반(+3.3%p 초과). 디렉터 §7.2가 ×1.35를 상한으로 확정(×1.25는 보수적 하단). 재시뮬: `sim/codex_race.py`.

- **규칙**: 도감 100% 완성 시 `holographic_mult`에 **최대 ×1.35 기여**(즉 +0.35×). codex_bonus_factor를 이 상한에서 역산 → **codex_bonus_factor = 0.35** (`holographic_mult += codex_completion × codex_bonus_factor`, completion=1.0 시 +0.35×). content-designer codex.md §0·§13-3에 동기화 완료.
- **race 안전성 재검증 (`codex_race.py` — 확정 시뮬)**: production_mult = (1 + 0.25·log₁₀(1+QF)) × holographic_mult(≤1.35) × research_mult.
  - **핵심 수식 사실 (정확)**: 엔진의 모든 생산율이 mult에 선형(dC/dt=g₁·mult, dg_k/dt=g_{k+1}·mult)이므로 **상수 배율 H는 정확한 시간 재스케일**(t'=H·t에서 동일계) → 임의 decade 도달시간이 1/H배, **dec26 단축률 = 1 − 1/H (근사 아님)**. ×1.5→−33.3%, **×1.35→−25.9%**, ×1.25→−20.0% (qa 독립재현·codex_race.py 0% 편차 일치).
  - **단조성은 안전 (구조적)**: 상수배율은 단조 시퀀스를 균일 압축할 뿐 재정렬 불가. 도감 ×1.35를 **극단적으로 PT1부터 곱해도** 캠페인 런 시퀀스 단조 유지: **3.16→5.32→7.11→11.38→15.26→17.48h** (×1.5 기준 2.84→4.78→6.40→10.24→13.74→15.73h, 양쪽 monotonic=True). 마일스톤(보유수 복리)과 달리 race 역전 없음.
  - **상한 ×1.35 채택 이유**: −25.9% < 30% 충족(헤드룸 4.1%p). ×1.25(−20.0%)는 더 안전하나 완주 컬렉터 체감(+25%)이 약함 → 디렉터 §7.2가 상한선을 ×1.35로 두고 content 램프 곡선 확정 후 **×1.25~1.35 내 미세조정** 지정.
  - **밴드는 주의 (유지)**: 빅 크런치 진입 mult 2.76× × 1.35 = **3.73×** → 명목 2~3× 밴드 초과하나, 도감 ×1.35는 **완주 시점에만** 도달하므로 첫 캠페인 race는 영향 없고, 영향 구간은 후기 재하강(거기선 이미 캡). **밴드 초과가 곧 붕괴는 아님**(단조 유지 확인).
  - **worst-case 보수성**: 위 −25.9%는 도감 100% 완성을 **PT1부터 가정**한 상한값. 실제 보너스는 수집에 따라 램프업하므로 실제 단축은 더 작음 → ×1.35 상한은 여유 충족.
  - **미검증 잔여**: 도감 보너스의 *실제 램프 곡선*(완성도별 비선형 증가)은 content 확정 후 재시뮬 필요. 위는 상수 ×1.35 가정의 상한 결과. content 베켄슈타인 홀론(L10, "도감 상한 증가" 노드)이 ×1.35를 넘기지 않도록 §7.2 중첩 상한과 함께 관리.

### 7.2 연구 트리 시너지 중첩 상한 (디렉터 §3.4 규칙)
- **대상 노드**: A13(전체 티어 연쇄), B8(3중 시너지), B12(완전 시너지), 힉스 보너스(전 티어 배율). 이들이 production_mult에 동시 곱.
- **상한 규칙 (디렉터 §3.4)**: 이 4개 + 도감(×1.35, §7.1 확정) + QF 부스트(×2.76)의 **총곱이 단일런 dec26 벽을 30% 이상 단축하면 안 됨**(qa 마일스톤 −30% 기준과 동일 잣대).
  - 환산 (codex_race.py 정밀): dec26 벽 −30% ↔ QF 기본 부스트 **위에 곱해지는** 도감×연구 시너지의 추가 곱 H에서 1−1/H = 0.30 → **H = ×1.43**(상한). **도감 단독 ×1.35(−25.9%)가 이 예산을 거의 소진** → 도감 완주 상태에서 연구 시너지(A13/B8/B12/힉스) 추가 곱은 ×1.43/1.35 ≈ **×1.06**까지만 허용. 초과 시 노드 effect_value 하향(디렉터 §3.4, 노드 삭제 아님).
  - 초과 시 **노드 삭제가 아니라 effect_value 하향**(디렉터 §3.4).
- **후속 시뮬 과제**: A13/B8/B12/힉스 effect_value 후보를 받아 총곱 복리를 시뮬, dec26 벽 단축률 < 30% 확인. base_rate=0.25 단조성 동반 체크.
