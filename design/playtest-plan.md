## 0. 이 문서를 읽는 법 — 시뮬과 실테스트는 서로 다른 레인이다

qa-balancer는 지금까지 **시뮬레이션**으로 밸런스를 검증했다: 수치 페이싱·상전이 타이밍·폭주·진행막힘·오프라인 수학. 이것은 필요하지만 충분하지 않다.

**시뮬이 잡지 못하는 것:**
- "이거 왜 누르는지 모르겠는데?" (온보딩 혼란)
- "지루한데 멈출 타이밍을 못 찾겠어" (UX 지옥)
- "프레스티지했더니 강해진 느낌이 안 와" (체감 보상)
- "첫 상전이 후에 뭘 해야 하는지 모르겠어" (방향 소실)
- "한 판 더 하고 싶다"가 안 오는 이유

이 레인의 목적은 **숫자가 맞는데 재미가 없는** 상황을 탐지하는 것이다. 시뮬과 실테스트는 상충하지 않는다 — **시뮬은 수치 범위를 좁히고, 실테스트는 그 안에서 체감을 확인한다.**

```
시뮬 QA (qa-balancer)          실플레이어 테스트 (본 문서)
─────────────────────         ──────────────────────────
수치·페이싱·수식 검증           재미·혼란·체감 검증
진행막힘·폭주·오버플로 탐지      온보딩 이탈·방향 소실 탐지
결정론적, 재현 가능              정성적, 관찰 기반
모든 정거장 전 가능              프로토타입 실행 가능 시점부터
```

## 1. 무엇을 검증하는가 — 정거장별 핵심 질문

### 정거장 1 (프로토타입) — "코어 루프가 재미있는가?"

졸업 기준(roadmap §1-C)을 **개발자 외 타인**도 통과시키는지 확인.

| 검증 항목 | 핵심 질문 | 시뮬로 가능? |
|---|---|---|
| **작아짐=강해짐 체감** | 첫 30초 안에 r↓가 "강해지는 느낌"과 연결되는가 | 아니오 |
| **능동 클릭 손맛** | 자동화 전 1~2분이 "누를 이유"가 느껴지는가 | 아니오 |
| **첫 상전이 도파민** | QF 획득 후 "해냈다 + 또 하고 싶다"가 오는가 | 부분 |
| **오프라인 기분 좋음** | 복귀 시 진행이 보람차게 느껴지는가 | 아니오 |
| **"한 판 더" 욕구** | 세션 종료 후 다음 세션 의욕이 생기는가 | 아니오 |

> 기준: 졸업 기준 5개 중 3·5번은 **수치**가 아니라 **사람의 반응**으로만 확인 가능.

### 정거장 2 (수직 슬라이스) — "설명서 없이 이해되는가?"

FTUE(첫 경험)가 말 없이 가르치는지 확인.

| 검증 항목 | 핵심 질문 | 시뮬로 가능? |
|---|---|---|
| **온보딩 자력 이해** | 첫 5분을 설명 없이 통과하는가 | 아니오 |
| **압축=자원 직관** | "압축할수록 힘이 생긴다"를 물어보지 않고 파악하는가 | 아니오 |
| **스케일 경이** | 층 진입 줌·VFX에서 "우와" 반응이 오는가 | 아니오 |
| **사운드 피드백** | 클릭·상전이 소리가 기분 좋은가, 지루한가 | 아니오 |
| **스토어 GIF 매력** | GIF를 본 타인이 "해보고 싶다"를 말하는가 | 아니오 |

### 정거장 3 (v1.0) — "끝까지 4필러가 작동하는가?"

전체 캠페인 체험: 온보딩 → 첫 빅 크런치 → 재하강 안착.

| 검증 항목 | 핵심 질문 | 시뮬로 가능? |
|---|---|---|
| **재하강 권태 (PR4)** | 2회차가 1회차와 다르게 느껴지는가 | 부분 |
| **한 층=한 새로움 체감** | 6 미지 메커니즘이 각각 새롭게 느껴지는가 | 아니오 |
| **방치/개입 둘 다 보상** | 안 만져도 진행이 느껴지고, 만지면 보람이 있는가 | 아니오 |
| **빅 크런치 "시작점" 체감** | 빅 크런치를 "끝"이 아닌 "새 시작"으로 받아들이는가 | 아니오 |
| **장기 리텐션 신호** | 첫 빅 크런치 후 재하강 1~2회 후에도 계속할 의사가 있는가 | 아니오 |

## 2. 누구를·언제·어떤 규모로

### 단계 A — 개발자 + 소수 지인 알파 (정거장 1)

**목적:** "코어 루프가 재미있는가"를 개발자 시야 밖에서 첫 확인.

| 항목 | 내용 |
|---|---|
| **시점** | 정거장 1 졸업 기준 내부 통과 직후 |
| **대상** | 개발자 외 2~5명. 인크리멘탈 장르 경험자 1~2 + 비경험자 1~2 혼합 권장 |
| **모집 방법** | 지인·가족·동료 직접 요청. 공개 모집 아님 |
| **플레이 설정** | 개발자가 옆에서 (또는 화상으로) 관찰. 최소 간섭 원칙 |
| **세션 길이** | 30분~1시간 (첫 상전이까지 시간 단축값 사용 권장 — 온보딩~PT1 흐름만 확인) |
| **인원 이유** | 5명이면 온보딩 혼란점의 80%가 드러난다 (Nielsen 법칙). 많을수록 수렴 패턴이 빠름 |

**주의:** 이 단계에서 "못생긴 UI", "소리 없음", "임시 텍스트"는 허용. 단 **코어 루프(압축→작아짐→강해짐)와 첫 상전이는 반드시 작동** 상태여야 함.

---

### 단계 B — 클로즈드 베타 (정거장 2 후반~3 초반)

**목적:** FTUE 검증 + 더 넓은 타입의 플레이어로 체감 신호 수집.

| 항목 | 내용 |
|---|---|
| **시점** | 정거장 2 졸업(수직 슬라이스 완성) 직후. itch.io 빌드 배포 가능 시점 |
| **채널** | ① itch.io 클로즈드 베타 페이지 (key 배포) ② 디스코드 DM (인크리멘탈 장르 커뮤니티 내 관심자) ③ r/incremental_games 소규모 모집 포스트 ("초기 피드백 구합니다") |
| **인원** | 20~50명. 다양성이 중요 — 장르 고수·중수·신입 분산 |
| **플레이 범위** | 온보딩 ~ 첫 상전이 (정거장 2 슬라이스). 전체 게임 미필요 |
| **기간** | 1~2주 열어두고 피드백 수집 |
| **모집 문구 방향** | "실제 물리를 기반으로 물질을 압축해 플랑크 길이까지 내려가는 방치형 게임의 초기 피드백을 구합니다. 인크리멘탈 장르 팬 우선, 설문 5분 포함." |

**주의:** 이 단계에서 얻는 "GIF 공유 의사" 반응은 launch-plan §3 위시리스트 캠페인의 선행 신호가 된다. 베타 참가자 중 관심 높은 사람에게 Steam 위시리스트 유도 가능.

---

### 단계 C — 오픈 베타 / Steam Playtest (정거장 3 완성 후)

**목적:** EA 출시 전 마지막 대규모 검증. 전체 1차 사이클(온보딩→빅 크런치→재하강) 체험.

| 항목 | 내용 |
|---|---|
| **시점** | 정거장 3 졸업 기준 통과 후, EA 출시 1~4주 전 |
| **채널** | Steam Playtest 기능 (위시리스터 대상 선별 초대) + itch.io 공개 데모 + Next Fest 데모 (launch §3 W3) |
| **인원** | 100~500명 (Steam Playtest 규모). 실질 활성 피드백은 50~100명으로 충분 |
| **플레이 범위** | 전체 — 온보딩부터 빅 크런치, 재하강 1~2회 |
| **기간** | 1~2주. 장기 리텐션 관찰은 단계 C에서만 가능 |
| **텔레메트리 필수** | 이 단계부터 analytics-plan 텔레메트리(D_total 수집, 이탈 지점 로그)가 반드시 작동 상태여야 함 |

## 3. 어떻게 수집하는가 — 방법 × 도구

### 3-1. 관찰 (가장 중요 — 단계 A 필수)

목표: "어디서 멈추나, 어디서 헷갈리나, 어디서 지루해하나"를 말 없이 포착.

**방법:**
- 플레이어 옆에 앉거나 화상 공유. 개발자는 **질문하지 않는다** — 반응만 본다.
- 관찰 노트 항목:
  - 첫 클릭까지 몇 초 걸렸나
  - "이게 뭐야?" 또는 멈추는 지점
  - 표정·탄성·한숨 시점
  - 자발적으로 화면 가리키거나 설명하는 순간
  - 세션 종료를 먼저 선언하는 시점과 이유

**녹화(동의 하에):**
- 화면 + 반응 캡처. 편집 없이 보관 → 반복 검토 가능.
- 동의서 필수: "개발 피드백 목적으로만 사용, 비공개" 서면 또는 구두 확인.
- 단계 A: 관찰자 직접 관찰이 우선, 녹화는 선택. 단계 B~C: 화면 녹화 선택 (itch overlay, OBS 가이드 제공).

---

### 3-2. 설문 (단계 B~C 필수, 단계 A 선택)

**세션 후 즉시 진행 (5분 이내):**

| # | 질문 | 유형 | 목적 |
|---|---|---|---|
| Q1 | 지금 이 게임을 친구에게 추천하겠습니까? (0~10점) | NPS | 전체 인상 |
| Q2 | "작아질수록 강해진다"는 게임 핵심이 체감됐나요? (1~5) | 리커트 | 필러 ① 검증 |
| Q3 | 첫 5분 동안 뭘 해야 하는지 자연스럽게 알았나요? (1~5) | 리커트 | FTUE 검증 |
| Q4 | 가장 헷갈리거나 막힌 순간이 있었다면 언제인가요? | 주관식 | 혼란 지점 |
| Q5 | 세션 후 "다시 하고 싶다"는 느낌이 왔나요? (있음/없음) | 이진 | 후킹 검증 |
| Q6 | 오프라인 복귀(닫았다 다시 켰을 때)가 기분 좋았나요? | 이진/주관 | 방치 보상 |
| Q7 | 첫 상전이(프레스티지) 후 어떤 기분이었나요? | 주관식 | 도파민 체감 |
| Q8 | 이 게임의 어떤 부분이 가장 흥미로웠나요? | 주관식 | 강점 파악 |
| Q9 | 이 게임이 계속 나오면 사겠습니까? (단계 B~C) | 이진 | 구매 의사 |

**NPS 해석 기준:**
- 9~10점 추천자 비율 ≥ 40%: 핵심 체험 확인됨.
- 7~8점 수동자 과다: 체험에 걸림돌이 있음 → 질적 피드백 심화 필요.
- 6점 이하 비판자 다수: 코어를 다시 봐야 함. 정거장 1 졸업 조건 미달 신호.


### 3-4. 정성 피드백 결합 원칙

텔레메트리(무엇이 일어났나) + 설문/관찰(왜 그랬나)는 반드시 함께 분석한다.

- **텔레메트리만 보면:** 수치 이탈 지점은 알지만 이유를 모른다.
- **설문만 보면:** 이유는 알지만 얼마나 빈번한지 모른다.
- **결합하면:** "dec5에서 15%가 이탈하고, 그 이유는 '뭘 눌러야 할지 모르겠어서'이다."

## 4. 무엇을 보는가 — 관찰 포인트 체크리스트

### 4-1. 온보딩 이탈

시뮬이 못 잡는 영역 1순위.

- [ ] 처음 화면을 보고 3초 내에 무언가를 누르는가?
- [ ] "압축 버튼"의 목적을 설명 없이 파악하는가?
- [ ] 첫 업그레이드 구매 시 무엇을 사야 할지 아는가?
- [ ] "r이 작아진다"는 것을 알아채는가? 언제?
- [ ] 자동화 해금 후 방치 모드 전환 시점을 자연스럽게 찾는가?

### 4-2. 첫 상전이 도파민

- [ ] 상전이 전 "얼마나 남았나" 긴장감이 쌓이는가?
- [ ] QF 획득 순간 탄성·미소 등 물리 반응이 있는가?
- [ ] 상전이 후 부스트를 즉시 체감하는가?
- [ ] "다음 상전이"를 목표로 설정하는 언급이 나오는가?

## 5. 단계별 졸업 기준 (실플레이어 기준)

각 단계의 시뮬 졸업 기준(roadmap)과 별개로, **실플레이어 기준을 여기에 정의한다.**

### 단계 A 합격 기준

아래 3가지 중 2가지 이상이 테스터 과반에서 확인:
1. 첫 상전이 후 스스로 "다시 하고 싶다" 또는 "다음엔 어떻게 되나?" 발화.
2. "작아지는데 강해진다"를 물어보지 않아도 말함.
3. 오프라인 복귀 시 "오 진행됐네" 긍정 반응.

**불합격 시:** 정거장 2로 가지 않는다. 코어 루프(수식·UX·피드백)를 다시 본다.

### 단계 B 합격 기준

1. 테스터 80% 이상이 첫 5분을 개발자 도움 없이 통과.
2. NPS 7점 이상 비율 ≥ 60%.
3. Q3(FTUE 이해도) 평균 ≥ 3.5/5.
4. "이 GIF 다른 사람한테 보여주고 싶다" 자발 언급 ≥ 3명/10명.

**불합격 시:** FTUE 재설계(UX 재작업). 슬라이스 재폴리시 후 소규모 재테스트.

### 단계 C 합격 기준 (EA 출시 전)

1. 첫 상전이 실측 시간이 시뮬 예측 4~6h의 ±40% 이내 (실측 중앙값 2.4h~8.4h).
2. 첫 빅 크런치 후 이탈률(24h 내) ≤ 50%. (재하강 진입 비율 ≥ 50%)
3. NPS ≥ 30 (추천자 비율 - 비판자 비율).
4. Q5("다시 하고 싶다") 긍정 ≥ 70%.
5. "반복적이다/똑같다" 자발 언급이 2회차 테스터 30% 미만.
6. 세이브 손상 0건.

**불합격 시:** 출시 연기. 실패 항목에 따라 — 페이싱 실패→시뮬 재검증 요청(qa-balancer), FTUE 실패→UX 재작업, 리텐션 실패→재하강 집중 서브층 차별화 강화.

## 6. 운영 절차

### 6-1. 모집 → 플레이 → 수집 흐름

```
[모집]
  → 대상 섭외/공고
  → 동의서 발송 (녹화·피드백 사용 동의)
  → 빌드 링크/key 전달

[플레이]
  → 관찰자 배석 (단계 A: 필수, B~C: 선택)
  → 최소 간섭 원칙 (막히면 최대 30초 대기 후에만 힌트)
  → 관찰 노트 작성

[수집]
  → 설문 즉시 진행 (세션 직후 5분)
  → 자유 코멘트 녹취 (선택)
  → 텔레메트리 로그 자동 수집 (단계 C)

[분석]
  → 텔레메트리 + 설문 결합 분석
  → 시뮬 예측값 vs 실측 편차 계산
  → 졸업 기준 점수 산정
  → 디자인팀 공유 및 액션 항목 도출
```

### 6-2. 피드백 분류 규칙

수집된 피드백을 아래 4유형으로 분류 후 우선순위를 정한다.

| 유형 | 정의 | 액션 |
|---|---|---|
| **블로커** | 졸업 기준 불합격 항목. 다수 플레이어가 같은 지점에서 이탈·포기 | 즉시 수정, 재테스트 |
| **혼란** | 개별적이지만 반복되는 이해 실패 | FTUE·카피 수정 |
| **체감 미스** | "알긴 아는데 재미없다" 피드백 | 수식·리듬·피드백 튜닝 (시뮬 재검증 병행) |
| **취향** | "이건 내 취향이 아니다" 개별 의견 | 참고만, 즉시 반영 안 함 |

## 7. 시뮬 QA vs 실플레이어 테스트 — 완전한 보완 관계

| 구분 | 시뮬 QA (qa-balancer) | 실플레이어 (본 문서) |
|---|---|---|
| **검증 대상** | 수식 정합성, 페이싱 수치, 진행막힘, 폭주, 오버플로 | 재미, 혼란, 체감, 첫인상, 리텐션 욕구 |
| **언제** | 코드 구현 전후, 수식 변경 시 즉시 | 플레이 가능 빌드 존재 시 |
| **출력** | 통과/실패 + 수치 증거 + 수정 방향 | 관찰 노트 + 설문 결과 + 텔레메트리 |
| **한계** | 사람의 체감을 모른다 | 수치 오류를 스스로 찾지 못한다 |
| **상호 입력** | 시뮬로 수치 범위 좁히면 실테스트가 그 범위 안에서 체감 확인 | 실테스트에서 페이싱 편차가 나오면 시뮬 파라미터 재검증 요청 |

**이 두 레인이 모두 통과해야 정거장을 졸업한다.** 수치가 맞아도 사람이 재미없다고 하면 정거장 1 졸업이 아니다. 사람이 재미있다고 해도 수치가 폭주하면 출시 불가다.

## 8. PR4 리텐션 리스크 전용 관찰 프로토콜

`risk-register.md` PR4는 실플레이어 반응이 없이는 닫을 수 없는 리스크다.

**PR4의 두 갈래:**

**(a) 빅 크런치 0.82주 = "짧다" 인식**
- 관찰 포인트: 빅 크런치 도달 시 "이게 끝이야?"라는 반응이 나오는가.
- 탐지 설문: 빅 크런치 후 Q7에서 "끝난 것 같다" 언급 비율을 집계.
- 완화 확인: "시작점" 프레이밍 카피(narrative)가 그 반응을 "새 시작"으로 전환하는가.
- **통과 기준:** "끝이야?" 반응 ≤ 30%, "다음이 기대된다" 반응 ≥ 50%.

**(b) 재하강 권태**
- 관찰 포인트: 2회차 진행 중 "또 똑같네" 발화 시점과 빈도.
- 탐지 설문: Q5("다시 하고 싶다")를 2회차 테스터에게 별도 집계.
- 완화 확인: 회전 집중 서브층 선택이 "다른 결"로 체감되는가.
- **통과 기준:** 2회차 테스터의 "반복적이다" 언급 ≤ 30%.

**PR4가 단계 C에서 불합격이면:**
1. 빅 크런치 프레이밍 실패 → narrative-designer와 UX 카피 재작업.
2. 재하강 권태 실패 → systems-designer와 집중 서브층 차별화 재설계 + qa-balancer 시뮬 재검증.
3. 심화 콘텐츠(scope C1) 조기 투입 검토 (EA 초기 업데이트로).

## 9. 한 줄 요약

> **실플레이어 플레이테스트 = 시뮬이 닫은 수치 안에서 "재미·혼란·체감"이라는 사람의 반응을 단계별로 확인하는 레인이다.** 단계 A(지인 알파, 2~5명)로 코어 후킹을 확인하고, 단계 B(itch.io 클로즈드 베타, 20~50명)로 FTUE를 닫고, 단계 C(Steam Playtest, 100~500명)로 전체 캠페인과 PR4 리텐션을 최종 검증한다. 텔레메트리(수치)와 설문·관찰(이유)을 결합하고, 실측 페이싱을 시뮬 예측과 대조해 두 레인이 모두 통과해야 정거장을 졸업한다.
