## 1. 산출물 1 — 도감 완성도 램프 곡선

### 1-1. 현행 기준선: 선형 곡선

```
bonus(c) = 0.35 × c       (c = codex_completion, 0~1)
```

선형이면 PT4~PT6 구간(도감 완성도 ~60~85% 예상)에서 기여가 0.21~0.30× 수준. 컬렉터 체감은 꾸준하지만 완주 보상 클라이맥스가 없다.

### 1-2. 후보 곡선 4종

아래 모든 곡선은 bonus(0)=0, bonus(1)=0.35, 중간값에서만 차이.

### 1-4. 권장 곡선: 후보 B (제곱 곡선)

```
bonus(c) = 0.35 × c²
holographic_mult += codex_completion² × 0.35
```

**권장 이유:**

1. **필러 ① (수집→힘) 체감**: 초반 수집(c<0.5)에서 보너스 증분이 작아 "아직 힘이 없다" → 후반 완성(c>0.8)에서 급격히 올라가며 "수집이 힘이 됐다"는 체감이 가장 강함. 완주 직전(c=0.9~1.0)에서 +0.028~0.066×의 추가 상승 — 컬렉터의 마지막 입자 발견 클라이맥스와 연동.
2. **중반 페이싱 안전**: c≈0.65 시점 기여 ×1.148 → dec26 단축 −12.7%. 30% 가드레일 대비 17.3%p 헤드룸. 연구 시너지(×1.06 예산)와 합산해도 ×1.148×1.06≈×1.217 → dec26 −17.8% (−30% 충족).
3. **economy 재시뮬 용이**: c²는 미분 가능 단순 다항식. 완성도별 유효 H 계산이 직접적이고 codex_race.py 수정 1행으로 재시뮬 가능.
4. **분모 정의(산출물 3)와 정합**: completion = collected/76으로 정의 시 c²의 증가가 LEGENDARY 11개와 독립 → 분모 변경 없이 곡선만 교체 가능.

**economy 재시뮬 포인트:**

- 곡선 B 기준 "중반 유효 기여 배율" 계산 지점: c=0.65 (PT4 진입 예상 완성도)
  - bonus = 0.35 × 0.65² = 0.148 → holographic_mult 기여 ×1.148
  - dec26 단축 = 1 − 1/1.148 = −12.9% (경제 예산 17.1%p 여유)
- worst-case 재확인 지점: c=1.0, H=×1.35, dec26 단축 −25.9% (기존 codex_race.py 결과 그대로)
- economy 미세조정 밴드 (×1.25~1.35 내): codex_bonus_factor를 0.32~0.35 사이 조정 가능. 0.32이면 완주 H=×1.32, dec26 −24.2%; 0.35이면 −25.9%. 두 경우 모두 −30% 미만.

## 2. 산출물 2 — 베켄슈타인 홀론(L10) 수치 해소

### 2-1. 문제 정의

codex.md §13-3 항목:
```json
{
  "id": "bekenstein_holon",
  "rarity": "EPIC",
  "unlock_condition": "L10 도감 2개 완료 + 도감 수집률 85%",
  "unlock_bonus_type": "도감 완성도 보너스 상한 증가 (베켄슈타인 경계 = 더 많은 정보 저장)"
}
```

"도감 완성도 보너스 상한 증가"가 ×1.35 상한을 올리면:

- economy.md §7.2: 도감 단독 ×1.35가 예산 ×1.43의 −25.9%를 소진 → 연구 시너지 추가 허용치 ×1.06만 남음
- 베켄슈타인이 상한을 추가로 올리면 도감 기여가 ×1.35+ → dec26 단축 −25.9%+로 §3.4 예산 충돌 및 연구 시너지 ×1.06 잠식

### 2-2. 해소안 3종 검토

**안 (a) — 상한은 그대로, 램프를 가파르게 (같은 ×1.35를 더 빨리 달성)**

베켄슈타인 홀론 해금 조건(도감 수집률 85%) 달성 시 bonus_factor를 제곱 곡선(B)에서 계단 가중으로 전환하여, c=0.85 이후 기울기를 높임.

```
c < 0.85:  bonus = 0.35 × c²                  (표준 제곱)
c >= 0.85: bonus = 0.35 × (c² + k×(c−0.85))  (k로 c=1에서 정확히 0.35 도달)
```

단 상한은 여전히 0.35. 결과: 베켄슈타인 해금 후 c=0.85~1.0 구간에서 기여가 더 빨리 오름 → 완주 직전 5~10개 입자의 발견이 더 빠른 보너스 상승으로 체감. **예산 충돌 없음** (상한 불변).

**평가**: 구현 단순, 예산 안전, 베켄슈타인의 "더 많은 정보 저장" 내러티브는 "같은 정보를 더 빨리 처리"로 해석. 필러 ① 충족. 단 "상한 증가"라는 현행 텍스트와 명칭이 어긋나므로 unlock_bonus_type 텍스트를 수정해야 함.

---

**안 (b) — 효과를 다른 축으로 전환 (D 트리클 / 특정 메커니즘)**

베켄슈타인 홀론의 보너스를 holographic_mult 기여가 아닌 별도 축으로 이전:

- 안 b-1: D(발견 데이터) 획득 속도 +X% (베켄슈타인 경계 = 더 많은 정보 저장 → D 생산 가속)
- 안 b-2: 홀로그래픽 인코딩(L10 메커니즘 §2-I) 특정 슬롯 해금
- 안 b-3: 빅 크런치 QF 보너스 계수 소폭 증가 (베켄슈타인 엔트로피 = S = A/4 → QF 보상 증가로 연결)

안 b-3이 내러티브 정합성 가장 강함. 베켄슈타인 경계 = 블랙홀 엔트로피는 표면적(A)에 비례 → "표면에 더 많은 정보를 담으면 QF(양자 거품) 산출이 늘어난다"는 테마 = 메커니즘 연결. 수치: 빅 크런치 QF 계수를 K=1.0 → K=1.05로 소폭 상향. 영향: 재하강 이후 QF 누적 가속 → 재하강 차별화(후속-A1)와 시너지.

**평가**: 예산 충돌 완전 해소, 내러티브 정합 강함, 재하강 차별화와 연동. 단 QF 계수 변경은 economy 재시뮬 필요 (K=1.05 → QF_total = 1.05×(lifetime_C/1e26)^0.5, mult 변화 추정 +2.5% 수준, race 영향 미미).

---

**안 (c) — 소량 상한 증가를 §7.2 헤드룸 내에서 허용**

§7.2: 도감 ×1.35 + 연구 ×1.06 = 총 ×1.431 ≈ ×1.43 한계. 베켄슈타인이 도감 상한을 ×1.35 → ×1.40으로 소폭 올리면:

- 도감 ×1.40 단독: dec26 −28.6% (<−30% 충족, 헤드룸 1.4%p)
- 남은 연구 시너지 예산: ×1.43/1.40 ≈ ×1.021 (연구 시너지 거의 소진)

**수치**: ×1.40 상한은 베켄슈타인 해금 조건(수집률 85%)에서만 활성화. 즉 c=1.0일 때 bonus = min(0.35×c², 0.40). 완주자에게만 추가 +0.05×. dec26 단축 −28.6%는 −30% 바로 아래로 헤드룸이 1.4%p밖에 없어 연구 시너지 노드(A13/B8/B12)가 어떤 값이라도 추가 곱이 들어오면 위반 위험 존재. **economy와 협의 없이 단독 설정 불가.**

**평가**: 가능하나 헤드룸이 매우 좁아 연구 시너지 노드 값 변경 시 즉시 위반. 관리 부담 높음.

---

### 2-3. 권장 해소안: 안 (b-3)으로 전환

**결정 근거:**

1. **예산 충돌 완전 해소**: holographic_mult 경로에서 나와 QF 계수 경로로 이전 → 연구 시너지 ×1.06 예산 잠식 없음.
2. **테마 = 메커니즘(필러 ②) 강화**: 베켄슈타인 엔트로피(S∝A, 표면에 정보가 산다)가 QF(양자 거품, 상전이마다 얻는 영구 통화)와 직결되는 내러티브 논리. 현행 "도감 완성도 보너스 상한 증가"보다 물리 정합성 높음.

## 3. 산출물 3 — 도감 분모 정의 확정

### 3-1. 현황 불일치

### 3-2. 분모별 의미 분석

**87 (전체 엔트리):**
- LEGENDARY 11개(층별 완성 보너스 엔트리)를 포함.
- 이 11개는 "층 76개 중 층별 요건 달성 시 자동 해금"이므로 능동적 발견이 아닌 완성 이벤트로 수여됨.
- 분모=87이면 completion=collected/87이 됨. LEGENDARY 자동 해금이 쌓이며 completion이 올라가 codex_bonus_factor 기여가 자동 증가 → 수집 행동이 아닌 완성 이벤트가 보너스를 올리는 혼선.

**76 (수집 대상):**
- 플레이어가 직접 발견·수집하는 입자만 분모.
- LEGENDARY 11개는 completion 분모 밖, bonus에 영향 없음(자동 해금은 별도 이벤트).
- bonus = 0.35 × (collected_76/76)² → 수집 행동에만 보너스가 연동.

### 3-3. 확정 정의

```
codex_completion = collected_discoverable / 76
```

- **collected_discoverable**: 플레이어가 발견 조건을 충족해 수집한 입자 수 (LEGENDARY 자동 해금 11개 제외).
- **LEGENDARY 11개**: 층 완성 이벤트로 별도 트래킹. completion 분모에 불포함. unlock_bonus는 각 LEGENDARY 엔트리의 "unlock_bonus_type" 필드에 따라 별도 발동.
- **층별 완성도**: layer_completion[L] = layer_collected_discoverable[L] / layer_discoverable_total[L] (층별 수집 대상만). LEGENDARY 자동 해금은 layer_completion이 100%에 도달했을 때 발동하므로 이미 구조 정합.

**정합 확인:**
- codex.md §0 "총 입자 수: 87"은 UI 표시(도감 목록 전체 크기) — 유지.
- codex.md §13-3 "수집 대상 76개 기준: codex_completion = collected_particles / 76" — 확정·유지.
- codex.md §0 line 36: "codex_completion = collected / 87"은 **오기** → 후속 sync 시 "collected_discoverable / 76"으로 수정 필요. (본 문서는 기존 문서 수정 금지이므로 수정 대상만 명시.)

**보너스 상한에 미치는 영향**: 분모가 76이든 87이든 bonus 상한 +0.35×는 completion=1.0 시 동일하게 적용. 분모 정의는 순수 표기·행동 정합 문제로 상한·factor에 영향 없음. (task 명세 확인 — "보너스 상한엔 무영향, 순수 표기 정합" 일치.)
