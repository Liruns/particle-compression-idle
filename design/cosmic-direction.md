# 우주적 현미경 — 통합 방향 문서 (cosmic-direction.md v1.0)

- 작성: game-director
- 작성일: 2026-06-26
- 상태: **확정 피벗 비전의 단일 소스(SSOT).** 비주얼 + 게임플레이/UX + UI 모델 + "무엇이 바뀜" + 이관 진행상태를 한 문서로 통합.
- 역할: 다음 작업자(2단계 구현자)가 **이 한 문서로 방향을 잡는다.** 비주얼 상세값은 `art-direction-cosmic.md`(권위) 참조, 경제·시스템 수식은 `economy.md`/`systems.md`(불변) 참조. 이 문서는 *방향의 지도*.
- 관계: `GDD.md`(비전·필러·경제 SSOT)와 충돌 시 — **방향·표현 = 이 문서 + art-direction-cosmic.md, 경제·메커니즘 로직 = GDD/economy/systems.** 이 문서는 GDD를 대체하지 않고 "표현 레이어"를 확정한다.

## 0. 한 문장

> **다크 대시보드를 버리고, 물질 속으로 떨어지는 "우주적 현미경"으로 전면 피벗한다. 메커니즘·경제·밸런스는 한 줄도 바뀌지 않는다 — 바뀌는 것은 그것이 화면에 드러나는 방식(surfacing)뿐이다.**

피벗의 계기: 기존 다크 대시보드(`overhaul-desktop-wide.png` 류)가 오너에게 **"AI 느낌"** — 모든 개별 선택이 "가장 무난한 정답"이었고, 무난한 정답의 합 = 통계적 중앙값 = 무개성 — 으로 읽혔다. 진단·처방의 비주얼 권위 문서는 `art-direction-cosmic.md`(§0 진단). 이 문서는 그 비전을 게임플레이/UX/UI 모델까지 확장한 통합본이다.

## 2. 비주얼 방향 — "우주적 현미경"

> 상세 스펙(실제 hex·서체 이름·금지목록 10개·머니샷)은 **`art-direction-cosmic.md`가 권위**. 여기서는 방향의 골격만.

### 2-A. 북극성
- **너는 물질 속으로 영원히, 천천히 떨어진다. 작아질수록 더 광막하고 빛난다.** 작음 = 숭고(the sublime). 강함이 아니라 *경외*. 파워포즈가 아니라 심해 하강.
- 세 감정: **명상 · 경외 · 하강.** 이 셋에서 벗어나는 연출은 의심한다.
- 레퍼런스: Osmos / Powers of Ten / 2001 스타게이트(안쪽) / JOURNEY / 형광현미경·체렌코프·심해 생물발광 / 버블체임버. 공통점 = "하나의 감각에 끝까지 헌신."

### 2-B. 11개 distinct 세계 (연속 줌 단일 룩 폐기)
각 층 = **그 스케일의 의도된 구조(형태로 구별)** + **층별 탈채도 발광** + **색온도 아크**(따뜻→차가움→무색):

| # | 층 | 세계 형태 | 발광색 | 색온도 |
|---|---|---|---|---|
| L1 | 분자 | 세포·막·유기물 | `#9fb89a` 흐린 엽록 | 따뜻 (넓은 산란) |
| L2 | 원자 | 전자 오비탈 | `#7fb0b8` 식은 청록 | 따뜻 |
| L3 | 핵 | 밀집핵 | `#d99a6c` 호박 주황 | 따뜻 |
| L4 | 핵자 | 색전하 삼색 | `#c98a9a`/`#7a9fc0`/`#8fc0a0` 탈채도 RGB | 식어감 |
| L5 | 쿼크 | 공허 + 점(응축) | `#d8e0e4` 차가운 흰빛 | 색이 빠짐 |
| L6 | 프리온 | 위상 간섭 | `#8a7fd0` 차가운 보라 | 차가움 (미지 첫 문턱) |
| L7 | 끈 | 진동 필라멘트 | `#c47a9a` 식은 자홍 | 차가움 |
| L8 | 루프 | 격자(스핀 네트워크) | `#5f7fd0` 깊은 청 | 차가움 |
| L9 | 거품 | 진공 요동 | `#8fa8b0` 희박한 은청 | 가장 차갑고 비어있음 |
| L10 | 정보 | 홀로그래픽 경계 | `#5fb0c0` 청록 발광 | 차가움 |
| L11 | 플랑크 | 픽셀 붕괴 | `#f4f6f8`→`#ffffff` 무색 순백 | 무색 |

- **색온도 아크가 "11개 다른 세계"(필러 ④)를 색만으로 만든다.** 현 화면처럼 모든 층이 같은 청록인 것의 정반대.
- **단일 브랜드색 `#3ecf8e` 폐기.** 이 게임에 하나의 브랜드 컬러는 없다 — 층마다 다른 빛이 있을 뿐. 자원도 각자 색(§3-C).
- **공허 = 캔버스.** near-black 배경(스케일 깊어질수록 더 깊어짐) + radial vignette(현미경 시야). 화면의 90%는 어둠. 빛은 *광원에서만* 방출(reflective 아님), 탈채도(채도 35~70%), 작은 halo.

### 2-C. 사이를 잇는 전환 = "떨어지며 이어짐" (플래시 아님)
- 층 전환은 **화이트 플래시가 아니라 연속 블렌드 하강.** 배경 검정이 더 깊어지고, 이전 층 빛이 멀어지며 페이드, 새 층 빛이 아래에서 떠오르며, 줌이 한 번도 멈추지 않고 계속된다. 단절(cut)이 아니라 *통과(passing through)*. 2001 스타게이트의 안쪽 버전.
- **평소 = 관조.** 상시 줌 없음(정적 세계는 머물 수 있는 화면), 번쩍임 0. 전환만 느린 하강(프로토타입 1.7s).
- 모션 철학: 물(water)처럼 — 관성, 연속, 갑자기 멈추지 않음. 앰비언트 = 호흡(중심 빛이 6~8초 주기로 들숨/날숨). 정점조차 *개화*(폭발 아님).
- reduced-motion: 연속 줌·호흡 정지 + 전환 즉시 스냅(멀미 방지, `accessibility.md` §2 준수).

## 3. UI 모델 — "공허가 곧 게임판" (대시보드 폐기)

### 3-A. 폐기되는 것 (전부 SUPERSEDED)
**대시보드 패러다임 전체:** 좌우 대칭 3패널 / 체인 스프레드시트 테이블 / 1px 테두리 라운드 카드 / 얇은 링 게이지 / 거대 볼드 숫자 / 탭 전환=화면 교체. 이것들이 `ux.md`·`ui-flow.md`에 상세히 스펙됐으나 **표현 레이어로서 대체된다.** (그 문서의 *로직* — 진입 조건, 점진 공개, 경제 연동 — 은 §6에서 무엇이 이어지는지 명시.)

### 3-B. 새 모델 — 관조 ↔ 공허에 손 뻗기
명상적 톤과 방치형 정보 밀도의 충돌을, "정보를 다 카드에 담는" 대신 두 모드로 푼다:

**관조 모드 (default — 화면이 비어있을 때):**
- 화면을 지배하는 것은 **중앙의 하강하는 물질(현재 층 세계) + 유령 같은 최소 바이탈.** 나머지는 어둠. 심해 관측창처럼 — 살아있되 조용하다.
- 자원(E/C/D/QF)은 카드 박스가 아니라 **공허 가장자리에 뜬 희미한 성표(星表) 주석(annotation).** 좌표 옆 측정값처럼. 테두리·배경 없음, 어둠 위에 직접 뜬 발광 텍스트.
- 위계는 크기가 아니라 **밝기와 거리**로. 중요한 것이 더 가깝거나 더 밝다.

**개입 모드 (손을 뻗을 때):**
- 장치(8단 체인 / 연구 / 도감 / 상전이)는 평소 화면에 없다. **가장자리에 발광 노드로 잠들어 있다.** 다가가거나 부르면 *피어나고(bloom)*, 볼일이 끝나면 다시 어둠으로 *물러난다(recede)*.
- **핵심 은유: 대시보드에 들어가는 게 아니라, 공허에 손을 뻗는다.** 정보는 다 있다 — 단지 항상 *동시에* 떠 있지 않을 뿐.

### 3-C. 자원·엔진의 다이제틱 표현
- **자원 색 = 각자 다른 발광체**(단일 악센트 해체, 모두 탈채도): E `#d9b86a`(식은 금) / C `#7a9fc0`(차분한 청) / D `#9a8fc0`(흐린 보라) / **QF = 현재 층 발광색을 따라감**(고정색 없음 — "지금 얼마나 깊은가"가 곧 QF의 빛).
- **8단 체인/엔진 = 입자 구조.** 스프레드시트 테이블이 아니라, 흡수한 입자가 **궤도 껍질로 쌓이는** 동심 궤도/켜의 빛. 각 티어 = 중심에서 멀어지는 한 겹의 발광 링. 구매하면 그 켜가 밝아지고 회전이 빨라진다. *생산이 시각적으로 중심을 키운다* = "작아짐=강해짐"의 직접 구현. 테이블은 hover/개입 시 보조로만.
- **수치 = 작고 조용하게.** 거대 볼드 숫자 폐기. 1~2초 부드러운 갱신(LED 명멸·롤링 카운터 금지). 과학 표기 유지(`1.616×10⁻³⁵ m`).
- 비대칭 레이아웃, UI 위젯 총 면적 < 화면의 30%, 중앙 물질이 주인공.

## 4. 능동 메커니즘 — "떠다니는 입자 직접 만지기"

능동 손맛(수동압축·오비탈공명·위상겹침·하모닉스 등)은 **이미 전부 물리-공간적**이다. 표현을 슬롯/버튼/토글에서 **입자 직접 조작**으로 옮긴다. *메커니즘은 동일* — 입력 표면만 바뀐다:

| 메커니즘 (로직 불변) | 기존 표현 (폐기) | 새 표현 (다이제틱) |
|---|---|---|
| 수동 압축 | `[압축]` 버튼 | **물질을 직접 만져 압축** (떠다니는 덩어리를 누르면 응축) |
| 오비탈 공명 | 슬롯 3개 + 카운트다운 링 | **궤도를 도는 전자를 공명 타이밍에 클릭** |
| 위상 겹침 (프리온) | `[응집][분산][공명]` 토글 버튼 | **간섭 무늬의 위상을 직접 겹쳐 맞춤** |
| 진동 하모닉스 (끈) | 게이지 + 스케줄 미리보기 | **진동하는 필라멘트를 직접 튕겨 하모닉 맞춤** |

- **관조 ↔ 개입의 손맛:** 평소엔 입자가 떠다니는 것을 *지켜보다가*, 개입할 때 그것을 *만진다.* 클릭이 "버튼 누르기"가 아니라 "물질에 손대기"로 느껴지게.
- 입력→효과 매핑(배율값·지속시간·쿨다운)은 `economy.md`·`systems.md` §2가 권위. 표현이 그 값을 바꾸지 않는다.

## 5. 정합 — 보이스·색온도·머니샷

- **색온도 아크(따뜻→차가움→무색) = 보이스 아크(확인→경이→충격→침묵→붕괴)와 1:1.** 미지로 갈수록 빛이 식고, 연구소장의 목소리도 신중·간결해진다(`narrative.md` §1-C).
- **머니샷:**
  - **첫 상전이(쿼크→프리온, dec19)** = 알려진 물리→미지의 유일한 문턱. 가장 긴 머니샷. 따뜻한 빛이 마지막으로 식어 *완전히 빠지고*, 차가운 미지의 보라(`#8a7fd0`)가 어둠에서 처음 떠오른다 (narrative §4 비트 5와 1:1).
  - **빅 크런치** = 최정점. 순흑 → 침묵 → 재압축. 느낌표 없이 빛 하나가 다시 켜진다. **"종착점 언어" 금지**(클리어/완료/엔딩) — "재압축/더 작은 것이 있다" (narrative §5-B, `followup-bigcrunch-framing.md`).
- 스토어 후킹: "끝없는 하강" GIF, "같은 화면, 다른 세계" 3컷, "첫 문턱" 상전이 GIF, "빅 크런치" 정지샷. 시끄러운 EDM/데모 금지 — 침묵과 저음 (art-direction-cosmic §8).

## 6. 경제·시스템 — 불변, 단 surfacing만 피벗

**이 게임의 모든 수식·비용곡선·페이싱·밸런스는 한 줄도 바뀌지 않는다.** `economy.md`(수식·시뮬·v0.3 §7 후속 패스)·`systems.md`(메커니즘 구조)는 **그대로 유효**하다. 검증된 것:

- 코어 수식: `dec=α·log₁₀C`(α=0.65), `cost_k=base_k·growth_k^n`(base_k=10^(1+1.3k), growth 2.2→1.8), `QF=floor((lifetime_C/1e26)^0.5)`, 부스트 `1+0.25·log₁₀(1+QF)`.
- 페이싱: WALLS=[19,21.5,23,24.5,25.5,26], 첫 상전이 4.26h, 빅 크런치 진입 2.76×, 각 미지 서브층 체류 ≥4h.
- 후속 경제(R3): 도감 곡선 B(`0.35·c²`, ≤×1.35), 연구 시너지 C안, D보존(lifetime 100%/current 50%, 회차곡선), 베켄슈타인 K=1.05, 재하강 차별화(회전 집중 서브층 + 방치 ~5.8일/개입 ×0.80).

**바뀌는 것은 오직 *surfacing*:** 이 값들이 화면에 드러나는 방식이 "대시보드 위젯/테이블/게이지"에서 "다이제틱 입자 상호작용/공허 주석/궤도 껍질"로 옮겨간다. 어떤 surfacing 작업도:
- 수식·임계값·배율을 바꾸지 않는다.
- `data/`에 박힌 입자·노드 수치를 바꾸지 않는다.
- 검증된 페이싱(시뮬 dt=1~2s 결정적 가정)을 깨지 않는다.

> **무엇이 이어지는가(잃지 말 것):** `ux.md`·`ui-flow.md`의 *로직* 정보 — 탭/화면 진입 조건, FTUE 점진 공개 순서, 상전이 점등 조건(미지 6벽+빅크런치에만), 오프라인 복귀 요약, dec2~3 험프 넛지 — 는 **경험 설계로서 유효**하다. 단 그 *표현 컨테이너*(3패널·카드·테이블)가 공허 게임판으로 대체된다. 2단계 구현 시 "이 정보를 공허 모델에서 어떻게 드러낼까"로 재해석하되, 정보 자체(언제 무엇이 보이는가)는 보존한다.
