## 0. 읽는 법

**단계** 열은 에셋이 처음 필요해지는 시점.

| 단계 코드 | 뜻 |
|---|---|
| `P` | 프로토타입 (단계 1) |
| `VS` | 수직 슬라이스 (단계 2) |
| `V1` | v1.0 Steam (단계 3) |
| `POST` | 출시 후 (단계 4) |

**우선순위** 열: `★★★` = 블로커(없으면 해당 단계 불가) / `★★` = 중요(없으면 해당 단계 품질 저하) / `★` = 있으면 좋음.

**담당** 열: `art` = art-director + graphics-programmer / `audio` = audio-designer + graphics-programmer / `content` = content-designer + narrative-designer / `tech` = tech-architect + game-programmer.

**상태** 열(팀이 채워 넣음): `[ ]` 미착수 / `[~]` 진행 중 / `[x]` 완료.

## 5. 가장 무거운 제작 구간 — 리스크 상위 5

> "분량이 가장 많거나, 병목이 되거나, 전문 외주·협업이 필요한 구간."

**1. 오디오 앰비언트 스템 제작 (전체)**
11층 × 스템 2~4레이어 = 25~35개 오디오 파일. 각 파일이 60~120s 심리스 루프 + 도입 스웰 별도. 아날로그 신스·모듈러·물리 모델링 소스 확보 필요. AI 슬롭 금지 원칙이 상용 팩 사용을 막음. **발주→QA까지 리드타임이 가장 긴 카테고리.** VS 단계에 3루프가 먼저 필요하므로 프로토타입 완료 직후 즉시 발주 필요.

**2. 내러티브 텍스트 — 도감 87입자 플레이버**
입자당 1~2줄이지만 87개. "AI 슬롭 금지" + "실제 물리 정확 + 경이" 조건이 각각 수작업. narrative-designer가 물리 데이터를 먼저 확인해야 쓸 수 있으므로 content-designer → narrative-designer 의존 체인. 이 체인이 느리면 VS 단계 도감 카드가 빈 상태로 나간다.

**3. 비주얼 — 층 배경 질감 9개 (V1 단계)**
L3~L5·L7~L11 총 9층의 배경 질감을 graphics-programmer가 Canvas2D/WebGL로 구현. 각 층이 고유 질감(Perlin noise, 동심원, 삼각 분할, 격자, 버블 등). 60fps 성능 예산 공유 + 파티클과 동시 실행 부하 점검 필요. 한 층 당 구현→QA→polish 사이클이 예상보다 길어지기 쉬움.

**4. SFX 변주 풀 — 클릭 12개·구매 16개 + 층 연동 변종**
총 ~100개 SFX 파일. 핵심은 "같은 클릭이 두 번 똑같이 나지 않는다"는 원칙(라운드로빈+피치 변주). 파일 수보다 **변주 품질과 런타임 라운드로빈 로직** 구현이 무게. audio-designer(제작)·graphics-programmer(재생 로직)·voice cap·디바운스 설계가 한 스프린트에 묶임.

**5. 콘텐츠 — 연구 트리 52노드 수치 확정**
economy-designer의 effect_value·cost_D 52세트가 확정돼야 content-designer가 데이터를 채울 수 있고, QA가 "2회차 즉시 재구매 ≥5노드" 검증을 할 수 있다. 이 체인이 늦으면 V1 단계 연구 탭 전체가 블로킹됨. 52노드 중 핵심 36노드(A+C 전량+B/D 코어)부터 우선 확정 권장.
